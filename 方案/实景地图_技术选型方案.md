# 实景地图 · 技术选型与集成方案

> 目标：在现有仪表盘中嵌入一个轻量 2D 瓦片地图，让线下空间的状态（房间/田地/设施）可视化，线下行动（打扫/种植/收获）同步到线上标记。

---

## 一、我们要做什么

### 1.1 核心场景

```
线下世界                          线上地图
────────                          ────────
素社 101 房间打扫完成     →        🟢 101 标记为「已打扫」
菜地 A 区种了番茄          →        🍅 图标出现在 A 区，显示「生长中 · 还剩 45 天」
菜地 B 区玉米成熟了        →        🌽 图标闪烁「可收获」
收获玉米 30 根             →        线上记录「B 区 · 玉米 · 30根 · 7/20」
```

### 1.2 不是 AR，是「线下做、线上记」

不做摄像头识别（太重）。而是：线下完成动作 → 手机打开网页 → 在地图上点一下标记状态变化。地图和现实空间的对应关系靠**人的认知**（"我在 A 区种的"），不靠 GPS 定位。

### 1.3 地图上的元素

| 元素 | 示例 | 交互 |
|------|------|------|
| 建筑/房间 | 🏠 素社、🎨 画室、🖼️ 展览厅 | 点击 → 显示状态（已打扫/待打扫/使用中） |
| 田地/种植区 | 🥬 A区、🍅 B区、🌽 C区 | 点击 → 显示作物、种下日期、预计成熟日 |
| 设施 | 🚿 浴室、🍳 厨房、📚 图书角 | 点击 → 显示维护状态 |
| 路径 | 连接各区域的走道 | 纯视觉 |

## 二、推荐的开源项目

### 2.1 🥇 首选学习对象：Sprite Garden

| | 详情 |
|------|------|
| **仓库** | [github.com/kherrick/sprite-garden](https://github.com/kherrick/sprite-garden) |
| **在线体验** | [kherrick.github.io/sprite-garden/](https://kherrick.github.io/sprite-garden/) |
| **技术** | 纯 HTML5 Canvas + JavaScript，**零依赖** |
| **大小** | 单文件即可运行，极其轻量 |
| **核心功能** | 2D 瓦片地图、种植/收获作物、挖矿、放置方块、内置地图编辑器 |

**我们为什么选它**：
- 零依赖 → 一个 `<script>` 标签就能嵌入现有 dashboard
- 已有种植/收获机制 → 直接把"作物生长时间"改成我们需要的时间计算
- 内置地图编辑器 → 可以用来画出素社/画室/田地的布局
- 纯 Canvas → 手机浏览器完美支持，加载 < 1 秒
- 代码可读性高 → AI 能直接学习和修改

## 三、技术方案：怎么嵌入现有仪表盘

### 3.1 不引入新框架

现有 dashboard 是**纯 HTML + CSS + JS，零构建工具**。保持这个原则——地图也用纯 Canvas 实现，作为 dashboard 的一个新标签页或组件嵌入。

### 3.2 架构

```
dashboard.html
    │
    ├── 现有：data 对象（tasks, members, budget, schedule...）
    │
    └── 新增：data.world_map = {
          tiles: [          // 瓦片地图数据
            { x: 0, y: 0, type: 'grass', building: null, crop: null },
            { x: 3, y: 5, type: 'room', building: '素社101', status: 'clean' },
            { x: 8, y: 2, type: 'field', crop: { name: '番茄', planted: '2026-07-15', harvest_days: 45 } },
          ],
          rooms: [          // 房间/建筑定义
            { id: '素社101', name: '素社 101 室', type: '住宿', size: '4×3', responsible: '朝林' },
            { id: '画室',    name: '画室',       type: '创作', size: '6×4', responsible: '若曦' },
          ],
          fields: [         // 田地定义
            { id: 'A区', name: 'A区·番茄', crop: '番茄', planted: '2026-07-15', harvest: '2026-08-29' },
          ]
        }
```

### 3.3 新增标签页

在 dashboard 现有标签栏加一个标签：

```
📝营地设置 | 📊任务总览 | 📋任务大厅 | 🗺️攻略地图 | 🌍实景地图 | 👥策划组 | ...
```

`🌍 实景地图` = 新的 Canvas 标签页。共建者和冒险者都可以看到（共建者可以编辑，冒险者只能查看）。

### 3.4 地图绘制

从 Sprite Garden 学习核心逻辑，简化为：

```javascript
// 画 2D 瓦片地图
function renderWorldMap() {
  var canvas = document.getElementById('worldMapCanvas');
  var ctx = canvas.getContext('2d');
  var tileSize = 32; // 每个格子 32px，手机上也能看清

  data.world_map.tiles.forEach(function(tile) {
    var x = tile.x * tileSize;
    var y = tile.y * tileSize;

    // 画地板
    ctx.fillStyle = getTileColor(tile.type);
    ctx.fillRect(x, y, tileSize, tileSize);

    // 画建筑
    if (tile.building) {
      drawBuildingIcon(ctx, x, y, tile.building, tile.status);
    }

    // 画作物
    if (tile.crop) {
      drawCropIcon(ctx, x, y, tile.crop);
    }
  });
}
```

### 3.5 交互：点击标记状态

```
点击房间 → 弹出状态面板：
┌─────────────────────┐
│ 🏠 素社 101 室        │
│ 状态：🟢 已打扫       │
│ 负责人：朝林          │
│ 上次打扫：7/15        │
│                      │
│ [标记为待打扫]        │
│ [标记为已打扫]        │
│ [添加备注]           │
└─────────────────────┘

点击田地 → 弹出作物面板：
┌─────────────────────┐
│ 🥬 A区·番茄           │
│ 种下：7/15            │
│ 预计成熟：8/29（还剩 43 天）│
│ 状态：🌱 生长中       │
│                      │
│ [标记已收获]          │
│ [记录收获数量]        │
│ [添加新种植]          │
└─────────────────────┘
```

### 3.6 时间计算 + 提醒

每个作物有 `planted`（种植日）+ `harvest_days`（成熟天数）。`harvest = planted + harvest_days`。

系统每天检查：哪些作物今天或之前成熟了但还没收获 → 在共建者监察面板的「今日焦点」中提醒：

```
🔔 菜地提醒：A区·番茄 今天成熟！B区·玉米 已逾期 3 天未收！
```

这个检查在页面加载时跑一次即可，不需要后台定时任务。

## 四、集成计划

### 第一阶段：静态地图（先画出来）

- 学习 Sprite Garden 的 Canvas 瓦片渲染代码
- 用硬编码的 `data.world_map` 画出素社/画室/田地的初始布局
- 嵌入 dashboard 作为新标签页
- 手机打开能看、能缩放（触摸手势）

### 第二阶段：交互标记（能点了）

- 点击房间 → 弹出面板 → 切换状态
- 点击田地 → 弹出面板 → 记录种植/收获
- 状态变化写入 `data.world_map` + localStorage 持久化

### 第三阶段：时间提醒（自动提示）

- 作物成熟日计算
- 共建者监察面板加「菜地提醒」
- 逾期未收标红

### 第四阶段：地图编辑器（共建者自己画）

- 参考 Mapvania 的编辑器交互
- 共建者可以拖放房间、田地、路径图标
- 导出的 JSON 存入 `data.world_map`

## 五、代码库使用策略

| 代码库 | 学什么 | 怎么用 |
|--------|--------|--------|
| **Sprite Garden** | Canvas 瓦片渲染、种植/收获机制 | 提取 `render` 和 `crop` 相关函数，简化后嵌入 |
| **GameFarmerJS** | 极简农场逻辑、MVC 架构 | 参考其 `Game` 类和 `Map` 类的结构 |
| **Mapvania** | 地图编辑器 UI、JSON 导出 | 参考其房间拖放和瓦片选择的交互 |
| **桃源乡** | 作物生长时间计算、中文UI | 参考其 `cropGrowth` 逻辑和界面文案 |

**使用原则**：不复制整个项目，只提取需要的函数和逻辑。AI 帮我读懂 → 提取 → 适配到 dashboard 的数据结构中。

## 六、为什么不选 AR / 3D 方案

| 方案 | 问题 |
|------|------|
| AR.js / 摄像头识别 | 需要 HTTPS、摄像头权限、iOS 兼容性差、加载重 |
| Three.js 3D 地图 | 手机性能消耗大、开发复杂度高、与现有 dashboard 风格不搭 |
| GPS 定位 | 精度不足（民用 GPS 误差 5-15 米），无法区分相邻房间 |

我们的场景不需要这些。2D 瓦片地图 + 手动标记 = 最轻量、最可靠、最快出成果。
