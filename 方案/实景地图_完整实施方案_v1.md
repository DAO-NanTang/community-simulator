## 二、三层详解

### 第一层：大地图（鸟瞰图）

**用途**：展示各建筑的相对位置、路网、周边环境。面向访客和成员，快速了解"合作社在哪里、附近有什么"。

**视觉风格**：
- 暗色背景 + 文字画 stamp（纯 Unicode 字符）
- 道路用 box-drawing 线（/ \ │ ─ + ┌ ┐ └ ┘）
- 建筑用文字画 stamp（简化鸟瞰图标，5-9 字符宽）
- 可缩放平移

**交互**：
- 点击建筑 → 进入该建筑的格局图（第二层）
- 点击田地 → 进入田间管理面板
- 管理员可在编辑模式下移动 stamp、画路
- 成员头像（圆形标记）可在地图上显示当前位置

**数据**：
```javascript
data.world_map = {
  grid: { rows: 40, cols: 50 },
  tiles: [
    { r: 10, c: 20, char: "┌", color: "c-wall", stampId: "bld_pingfang_l", instanceId: "si_001" },
    // ... 每个格子存储一个字符 + 颜色 + 所属 stamp
  ],
  instances: [
    { id: "si_001", stampId: "bld_pingfang_l", row: 8, col: 18, label: "合作社大院" },
    // ... 每个放置的 stamp 是一条记录
  ],
  roadCells: [
    { r: 15, c: 20, char: "─", color: "c-path" },
    // ... 道路是特殊的非 stamp 格子
  ]
};
```

### 第二层：格局图（建筑平面布局）

**用途**：展示一个建筑内部的楼层平面图——房间分割、走廊、门的位置。面向管理者和成员，了解"这个建筑里有什么房间"。

**视觉风格**：
- 与大地图同一套渲染引擎（文字画 + 颜色）
- 但比例放大（每个字符 ≈ 0.5 米）
- 屋顶遮盖效果：
  - 瓦房顶层之上叠一层 `/\` 瓦片纹理
  - 平房顶层之上叠一层 `===` 平台纹理
- 楼层切换标签：`[1F] [2F] [3F] [屋顶]`

**编辑模式**：
- 画墙：拖拽画线，自动生成墙壁字符（│ ─ + ┌ ┐ └ ┘）
- 分割房间：在墙围成的区域内点击 → 标记为房间 → 输入房间名
- 放门/窗：在墙线上放门标记（▐）或窗标记（▢）
- 标注楼梯：放置楼梯标记（≈≈ 或 ▓▓）

**数据**：
```javascript
data.buildings["bld_coop"] = {
  name: "合作社大院",
  floors: 3,
  floorPlans: {
    1: {
      walls: [
        { r1: 0, c1: 0, r2: 0, c2: 20 },  // 横墙 从(0,0)到(0,20)
        { r1: 0, c1: 0, r2: 1, c2: 0 },    // 竖墙 从(0,0)到(10,0)
        // ...
      ],
      rooms: [
        { id: "r001", name: "大客厅", r: 1, c: 1, w: 8, h: 5, type: "活动" },
        { id: "r002", name: "厨房",   r: 1, c: 10, w: 4, h: 3, type: "设施" },
        { id: "r003", name: "储藏室", r: 5, c: 10, w: 4, h: 2, type: "储藏" },
      ],
      doors: [
        { r: 0, c: 10, direction: "horizontal" },  // 门在墙上的位置
      ],
      background: null  // 可选：上传的平面图照片（floor-planner 模式）
    },
    2: { /* 同上结构 */ },
    3: { /* 同上结构 */ }
  },
  roofType: "flat",  // "flat" | "tile"
};
```

**墙线字符自动选择**（参考 homeRoughEditor 的 getWallNode）：
```
墙的一头连着另一面墙 → ├ ┤ ┬ ┴ ┼
墙的两头都连着墙   → ┼
墙的孤立端点       → ┌ ┐ └ ┘
墙的中间段         → │（竖）─（横）
```

### 第三层 A：物品管理（房间物品列表）

**用途**：记录每个房间里有什么物品、数量和状态。纯数据管理，不需要图形化。

**形式**：点击格局图中的房间 → 弹出物品列表面板。

```
┌─────────────────────────┐
│ 📦 101 宿舍 · 物品清单    │
│                         │
│  🛏️ 上下铺床    ×2  良好 │
│  📚 书桌        ×2  良好 │
│  💡 台灯        ×1  损坏 │  ← 红色
│  🪑 椅子        ×4  良好 │
│                         │
│  [+ 添加物品]           │
│  [保存] [取消]          │
└─────────────────────────┘
```

**数据**：
```javascript
data.rooms["r001"] = {
  name: "101 宿舍",
  items: [
    { name: "上下铺床", qty: 2, status: "good", note: "" },
    { name: "台灯",     qty: 1, status: "broken", note: "灯泡坏了" },
  ]
};
```

**物品状态**：`good`（良好）| `worn`（老旧）| `broken`（损坏）| `missing`（缺失）

**生长阶段自动计算**：
```javascript
function getCropStage(cropName, plantedDate) {
  const preset = data.crop_presets[cropName];
  const elapsed = daysBetween(plantedDate, today());
  let accumulated = 0;
  for (const stage of preset.stages) {
    accumulated += stage.days;
    if (elapsed < accumulated) return stage.name;
  }
  return "已成熟";
}
```

## 四、技术选型

| 模块 | 技术 | 参考 |
|------|------|------|
| **大地图渲染** | DOM 网格（table）+ CSS | 已有 editor.html 可复用 |
| **格局图渲染** | 同上 DOM 网格 | 同一套渲染引擎 |
| **墙线绘制** | 拖拽画线 + 节点检测 | 参考 homeRoughEditor/editor.js 的 wall 和 getWallNode |
| **大地图 stamp 编辑** | 参考 ASCIIFlow 的 Tool 架构 | ASCIIFlow：Box/Line/Text/Select 工具模式 |
| **缩放平移** | CSS transform + 触摸事件 | 已有实现 |
| **数据持久化** | localStorage + JSON 导出 | 已有实现 |
| **UI 风格** | 复用 dashboard 的 CSS 变量 | 暖色系、暗色背景 |
| **作物计算** | JS Date 差值 | 纯前端计算，无需后端 |

**不引入新框架** — 全部纯 HTML + CSS + JS，与现有 dashboard 保持一致。

