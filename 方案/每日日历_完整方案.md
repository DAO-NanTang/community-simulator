## 一、功能概述

步骤①「统筹定框架」中的每日安排模块。用户设置开营日 + 天数 + 选模板 → 一键生成完整日历。日历支持整列拖动、复制、增删天数。锚点日（开营/合照/展览/结营）自动匹配。确认天数后生成里程碑。

| 锚点 | 位置 | 锁定 |
|------|------|------|
| 开营日 | D1 | 日期不可移，内容可微调 |

### 列选中与操作

点击列内容区（不含日期头）→ 整列高亮 → 出现按钮：

```
  [在此列前插入一天] [在此列后插入一天]
  插入 = 复制整列内容到新列，日期自动顺延
```

拖住列内容 → 拖到另一列位置 → 整列互换。日期行始终按时间顺序不变。

### 增加天数后

```
📐 初始设置：7/20 开营 · 15 天
📐 当前实际：7/20 开营 · 17 天（+2天）
[🔮 确认天数并生成里程碑 →]
```

确认后 → 里程碑重算 → 18个节点日期刷新 → 锚点重新匹配。

## 六、数据模型

```javascript
// 向导专用，步骤⑤同步到 data.schedule
data.wizSchedule = {
  dates: ['07/20','07/21',...],
  weekdays: ['周一','周二',...],
  slots: [
    { section: '🌅 晨起·南塘三晨', time: '6:30-7:30', activities: [...] },
    // ... 11 slots
  ]
};

// 模板存储（camp_templates, pool: 'schedule'）
{
  "第三期日程模板": {
    pool: "schedule",
    slots: [ /* 11 slots with new names */ ],
    anchors: {
      day1: [ /* 11 activities */ ],
      photo_day: { offset: "mid", activities: [ /* 11 */ ] },
      exhibition_prep: [ /* 3 days × 11 */ ],
      exhibition_day: { offset: "end-1", activities: [ /* 11 */ ] },
      last_day: { offset: "end", activities: [ /* 11 */ ] }
    },
    middle_days: [ /* 各非锚点天的11项活动内容 */ ]
  }
}
```

## 七、生成逻辑

```
function generateCalendar(startDate, duration, tplName) {
  1. 从 camp_templates 加载模板
  2. 确定锚点位置：
     day1 = 0
     photoDay = Math.floor(duration / 2)
     exhibitionDay = duration - 2   // 结营-1
     lastDay = duration - 1
     exhPrep = [exhibitionDay-3, exhibitionDay-2, exhibitionDay-1]
  3. 创建 11 个 slot，每个 activities[duration]
  4. 填入锚点内容
  5. 非锚点天从 middle_days 依次复制，标注「📋」
  6. 存入 data.wizSchedule
  7. 渲染表格
}
```

## 八、关键约束

1. 改日期/天数不自动生成日历，必须点按钮
2. 生成前已有数据 → 弹确认
3. 向导数据独立于日历系统（`data.wizSchedule` ≠ `data.schedule`），步骤⑤同步
4. 日期行不可选中不可拖，仅内容列可操作
5. 锚点日内容不可整列拖动
6. 增加天数 > 模板天数 → 超出部分显示空，可手动填
7. 保存模板时锚点 offset 重新计算
