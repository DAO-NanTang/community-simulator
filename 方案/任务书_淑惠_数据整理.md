# 📋 任务：共创营仪表盘数据整理与教学

## 任务目标

梳理「南塘 DAO 共创营仪表盘」（dashboard.html）的完整数据架构，让这个静态网页能够更规范地存储信息、读取信息，并让砚仁能够独立管理和维护。

## 工作内容

### 1. 数据盘点（核心）
- 梳理 `data` 对象的所有顶层字段（tasks、members、budget、finance_cny、schedule、camp_dates、camp_progress、bundles、staff_cards 等）
- 标注每个字段的来源（哪个步骤产生）和去向（哪个标签页使用）
- 画出数据流向图：步骤①-⑤ → 各标签页的数据对应关系

### 2. 存储机制梳理
- 当前数据以 JSON 格式存入浏览器 localStorage（key：`camp_data`）和模板库（key：`camp_templates`）
- 梳理导出/导入流程：`exportData()` → JSON 文件 → `loadData()` 恢复
- 评估是否需要补充：数据版本号、备份机制、旧格式兼容

### 3. 信息整理
- 整理当前预设模板的内容清单（10 个任务模板 + 日程模板 + NT 设计模板）
- 整理各标签页的渲染函数与数据依赖关系
- 标注哪些数据是「营前设定」哪些是「营中新增」哪些是「营后归档」

### 4. 教学示范（腾讯会议）
- 演示如何通过浏览器开发者工具（F12 → Console）查看和修改数据
- 演示如何导出 data.json、如何加载、如何备份
- 演示如何在营地设置中走完 5 步流程并验证数据正确性
- 演示如何新增模板和保存

## 产出要求

1. 一份数据字段清单（Markdown 或表格），标注每个字段的类型、来源、去向、示例值
2. 一份数据流向简图（手绘拍照或 draw.io 均可）
3. 腾讯会议 30-40 分钟的教学示范

## 验收标准

- 砚仁能够独立：导出数据 → 修改数据 → 重新加载 → 验证页面正确渲染
- 砚仁能够独立：新建一个模板 → 使用模板 → 提交到任务大厅
- 砚仁理解每个标签页背后的数据来源
