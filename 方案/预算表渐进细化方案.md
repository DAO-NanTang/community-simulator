# 预算表渐进细化方案

> 设计目标：预算表不是一次性填完的静态表单，而是随着 5 个步骤的推进，从粗略框架逐步「长成」详细预算表的过程。最终形态参考 Excel 表《项目预算与人力经费明细表》。

---

## 一、渐进细化的五个阶段

```
步骤①          步骤②          步骤③④            步骤⑤
粗略框架  →   任务NT确定   →  人员认领+工资  →   预算锁定
(估计值)      (精确NT)       (名字+个人NT)      (最终版)
```

### 阶段 1：步骤① — 粗略框架

**输入**（用户填）：
- 员工人数
- 共创人人数
- 早鸟价/占比/原价
- 住宿单价（¥30 + NT40）/人/天
- 吃饭单价（¥8 + NT10）/人/天
- 回购画作 NT300/人
- 课程任务 NT100/人/天（粗估）
- 赏金池 RMB / NT

**预算表显示**：

|------|-----|-----|------|
| 学费收入 | ¥5,990 | — | 早鸟×N₁+原价×N₂ |
| **收入合计** | **¥5,990** | — | |
| 人工费 | ¥12,000 | — | 员工×2000 |
| 员工NT工资 | — | — | 待认领后填入 |

### 阶段 2：步骤② — 任务NT精确化

**触发**：用户在步骤②完成统一任务池梳理后

**变化**：
- 「课程任务(粗估)」→ 替换为「任务NT合计」，数值 = 所有任务激励点之和
- 如果任务分主线/支线/共创/隐藏，自动展开子行

- 「员工NT工资」从 `--` 变为实际值
- 任务按角色拆分到对应的人名下

## 二、输入区域布局（两行对称）

```
┌─ 📐 输入参数 ───────────────────────────────────────────────────┐
│                                                                  │
│  第1行：👥员工 [6]人  RMB工资[2000]/人 │ 🎨共创人[10]人  早鸟[499]×[60]%  原价[599] │ 💰资助(自动算) │
│                                                                  │
│  第2行：🏠住宿 ¥[30]+NT[40]  吃饭 ¥[8]+NT[10] /人/天 │ 🎁回购[300]NT/人  课程[100]NT/人/天 │ 🔴赏金RMB[3000]  NT[4500] │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

对称逻辑：
- 第1行 = 人（员工参数 + 共创人参数 + 学费参数）
- 第2行 = 钱（食宿单价 + NT单价 + 赏金池）

## 三、技术实现方案

### 3.1 数据结构

所有预算相关数据存在 `data.budget` 下，随着步骤推进逐步填充：

```javascript
data.budget = {
  // === 步骤①填入 ===
  staff_count: 6,
  participant_count: 10,
  salary_base_rmb: 2000,
  early_bird_price: 499, early_bird_pct: 60, full_price: 599,
  lodging_rmb: 30,    // 住宿RMB单价
  lodging_nt: 40,     // 住宿NT单价
  meal_rmb: 8,        // 吃饭RMB单价
  meal_nt: 10,        // 吃饭NT单价
  buyback_nt: 300,    // 回购画作NT/人
  task_nt_estimate: 100, // 课程任务NT粗估/人/天
  rmb_pool: 3000,
  nt_pool: 4500,

  // === 步骤①自动计算 ===
  tuition: 5990,           // 学费收入
  salary_rmb_total: 12000, // 人工费RMB合计
  lodging_rmb_total: 5700, // 食宿RMB合计
  lodging_nt_total: 7500,  // 食宿NT合计
  buyback_nt_total: 3000,  // 回购NT合计
  sponsor_needed: 14710,   // 需申请资助（自动）

  // === 步骤②填入（从任务池汇总）===
  task_nt_total: 14450,
  task_nt_breakdown: {
    '主线': 8800, '支线': 2850, '共创': 2800, '隐藏': 0
  },

  // === 步骤③④填入（从认领和卡片）===
  staff_salaries: [
    { name: '砚仁', role: '总协调', rmb: 2700, nt: 4500 },
    { name: '小红', role: '宣传',   rmb: 1800, nt: 3000 },
    // ...
  ],

  // === 步骤⑤最终锁定 ===
  // material/activity costs from task cards
  materials: [
    { item: '课程材料', rmb: 800, nt: 0 },
    { item: '小红书推广', rmb: 1200, nt: 0 },
    // ...
  ]
};
```

### 3.2 渲染逻辑

`renderBudgetCard()` 根据 `data.camp_progress.step` 决定展示粒度：

```javascript
function renderBudgetCard() {
  var step = data.camp_progress.step || 1;
  
  // 输入行：始终显示
  renderBudgetInputs();
  
  // RMB/NT 结果区
  if (step >= 1) renderBudgetRough();     // 粗略框架
  if (step >= 2) renderBudgetTasksNT();   // + 任务NT明细
  if (step >= 3) renderBudgetStaffDetail(); // + 人员工资明细
  if (step >= 5) renderBudgetFinal();     // + 物料活动明细
  
  // 总预算行
  renderBudgetSummary();
}
```

### 3.3 数据流入时机

| 触发事件 | 写入 budget 字段 | 预算表变化 |
|---------|-----------------|-----------|
| 步骤①保存 | `staff_count`, `participant_count`, 所有单价, 自动计算值 | 显示粗略框架 |
| 步骤②保存 | `task_nt_total`, `task_nt_breakdown` | 课程任务→精确NT，展开子行 |
| 步骤③④填写 | `staff_salaries[]` | 人工费展开为每人一行，员工NT出现 |
| 步骤⑤同步 | `materials[]`, 最终锁定 | 完整明细表，与Excel一致 |

### 3.4 资助自动计算

```javascript
function calcSponsorNeeded() {
  var income = tuition;                    // 学费收入
  var expense = salary_rmb_total          // 人工费
              + lodging_rmb_total         // 食宿费RMB
              + rmb_pool;                 // 赏金池RMB
  return Math.max(0, expense - income);   // 缺口 = 需要申请的资助
}
```

资助**不在输入参数中**，始终自动显示。用户看到的是「你需要申请 ¥14,710 的资助来实现盈亏平衡」。

## 四、与 Excel 表的对应关系

最终形态（步骤⑤）的预算表结构与 Excel 表对应：

| Excel 列 | 数据来源 |
|----------|---------|
| 组别 | = `data.bundles[].role` |
| Task/工作包 | = `data.bundles[].task_names` |
| 人力￥ | = `data.budget.staff_salaries[].rmb` |
| 人力nt | = `data.budget.staff_salaries[].nt` |
| 经费￥ | = `data.budget.materials[].rmb` |
| 经费nt | = `data.budget.materials[].nt` |
| 激励nt | = `data.tasks[].points` 汇总 |
| 预算科目 | = 自动分类（人力/任务/物料/赏金池） |
| 已使用 | = 步骤⑤后从财务流水汇总 |
| 余可用 | = 预算 - 已使用 |

## 五、渐进感的关键设计

1. **步骤①的预算表有「空洞」**：员工NT工资显示 `-- 待认领`，任务显示 `(粗估)`
2. **空洞随步骤消失**：②填上任务NT → ③填上人员名字 → ④填上个人NT → ⑤全部补完
3. **总预算数字在变**：每一步完成后底部汇总自动刷新，用户看到数字从粗略到精确
4. **行数在增长**：①只有5行 → ②增加任务子行 → ③增加每人一行 → ⑤增加物料行
5. **视觉提示**：未完成的区域用虚线边框 + 灰色文字标记「待后续步骤填入」
