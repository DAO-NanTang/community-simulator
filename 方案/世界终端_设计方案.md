# 贪婪之岛·世界终端 — 设计方案

> 基于《Hunter × Hunter》贪婪之岛的分析，将 dashboard 从「项目管理工具」转变为「世界创造终端」。
> 这不是换一套图标和标签，而是一次心智模型的迁移。

---

## 零、先理清：这个仪表盘到底是干什么的

在讨论"怎么改得像贪婪之岛"之前，先明确当前工具的两个面孔：

| | 面孔 A：建造者视角 | 面孔 B：冒险者视角 |
|------|-----------------|-----------------|
| **谁看** | 砚仁 + 工作人员（筹备期） | 共创人 + 工作人员（营期中） |
| **干什么** | 设计世界、配置规则、分配资源 | 查看任务、认领委托、追踪进度 |
| **对应标签** | 🌍 营地设置、👥 策划组、💰 资金池 | 📋 任务大厅、🗺️ 攻略地图、🧾 结算 |
| **贪婪之岛类比** | 金 + 11 个朋友在建造游戏 | 小杰 + 奇犽在岛上冒险 |
| **使用时机** | 开营前（5 步流程） | 开营后（日常使用） |

**关键洞察**：当前仪表盘这两个面孔混在一起了。建造者用的设置页和冒险者用的展示页放在同一个标签栏里，没有任何视觉区分。要把它们分开——

- **世界终端**（建造者视角）= 营地设置的 5 步 + 全境监控 + 资源金库
- **冒险界面**（玩家视角）= 任务大厅 + 攻略地图 + 结算

## 一、对象与能力假设 → ① 统筹定框架

### 原文

> 金：假設小傑已具備基礎念與獵人資格，下一階段要的是「實戰、協作、創造性運用念」。你做生活副本時先問：這個副本主要要服務誰？他們目前的基線能力是什麼？你希望他們在 1–3 週內在哪一組能力上有可見的升級？

### 当前仪表盘的对应

步骤①「统筹定框架」已经在做这件事——设开营日、营期天数、预算、团队名单。但当前把它呈现为一个**管理表单**，用户看到的是"开营日选择器"和"预算输入框"。

### 怎么改

把步骤① 重新命名为「① 创世设定」，分为三个区域：

```
┌─ ① 创世设定 ─────────────────────────────────────────────┐
│                                                            │
│  ┌─ 世界参数 ──────────────────────────────────────────┐   │
│  │                                                     │   │
│  │  🌍 世界启动日 [2026/07/15]  副本持续 [15] 天         │   │
│  │     预计冒险者 [10] 人                              │   │
│  │                                                     │   │
│  │  ┌─────────────────────────────────────────┐        │   │
│  │  │ 这个世界要训练什么？                      │        │   │
│  │  │ [📝 艺术创作基础 + 乡村协作 + NT系统认知]  │        │   │
│  │  │ ← 这段文字自动出现在冒险者的「进入世界」   │        │   │
│  │  │   引导页中，告诉他们来这里能获得什么        │        │   │
│  │  └─────────────────────────────────────────┘        │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                            │
│  ┌─ 世界规则（预算）───────────────────────────────────┐   │
│  │  食宿 ¥30+NT40/人/天  回购 NT300/人                  │   │
│  │  💰 RMB 资源池 ¥20,700    💎 NT 发行池 NT 30,000     │   │
│  │  📌 需申请资助 ¥14,910                               │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                            │
│  ┌─ 世界建造者（团队）─────────────────────────────────┐   │
│  │  🧙 [(砚仁)] [(朝林)] [(小红)] [(若曦)] ...          │   │
│  │  这些人是「世界的共建者」，各自负责一片区域            │   │
│  └─────────────────────────────────────────────────────┘   │
└────────────────────────────────────────────────────────────┘
```

**关键改动**：加了一个 `这个世界要训练什么？` 的文本框。这个文本会流到冒险者看到的界面上——就像贪婪之岛的入口石碑。它告诉每个进入这个世界的人：你来这里能得到什么。

## 二、世界观与边界 → ① + ② 的任务池

### 原文

> 貪婪之島給了一個完整但有限的世界：有地圖、城鎮、卡片系統、NPC、明確出入口與通關條件。生活副本也需要清楚的邊界：在什麼時間範圍內運作？在什麼空間內發生？有哪些資源是副本內資源？

### 当前仪表盘的对应

步骤① 设定了时间边界（开营日→结营日），步骤② 列出了这个世界里所有可能发生的事（任务池）。日历系统定义了每天的世界状态。

### 怎么改

**② 任务池 → 改名为「任务之书」**。每条任务不再叫"任务"，而叫"委托"或"卡片"（直接借用贪婪之岛的卡片系统）：

```
当前：任务名 / 类别 / 激励点 / 类型 / 来源
改为：委托名 / 类型(主线/支线/隐藏) / NT奖励 / 发布者 / 出处

当前「类别」的下拉选项：宣传、课程、生活、财务、管理
改为：每个类别对应一种卡牌颜色：

  宣传 → 🟡 传令卡（金色边框）
  课程 → 🟢 修行卡（绿色边框）  
  生活 → 🟤 生活卡（棕色边框）
  财务 → 🟣 财富卡（紫色边框）
  管理 → 🔵 秩序卡（蓝色边框）
```

**日历系统 → 改名为「世界时间线」**。甘特图不只是"日程表"，而是这个世界的"昼夜循环"——每天太阳升起时世界进入新状态，夜晚降临时有晚间事件。

**空间边界**：当前没有。建议在攻略地图标签页加一个"世界地图"——不是真的地图，而是用文字 + emoji 画出营地的空间结构：

```
┌─ 🌍 南塘·第四期 世界地图 ──────────────────────────────┐
│                                                           │
│   🏠 素社（住宿·食堂）         🌾 农田（农务体验）         │
│        │                          │                        │
│        └──────── 🎨 画室 ────────┘                        │
│                     │                                      │
│              🖼️ 展览厅                                     │
│                     │                                      │
│              🏘️ 南塘村（入户闲谈）                          │
│                                                           │
│   这个地图在「攻略地图」标签中始终可见                       │
└───────────────────────────────────────────────────────────┘
```

## 三、规则系统 → ②③④ 的任务设计链

### 原文

> 金用卡片、念、限制條件構出一套「要強就得懂規則、利用規則」的系統。玩家在操作規則時自然學會策略、資源管理與風險判斷。對生活副本而言，可以用任務卡、行動點數、角色能力值等，把你希望玩家做的行動「遊戲化」成可操作的物件。

### 当前仪表盘的对应

这是仪表盘最核心的部分——NT 任务系统。步骤②③④ 就是在设计这个规则系统：
- ② 列出所有可能的"卡片"（任务）
- ③ 把卡片打包，指定谁来发布（NPC 角色）
- ④ 填写每张卡片的详细规则（NT 奖励、验收标准、时限）

### 怎么改

**核心改动：任务 → 卡片的视觉化**

当前每条任务在表格里是一行文本。改为每张"卡片"是一个可视化的物件：

```
当前（任务大厅的卡片）：
┌────────────────────────────┐
│ ⬜ 召集筹备会              │
│ 👤 砚仁 · 📅 6/15 · ⭐ 3点 │
└────────────────────────────┘

改为（冒险者看到的委托卡）：
┌────────────────────────────┐
│ 🟡 传令卡 · M1             │
│ ─────────────────────────  │
│ 📖 入营注册                │
│ 完成数字身份注册+NT账户激活  │
│                            │
│ 🎁 NT 30                  │
│ 📅 第 1 天                 │
│ 🧙 发布者：砚仁            │
│                            │
│ [⚔️ 接受委托]              │
└────────────────────────────┘
```

卡片的视觉元素：
- **边框颜色** = 类别（传令/修行/生活/财富/秩序）
- **左上角徽标** = 主线 M / 支线 S / 共创 C / 隐藏 H
- **右下角 NT 值** = 奖励数字用金色大字
- **已完成的卡片** = 加盖一个半透明的「✅ 已完成」印章覆盖

## 四、难度曲线与心流 → 日历系统 + 任务时间分布

### 原文

> 貪婪之島的難度不是線性，而是有起伏，穿插探索、戰鬥、解謎、學習。Day 1–2：熟悉規則，中段：難度拉高、需要合作與創造性解法，尾聲：收斂與通關儀式。

### 当前仪表盘的对应

日历系统定义了每天的时间段。任务按 milestone 分配日期。但当前**没有人检查难度曲线是否合理**——可能第一天就堆了 20 个任务，中间几天空的。

### 怎么改

在日历视图中增加**难度热力图**——每个日期格子下方加一个小进度条，显示该天的任务密度：

```
│ 7/20 D1  │ 7/21 D2  │ 7/22 D3  │ ...
│ 开营日   │          │          │
│ ░░░░     │ ████     │ ██████   │  ← 任务密度
│ 3项·简单 │ 8项·中等 │ 12项·密集│
│ 🟢       │ 🟡       │ 🔴       │  ← 自动标注
```

超过 N 项/天 → 自动提示「这天任务过于密集，建议分散到前后日期」。

**心流曲线可视化**：在日历顶部加一条曲线示意：

```
难度 ↑
     │      ╭─╮
     │     ╱  ╲
     │    ╱    ╲___╱╲
     │   ╱            ╲___
     │  ╱                  ╲
     └──────────────────────────→ 天数
       D1  D5  D10  D15
       
🟢 破冰期  🟡 修炼期  🔴 冲刺期  🟣 收尾期
```

这不是精确计算，而是给建造者一个直觉判断——「我这期的难度安排合理吗？」

## 五、通关奖励与离开副本 → ⑤ 同步 + 结算

### 原文

> 小傑通關後得到的是「去見金的線索」與念上的突破，而不是單純的物質獎勵。金在世界樹上跟小傑談的其中一層意思是：遊戲只是階段性的，真正的世界在外面。

### 当前仪表盘的对应

⑤ 同步展示 + 结算界面。当前的结算界面只有 NT 数值和任务清——这是"物质奖励"。

### 怎么改

结算界面加一个区块：

```
┌─ 🏆 通关记录 ─────────────────────────────────────────────┐
│                                                            │
│  🎨 陈若雨                                                 │
│  ────────────────────────────────────────                 │
│  完成主线 12/12 · 支线 7/9 · 共创 5/7 · 隐藏 2/8           │
│  NT 收入 8,800 · 食宿支出 4,200 · 剩余 4,600               │
│                                                            │
│  📜 获得的称号：                                           │
│  「全勤猎人」— 17天无缺席                                  │
│  「拍卖之星」— 作品拍卖最高价                               │
│                                                            │
│  🗺️ 带出副本的能力：                                       │
│  · 工笔画技法（完成 12 幅作品）                              │
│  · DAO 议事参与（参加 5 次集体会议）                         │
│  · NT 系统操作（学会使用 NT 钱包）                           │
│                                                            │
│  💬 结营感言：                                              │
│  "在南塘的 17 天让我重新理解了艺术与社区的关系..."            │
└────────────────────────────────────────────────────────────┘
```

这不只是结算 NT，而是**给每个冒险者一份"通关证书"**——他们从这个副本带走了什么，回到真实世界后能做什么。

## 六、总体的界面架构改动

### 当前架构

```
所有 9 个标签平铺在一行
📝营地设置 | 📊任务总览 | 📋任务大厅 | 🗺️攻略地图 | 👥策划组 | ⚔️玩家 | 📅日历 | 💰资金池 | 🧾结算
```

### 改为

**两层面板 + 一个世界状态条**：

```
┌─ 🌍 南塘·第四期 ─── 世界状态 ─────────────────────────────┐
│  ⏳ 距启动 15天  👥 冒险者 0/10  💰 ¥20,700  💎 NT30,000  │
│  世界阶段：🟢 筹备中（第③步）                              │
└──────────────────────────────────────────────────────────┘

┌ 建造者面板 ────────────────────────────────────────────────┐
│ [①创世设定] [②任务之书] [③打包命名] [④角色填写] [⑤世界启动]│
│ 这个面板只在「筹备期 / 建造者」登录时可见                    │
└────────────────────────────────────────────────────────────┘

┌ 冒险者面板 ────────────────────────────────────────────────┐
│ [📋猎人公会] [🗺️冒险地图] [👥冒险者档案] [📅世界时间线]      │
│ [💰资源金库] [🧾通关结算]                                    │
│ 这个面板所有人可见，但「猎人公会」和「冒险地图」按角色分流    │
└────────────────────────────────────────────────────────────┘
```

两个面板有**不同的视觉色调**：
- 建造者面板 = 深色底 + 金色文字（像控制台/终端）
- 冒险者面板 = 暖色底 + 羊皮纸纹理（像冒险世界）

## 七、改动分层：MVP → 完整版

### 第一层：文案 + 配色（1-2 小时）

| 改动 | 效果 |
|------|------|
| 标签改名（9 个标签 + 子标签） | 立即改变认知框架 |
| 状态条 | 加一行 6 个指标的 HUD |
| 按钮文案 | "新增任务"→"发布委托"、"提交到任务大厅"→"发布到猎人公会" |
| 卡片边框颜色 | 5 种类型对应 5 种颜色 |

### 第二层：卡片视觉化（2-3 小时）

| 改动 | 效果 |
|------|------|
| 任务卡片改为委托卡样式 | 冒险者看到的每条任务像一张游戏卡片 |
| NT 值改为金色大字 | 奖励更显眼 |
| 已完成卡片加盖"印章" | 视觉满足感 |

### 第三层：世界地图 + 难度曲线（3-4 小时）

| 改动 | 效果 |
|------|------|
| 攻略地图加空间地图 | 营地空间可视化 |
| 日历加任务密度 + 心流曲线 | 难度平衡一目了然 |
| 结算加通关证书 | 离开副本的仪式感 |

### 第四层：建造者/冒险者面板分屏（4-6 小时）

| 改动 | 效果 |
|------|------|
| 标签拆分为两个面板 | 建造者和冒险者看到不同的入口 |
| 建造者面板深色终端风格 | 视觉上区分"创造世界"和"进入世界" |
| 登陆时按身份自动切面板 | 工作人员看到建造者面板，共创人看到冒险者面板 |

---

## 八、不改的内容

这些保持原样，已经符合"世界建造"的隐喻：

| 保留 | 原因 |
|------|------|
| 攻略地图的三栏布局（NPC / 任务 / 玩家） | 已经是游戏界面风格 |
| 头像系统（DiceBear 卡通头像） | 符合 NPC / 冒险者角色 |
| 数据模型全部字段 | 逻辑不变，只改变呈现方式 |
| 5 步流程的顺序 | 对应"创世"的自然顺序，不乱动 |
| localStorage 存储 | 轻量、离线可用，符合"个人终端"概念 |
