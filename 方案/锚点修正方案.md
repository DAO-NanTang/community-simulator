# 锚点与里程碑修正方案

## 一、「锚点」的正确含义

**当前误解**：锚点 = 锁定不可修改
**正确含义**：锚点 = 相对位置标记

```
开营日 → 永远在 D1（不管营期几天）
合照日 → 永远在营期中点（15天→D8, 17天→D9）
展览日 → 永远在结营-1天
结营日 → 永远在最后一天
```

锚点的作用是：**模板加载时，把对应内容自动放到正确的相对位置**。加载完成后，用户可以修改任何格子的内容（包括锚点格）。🔒 图标只是标记"此内容来自模板锚点"，不是"锁定"。

## 二、里程碑与日历的 1:1 对应

```
日历                   里程碑
──────                ──────
D1   7/20    ←→    开营第一天 (offset 0)
D8   7/27    ←→    合照日 (offset mid)
D13  8/1     ←→    展览筹备3~1
D14  8/2     ←→    展览日 (offset ex = end-1)
D15  8/3     ←→    结营日 (offset end)
```

**修复**：`generateMilestones()` 必须从日历数据读取起止日期，而不是从 `wizStartDate` 输入框。因为日历可能被用户增减天数。

```javascript
// 当前（错）
var startD = new Date(data.camp_dates.start)  // 读输入框

// 修复（正确）
var schedDates = data.schedule.dates  // 读日历实际数据
var startD = parseMD(schedDates[0])
var endD = parseMD(schedDates[schedDates.length-1])
```
