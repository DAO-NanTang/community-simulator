# 实景地图 · 建筑/田地位置精确化方案

> 版本：v1.0 | 日期：2026-07-06
>
> 核心问题：大地图的方位地址是正确的，但如何让地图上的房屋布置、房屋位置、菜地位置做到准确无误？

---

## 一、问题分析

当前情况：
- **大地图参考源**：方位地址正确（知道合作社在十字路口北侧、路东侧等宏观位置）
- **内部布局**：合作社大院、西厢瓦房、宿舍民宿、戏台、后院菜地在院内的**相对位置**目前是手工估画的，不够精确

目标：
- 每个建筑的**位置**（在网格上的行列坐标）对应真实空间关系
- 每个建筑的**大小**（占几格）对应真实尺寸
- 建筑之间的**间距**和**对齐**符合实际

---

## 二、总体思路：基准点 + 比例尺 + 坐标系

```
真实世界（米）  ──比例尺──→  网格坐标（行, 列）
                                 ↓
                          Unicode 字符画地图
```

核心公式：

```
网格行 = 基准行 + (真实Y坐标 / 每格代表米数)
网格列 = 基准列 + (真实X坐标 / 每格代表米数)
建筑占格数 = 建筑实际尺寸 / 每格代表米数
```

### 2.1 确定比例尺

**推荐：1 字符 = 1 米**（大地图鸟瞰图）

理由：
- 合作社大院约 20m × 10m → 占 20 列 × 10 行，在 50 列宽的地图上比例合适
- 整个院落约 60m × 40m → 占 60 列 × 40 行，网格 60×50 刚好容纳
- 整数比例尺，计算简单，不易出错
- 当细节不足以表达时（比如 0.5m 的门），可以用门字符（🚪）占 1 格近似

### 2.2 选定基准点（原点）

推荐：**院落西南角（围墙西南角）** 作为原点 (row=0, col=0)

理由：
- 西南角是实体参考点，实地容易找到
- 坐北朝南的布局，南侧是正面，从西南角开始测量自然
- 网格坐标从(0,0)开始，row 向北增加，col 向东增加

坐标系约定：
```
       col → (东)
row   0    10   20   30   40   50
↓  0  ┌─────────────────────────
(北)  │  围墙·南（大门）
    10│         🟫广场
    20│   戏台      宿舍
    30│        合作社大院    西厢瓦房
    40│        后院菜地
      │              围墙·北
```

## 三、获取真实位置数据的方法（三种，按推荐度排序）

### 🥇 方法 A：卫星图描摹法（最推荐，零成本，精度够用）

**步骤：**

1. **截取卫星图**
   - 打开 Google 地图 / 高德地图 / 百度地图的卫星视图
   - 定位到合作社位置
   - 缩放到能看到整个院落的级别（通常 1:200 ~ 1:500）
   - 截图保存为 PNG

2. **在卫星图上标注参考线**
   - 在截图上画出围墙的四条边
   - 标注南北方向（卫星图默认上北下南）
   - 标出已知参照物（十字路口、道路边线）

3. **叠加网格**
   - 在图片上叠加一个半透明网格
   - 网格间距 = 卫星图上 1 米对应的像素数
   - 工具：可用 Photoshop/GIMP，或者直接用 map-editor 的图片背景功能

   ```
   ┌──┬──┬──┬──┬──┬──┬──┬──┬──┐
   │  │  │  │  │  │  │  │  │  │
   ├──┼──┼──┼──┼──┼──┼──┼──┼──┤
   │  │  │  │🏠│🏠│🏠│  │  │  │  ← 每格 = 1m
   ├──┼──┼──┼──┼──┼──┼──┼──┼──┤
   │  │  │  │🏠│🏠│🏠│  │  │  │
   └──┴──┴──┴──┴──┴──┴──┴──┴──┘
   ```

4. **读取每个建筑的网格坐标**
   - 在网格上找到每个建筑的四个角
   - 记录下来：

   ```
   合作社大院：左上角(row=22, col=20)，占 20 行 × 10 列
                         ↑ row 从西南角(原点)向北计算
   西厢瓦房：  左上角(row=22, col=8)，占 15 行 × 8 列
   戏台：      左上角(row=10, col=12)，占 8 行 × 6 列
   宿舍民宿：  左上角(row=10, col=25)，占 8 行 × 12 列
   后院菜地：  左上角(row=35, col=20)，占 8 行 × 20 列
   ```

5. **校验**
   - 检查建筑之间的间距是否合理
   - 检查道路宽度是否一致
   - 与实地照片比对

**实施方式**：直接在 `map-editor/editor.html` 中添加「加载背景图」功能，将卫星截图作为半透明底层，在上面放置 stamp。

---

### 🥈 方法 B：实地测量法（精度最高，需要去现场）

**工具**：卷尺（30m/50m）、激光测距仪、纸笔、手机拍照

**步骤：**

1. **选基准点**
   - 在院落西南角地面做标记（粉笔/贴纸）

2. **测围墙**
   - 沿南围墙向西→东测量：记录南围墙总长
   - 沿西围墙向南→北测量：记录西围墙总长
   - 得到院落总尺寸（如 52m × 38m）

3. **测每个建筑**
   从基准点出发，对每个建筑测量：
   - 建筑西南角距南围墙的距离（Y 偏移）
   - 建筑西南角距西围墙的距离（X 偏移）
   - 建筑的长度（东西方向）
   - 建筑的宽度（南北方向）

   记录表格：
   ```
   | 建筑       | 距南墙(m) | 距西墙(m) | 长度(m) | 宽度(m) |
   |------------|----------|----------|--------|--------|
   | 合作社大院  |    8     |    18    |   20   |   10   |
   | 西厢瓦房    |    8     |    2     |   15   |   8    |
   | 戏台       |    24    |    10    |   8    |   6    |
   | 宿舍民宿    |    24    |    24    |   8    |   12   |
   | 后院菜地    |    0     |    18    |   8    |   20   |
   ```

4. **换算为网格坐标**
   - 比例尺 1 格 = 1 米
   - row = 距南墙距离（米）
   - col = 距西墙距离（米）
   - 占行数 = 宽度（米），占列数 = 长度（米）

**优点**：精度最高，0.1m 级别；**缺点**：需要去现场。

---

### 🥉 方法 C：手机步行测距法（备选，适合无法获取清晰卫星图的情况）

**工具**：手机（GPS + 计步/测距 App，如"测距仪"、"GPS Fields Area Measure"）

**步骤：**

1. 站在院落西南角，打开测距 App，标记起点
2. 沿围墙边走，每到一个建筑角点就标记
3. App 自动记录轨迹和距离
4. 导出坐标数据，换算为网格坐标

**精度**：GPS 民用精度约 3-5 米，对于建筑级别的定位可能不够精确。建议结合计步器或手动输入距离来校正。

## 四、推荐的混合方案（最佳实践）

```
卫星图描摹（定大致位置）
      ↓
   实地测量（校准关键尺寸）
      ↓
   导入 map-editor（放置 stamp）
      ↓
   实地拍照验证（抽查核对）
```

具体流程：

1. **先用卫星图**（方法 A）获取所有建筑的相对位置和大致尺寸，放入网格
2. **再去一次现场**（方法 B），只测量 3-5 个关键距离：
   - 院落总长和总宽
   - 合作社大院的长和宽（最大的建筑，作为比例尺校准锚点）
   - 主路宽度
3. **用关键测量值校准**：按实测尺寸调整网格上的建筑大小，其他建筑按卫星图的相对比例同步缩放
4. **最终验证**：在地图上随机选 3 个点，实地验证距离是否吻合

## 五、在 map-editor 中的实现

### 5.1 加载背景图功能

在 `map-editor/editor.html` 中新增：

```javascript
// 背景图层：加载卫星截图作为半透明底图
const backgroundLayer = {
  image: null,  // Image 对象
  opacity: 0.5,
  offsetX: 0, offsetY: 0,  // 图片在网格上的偏移
  scale: 1,                 // 图片缩放（使图片上的1米 = 网格1格）
  visible: true,

  load(url) {
    this.image = new Image();
    this.image.src = url;
  },

  render(ctx, grid) {
    if (!this.image || !this.visible) return;
    ctx.globalAlpha = this.opacity;
    ctx.drawImage(
      this.image,
      this.offsetX, this.offsetY,
      this.image.width * this.scale,
      this.image.height * this.scale
    );
    ctx.globalAlpha = 1;
  },

  // 校准：用户在图片上标记两个点，输入这两点之间的实际距离（米）
  // 系统自动计算 scale，使图片上的距离 = 网格上的格数
  calibrate(px1, py1, px2, py2, realMeters) {
    const pixelDist = Math.hypot(px2 - px1, py2 - py1);
    this.scale = realMeters / pixelDist; // 每像素 = 多少格
  }
};
```

### 5.2 工作流程

```
① 加载卫星截图 → 背景半透明显示
② 拖拽/缩放图片，使其与网格对齐
③ 校准：在图片上选两个已知距离的点（如围墙两端），输入实测距离
④ 图片自动缩放至 1:1 比例（图片上的 1 米 = 网格上的 1 格）
⑤ 在透明底层上放置 stamp → 建筑位置 = 图片上建筑的网格坐标
⑥ 隐藏背景图，导出纯文字画地图
```

## 六、精度等级

| 等级 | 方法 | 精度 | 适用 |
|------|------|------|------|
| **A 级** | 实地激光测量 + 全站仪 | ±0.05m | 建筑施工图 |
| **B 级** | 实地卷尺测量 | ±0.2m | **推荐**，足够地图使用 |
| **C 级** | 卫星图 + 已知尺寸校准 | ±0.5m | 快速出图，精度够用 |
| **D 级** | 卫星图目测估画 | ±2m | 仅供示意，不推荐 |

**对于南塘合作社的场景，B 级或 C 级完全够用。** 地图的目的是让访客和成员了解空间关系、找到位置，不需要建筑级别的精度。

## 七、菜地位置的特殊处理

菜地与建筑不同，它没有明确的墙壁边界。建议：

1. **用卫星图的田垄/颜色变化**来判定菜地边界
2. **实地插 4 根竹竿**标记菜地四角，然后测量竹竿位置
3. 菜地内部的分区（A/B/C/D 区）用等分法：
   - 测量整块菜地的长和宽
   - 按 2×2 等分 → 得到 A/B/C/D 四区
   - 每区大小相同，边界清晰
