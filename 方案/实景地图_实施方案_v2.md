## 〇、我们为什么要做这个

### 背景

南塘合作社是一个线下实体空间，包含多栋建筑（合作社大院、宿舍、戏台、西厢瓦房等）、田地（后院菜地）以及周边的道路、邻居房屋等。目前合作社的管理依赖人工记忆和口头沟通——房间里有什麼、用到什么程度、田地种了什么、何时成熟——信息分散在不同人的脑子里。

### 核心目标

做一个**线上空间管理工具**，让线下空间状态可视化：

```
线下世界                        线上地图
────────                        ────────
宿舍 101 打扫完成      →        🟢 101 标记为「已打扫」
菜地 A 区种了番茄       →        🍅 显示「生长中 · 剩 40 天」
玉米成熟了             →        🔔 提醒「可收获」
```

### 使用场景

- **访客**：打开地图了解合作社布局，找到房间位置
- **共建者/管理员**：在地图上直接编辑布局、管理物品、追踪作物
- **后续**：嵌入 dashboard 仪表盘，与任务系统、成员系统联动

### 设计原则

1. **逐层下钻**：从宏观方位 → 建筑布局 → 房间细节，像地图 App 一样逐层放大
2. **所见即所得**：地图本身就是编辑器，不需要"先画好再导出再展示"
3. **零依赖**：纯 HTML+CSS+JS，一个文件即可运行，不引入任何框架
4. **风格统一**：与 dashboard.html 共用一套暖色系设计语言，预留合并接口

## 一、系统架构

### 1.1 三层导航结构

```
方位图 (宏观区域关系)
  │  格子 15×20，emoji 图标标注位置
  │  点图标 → 跳转到对应建筑的布局图
  │
  ├→ 宿舍布局图
  │    格子 22×30，显示两合院建筑 + 四周道路 + 邻居
  │    点房间格子 → 下方弹出物品表
  │
  ├→ 合作社布局图
  │    格子 22×28，显示三层平房 + 门前路 + 后院菜地
  │    子标签 [1F] [2F] [3F] 切换楼层
  │      点房间格子 → 下方弹出物品表
  │
  └→ 菜地管理
       格子 11×15，四区田间追踪
       点田块 → 下方弹出作物信息
```

### 1.2 楼层切换时的数据合并

合作社有 3 层楼，每层内部布局不同，但建筑外形和周边环境不变。

```
数据分为两层：
┌──────────────────────────────┐
│  surroundings（周边层）       │  ← 所有楼层共用
│  道路、围墙、菜地、建筑外框    │
├──────────────────────────────┤
│  floors['1F'] （1F平面）     │  ← 只有1F显示
│  客厅/厨房/储藏室 隔墙 + 标签  │
│  floors['2F'] （2F平面）     │  ← 只有2F显示
│  空的，待管理员编辑            │
│  floors['3F'] （3F平面）     │  ← 只有3F显示
│  空的，待管理员编辑            │
└──────────────────────────────┘
```

渲染时，每个格子先取周边层数据，如果格子落在建筑边界内且当前楼层有覆盖数据，则用楼层数据覆盖。这样切换到 [2F] 时，道路和菜地原样不动，只换建筑内部的房间布局。

### 1.3 两种模式

| | 🔒 查看模式 | ✏️ 编辑模式 |
|------|-----------|-----------|
| 工具栏 | 隐藏 | 显示 |
| 点击格子 | 显示信息（物品/作物） | 编辑格子（填色/画边/写字） |
| 谁用 | 所有人 | 共建者/管理员 |
| 切换按钮 | 右上角 🔒查看 → ✏️编辑 | 右上角 ✏️编辑 → 🔒查看 |

## 二、网格与图层设计

### 2.1 正方形格子，密度统一

所有视图的格子都是**正方形**，尺寸统一。不因文字内容撑大格子。

```
cell-width = cell-height = 24px（全局固定）
方位图：15×20 格
宿舍：  22×30 格
合作社：22×28 格
菜地：  11×15 格
```

### 2.2 三层视觉分离

每个格子的渲染由三个独立图层叠加，互不干扰：

```
第1层 · 网格线层（border / box-shadow）
  — 墙壁线 1.5px，暖棕色
  -- 围墙线 1px 虚线，灰色
  线条粗细固定，不被文字撑变形

第2层 · 填色层（background-color）
  🟫 道路色    🟩 田地色
  🟦 水域色    🟨 建筑色
  格子背景色，不影响格子尺寸

第3层 · 文字图标层（cell content）
  🏠 建筑标签  🥬 菜地标签  📦 房间名
  font-size 固定，溢出隐藏
```

## 四、编辑工具

编辑模式下，选工具后在网格上操作：

| 工具 | 快捷键 | 操作方式 | 效果 |
|------|--------|---------|------|
| 🖱️ 选择 | V | 点击格子 | 选中，方向键移动 |
| 🟫 填路 | R | 拖拽涂格子 | 背景变道路色 |
| 🟩 填田 | D | 拖拽涂格子 | 背景变田地色 |
| 🟦 填水 | A | 拖拽涂格子 | 背景变水域色 |
| 🟨 填房 | H | 拖拽涂格子 | 背景变建筑色 |
| 🧱 画墙 | W | 点格子边缘 / 拖拽 | 格子边=墙壁线 |
| 🚧 围墙 | F | 点格子边缘 / 拖拽 | 格子边=围墙线 |
| 📐 方框 | B | 对角拖拽 | 画出矩形墙壁框 |
| 📝 文字 | T | 点击弹出输入 | 写入 emoji / 文字 |
| 🧹 擦除 | E | 拖拽 | 清除填色+边框+文字 |

编辑楼层（[1F]/[2F]/[3F]）时，绘制自动限制在建筑边界内，周边区域（道路、菜地）不可修改。

## 五、技术实现

| 要点 | 方案 |
|------|------|
| 渲染引擎 | HTML `<table>`，每个 `<td>` = 一个格子 |
| 填色 | CSS `background-color`，不挤占格子尺寸 |
| 边框 | CSS `box-shadow: inset`，1.5px 细线 |
| 文字 | `<td>` 内容 + `overflow: hidden` |
| 编辑交互 | mousedown/move/up + Bresenham 线算法 |
| 性能 | 最大 30×30 = 900 格，table 渲染 < 50ms |
| 持久化 | localStorage（编辑后自动保存） |
| 依赖 | 零依赖，纯 HTML+CSS+JS |

## 六、预留合并接口

实景地图作为独立文件运行，但结构上为后续合并到 dashboard 做准备：

1. **CSS 变量对齐**：两套系统使用相同的变量命名（`--bg`、`--accent`、`--text` 等）
2. **容器隔离**：地图主体内容包裹在独立容器中，CSS 作用域不泄漏
3. **全局 API**：对外暴露 `initRealscapeMap(container)` 等方法
4. **数据互通**：地图数据对象 (`SITE_DATA`) 可被 dashboard 的 `data` 对象引用
5. **合并时只需**：在 dashboard 加一个 tab 按钮 + 引入地图容器 + 调用初始化方法
