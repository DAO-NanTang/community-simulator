# 南塘世界 · 完整愿景

> 最终形态：一个双面终端。共建者看到的是「创世面板」，冒险者看到的是「副本游戏」。同一套数据，两种完全不同的界面体验。

---

## 一、入口：登录与注册

所有人打开同一个网址，首先看到的是**世界入口**——不是管理后台，是一个世界的门前。

### 1.1 新冒险者注册

```
│                                                          │
└──────────────────────────────────────────────────────────┘
```

**新冒险者注册流程**：

```
[⚔️ 注册进入] → ① 选择头像 → ② 输入名字 → ③ 设置密码 → ④ 进入世界欢迎页
```

**新建造者注册流程**：

```
[🧙 建造者入口] → ① 选择头像 → ② 输入名字 → ③ 设置密码 
                 → ④ 输入建造者邀请码（由已有建造者提供）
                 → ⑤ 进入世界终端
```

建造者需要邀请码——不是谁都能当建造者。但注册流程本身是开放的，只是权限由邀请码控制。

### 1.3 身份与权限

| | 共建者 | 冒险者 |
|------|------|------|
| 注册方式 | 头像+名字+密码+邀请码 | 头像+名字+密码 |
| 看到什么 | 世界终端（全部数据） | 副本游戏（个人数据 + 排行榜） |
| 能否切到冒险者视角 | ✅ 可以（顶部有切换按钮） | ❌ 不能 |
| 能否开冒险者小号 | ✅ 可以（再注册一个冒险者账号） | — |
| 能否邀请新共建者 | ✅ 可以（生成邀请码） | ❌ |

## 二、共建者界面：世界终端

### 2.1 模式切换（手动，不自动）

世界终端顶部有一个模式切换栏。共建者手动选择当前工作的模式：

```
┌─ 🧙 世界终端 ── 砚仁 ── [⚔️ 切换到冒险者视角] ─────────────┐
│                                                            │
│  模式：[创世] [监察] [预测] [整理] [结算]                    │
│         ████                                           │
│         当前                                            │
│                                                            │
│  创世 = 开营前的配置模式（5步流程）                          │
│  监察 = 开营中的监控模式（实时数据 + 协调）                   │
│  预测 = 结营前的预测模式（完成率预测 + 资源预测）              │
│  整理 = 结营后的数据整理模式                                 │
│  结算 = 最终成就结算页                                       │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

- 5 个模式手动点击切换，不绑定日期
- 营期中需要回创世模式调任务 → 点一下即可
- 结营后想回顾监察数据 → 点监察即可
- 当前模式高亮，其余灰显

```
│  ║ 任务 47项 │ NPC 6人 │ 日程 14节点 │ 模板 10个       ║ │
│  ╚══════════════════════════════════════════════════════╝ │
│                                                          │
│  ┌─ 创世步骤 ──────────────────────────────────────────┐ │
│  │ ①创世设定 ✅ → ②任务之书 ✅ → ③打包命名 ⚠ → ④角色 ⬜ → ⑤启动 ⬜│

│                                                          │
```

**关键交互**：左侧是操作区，右侧是监控面板。共建者在左侧创世的同时，右侧实时反映全局状态。遗漏项、资源消耗、进度——不开新页面、不切标签，一眼看到。

## 四、两套界面的整体架构

### 不是两个网页，是两个"图层"

```
                    登录页（入口）
                         │
              点击头像 → 判断身份
                         │
          ┌──────────────┴──────────────┐
          ▼                              ▼
    ┌──────────┐                  ┌──────────┐
    │ 世界终端  │                  │ 副本游戏  │
    │ (共建者)  │                  │ (冒险者)  │
    └──────────┘                  └──────────┘
          │                              │
          │    共享同一份 data 对象        │
          │    ┌──────────────────┐      │
          └───→│ tasks, members,  │←─────┘
               │ budget, schedule,│
               │ finance, bundles │
               └──────────────────┘
```

- 同一个 HTML 文件
- 同一份 `data` 对象
- 切换只是 `switchView('builder')` vs `switchView('adventurer')`
- 无页面跳转、无加载时间
- 共建者可以切换到冒险者视角（预览效果）
- 冒险者**不能**切换到共建者视角

### 两个界面的信息可见性
