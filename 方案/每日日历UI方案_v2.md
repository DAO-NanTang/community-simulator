## 日程模板下拉内容

```
日程模板：[📅 默认日程模板(15天) ▾]
          ├── 📅 默认日程模板(15天)   ← 8时段，来自PRESET
          ├── 📅 第三期日程模板       ← 用户存的
          └── 📥 从当前日历系统导入…  ← 取 data.schedule.slots
```

选「从当前日历系统导入」→ 加载日历系统标签中的时段结构作为模板。

## 数据隔离

```
data.wizSchedule  ← 向导专用（步骤①-④编辑）
  └ 步骤⑤同步 ──→ data.schedule ← 日历系统标签专用
```

步骤①-④中修改日历不影响日历系统标签。步骤⑤点同步后一次性推送。

## 按钮行为

| 操作 | 行为 |
|------|------|
| 改日期 | 只更新 `data.camp_dates.start`，不触发日历重绘 |
| 点「生成日历」 | 加载模板时段 + 创建空表。已有数据时弹确认 |
| 点「重新生成」 | 同上，但按钮文案变了（已有数据时） |
| 点格子 | 弹出编辑框改内容 |
| 拖拽格子 | 互换两个格子的内容 |
| [+天] [−天] | 增减天数，联动 `data.camp_dates.duration_days` |
| [+行] | 新增时段行 |
| [📋 复制前一天] | 末列 = 倒数第二列 |
| [💾 存为模板] | 当前时段结构（不含活动）存为模板 |
| [🔮 从日历生成里程碑] | 基于 `data.wizSchedule` 的起止日期计算18个节点 |
