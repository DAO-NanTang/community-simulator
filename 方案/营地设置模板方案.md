# 营地设置 · 模板系统修改方案

> 基于 2026-07-03 对话，整合四份参考文档与现有 dashboard 代码的修改方案。

---

## 一、问题分析

### 1.1 当前状态（事实）

| 现状 | 具体描述 |
|------|---------|
| 营地设置标签存在 | 位于标签栏第一位，有 5 个子页（任务池/团队/预算/日程/AI导入） |
| 任务池是单一表格 | 工作人员任务和共创人任务混在同一张表里 |
| 无模板机制 | 用户必须逐条手动录入，或从已有任务导入（仅限已提交到任务大厅的） |
| 预算有默认值 | 预设了第三期数据，但无重置按钮 |
| 模板无持久化 | 关闭页面后用户自建的模板丢失 |

### 1.2 参考文档提供的内容（事实）

| 文档 | 可提取的模板内容 |
|------|-----------------|
| 流程手册 v5 | 5 阶段（筹备→招募→开营前→营期→结项），10 议程域，约 30 项工作人员任务 |
| 任务系统清单 | 4 类共创人任务（主线 12 项/支线 9 项/共创 7 项/隐藏 8 项），含 NT 值和验收标准 |
| 第四期课程草案 | 课程结构：茶话会→公共理论课(3节)→小组共同画→自学小组→两人会议 |
| 游戏化设计理念 | 底层隐喻（勇者/NPC/新手村），不直接产生模板 |
| data.json budget | 第三期实际预算数据 |

### 1.3 需求汇总

| # | 需求 | 来源 |
|---|------|------|
| 1 | 任务池拆分为「工作人员」和「共创人」两个独立子标签 | 用户明确指令 |
| 2 | 提供预设模板，基于前三期经验 | 用户提供四份参考文档 |
| 3 | 模板可一键导入到当前任务表 | 用户：模板条目可以增加减少 |
| 4 | 用户可保存当前任务表为新模板 | 用户：可以增加模板、减少模板 |
| 5 | 模板存储在 localStorage | 用户确认 |
| 6 | 预算页增加「重置为默认值」按钮 | 用户：防止改乱了回不去 |

## 二、修改方案

### 2.1 任务池拆分

**改动**：营地设置子页从 5 个变为 6 个。

```
① 工作人员任务池 → ② 共创人任务池 → ③ 团队 → ④ 预算 → ⑤ 日程框架 → ⑥ AI导入
```

每个任务池是独立的表格（共用同一套表格编辑逻辑），提交时写入 `data.tasks` 的不同区域（通过 `type` 字段或 `assignee` 区分）。

### 2.2 模板系统

**复用现有代码**：模板就是一组 `tasks` 数组，格式完全等同于 `data.tasks` 的条目格式。表格编辑逻辑（`renderSettingTasks`、`addSettingTask`、`removeSettingTask`）不变。

**新增**：
- 一个模板选择器（`<select>` 下拉框），位于「＋ 添加行」按钮左侧
- 「📥 使用模板」按钮：选中模板 → 覆盖当前表格
- 「💾 保存为模板」按钮：当前表格内容 → 弹出输入框命名 → 存入 localStorage

**模板存储结构**（localStorage key: `camp_templates`）：
```json
{
  "工作人员-筹备会完整包": {
    "type": "staff",
    "tasks": [ {...}, {...} ]
  },
  "共创人-主线任务包": {
    "type": "creator", 
    "tasks": [ {...}, {...} ]
  }
}
```

### 2.3 预设模板

基于四份参考文档提取，硬编码为默认模板（首次使用时自动写入 localStorage）。

| 模板名 | 类型 | 条目数 | 数据来源 |
|--------|------|--------|---------|
| 工作人员·筹备会完整包 | 工作人员 | ~30 | 流程手册 A2 议程域①-⑩ + A3 产出 |
| 工作人员·招募与招生 | 工作人员 | ~8 | 流程手册 B1-B3 |
| 工作人员·开营前准备 | 工作人员 | ~10 | 流程手册 C1-C3 |
| 工作人员·营期执行 | 工作人员 | ~12 | 流程手册 D-时间线 + D-动议 |
| 工作人员·结项与复盘 | 工作人员 | ~8 | 流程手册 E1-E3 |
| 共创人·主线任务包 | 共创人 | 12 | 任务系统清单 二 |
| 共创人·支线任务包 | 共创人 | 9 | 任务系统清单 三 |
| 共创人·共创任务包 | 共创人 | 7 | 任务系统清单 四 |
| 共创人·隐藏任务包 | 共创人 | 8 | 任务系统清单 五 |
| 课程·第四期课程框架 | 工作人员 | 6 | 课程设计草案 二 |

### 2.4 预算重置按钮

在预算子页的「💾 保存预算」按钮旁增加「🔄 重置默认值」按钮。点击后恢复为第三期数据。

默认值常量：
```
学费 5990 | 资助 25000 | 人工费 14750 | 物料 2000 | RMB赏金池 3000
NT总发行 30000 | NT赏金池 4500
```

### 2.5 不改动的内容

| 区域 | 不改的理由 |
|------|-----------|
| ③ 团队子页 | 已有增删功能，复用 `data.members`，无需模板 |
| ④ 日程框架子页 | 已有默认 8 时段模板和日期生成，够用 |
| ⑥ AI导入 | 逻辑独立，本次不改 |
| 任务大厅、攻略地图等其他标签 | 与模板系统无直接关系 |

## 三、实现步骤

1. 修改营地设置子标签按钮：①→⑥，文案调整
2. 新增「② 共创人任务池」HTML（复用①的表格结构）
3. 在两个任务池表格上方各加模板选择器 + 使用/保存按钮
4. 编写模板管理函数（load/save/delete/apply）
5. 硬编码 10 个预设模板的默认数据
6. 预算页加重置按钮
7. 修改 `switchSettingsTab` 支持 6 个子页

## 四、审查结果

### 4.1 少即是多 ✅
只加了模板选择器和两个按钮（使用模板/保存为模板），表格编辑逻辑完全复用。子页从 5 变 6，新增的共创人任务池 HTML 复用工作人员任务池的表格结构。

### 4.2 复用优先 ✅
- 模板数据结构 = `data.tasks` 条目结构，零转换成本
- 表格渲染函数 `renderSettingTasks` 不变
- 增删行函数 `addSettingTask`/`removeSettingTask` 不变
- 提交函数 `commitSettingTasks` 不变

### 4.3 需求对齐 ✅
每处改动都有明确来源（见 1.3 需求汇总表）。无"顺手优化"。

### 4.4 全局最优 ✅
- 模板存储在 localStorage，不依赖后端
- 导出 data.json 时模板可一并导出
- 模板系统与任务大厅共享同一数据结构
- 两个独立任务池（工作人员/共创人）对应两套不同的任务体系

### 4.5 长期主义 ✅
- 预设模板硬编码，不依赖外部文件
- 用户自建模板持久化在 localStorage，刷新不丢失
- 模板增删接口独立（loadTemplates/saveTemplate/deleteTemplate），后续可扩展为导出/导入 JSON 文件
- 新营期只需修改模板内容，不需要改代码
