# 营地设置 · 修正流程 v2

> 修正原因：上一版把卡片当作预制的输入，实际卡片是流程的最终产出。  
> 正确的流程是：定框架→列任务→打包→命名角色→填卡片→同步。

## 五、关键设计决策

| 决策 | 理由 |
|------|------|
| 6步默认顺序引导，但不强制 | 默认按顺序走，每步完成自动解锁下一步。但允许直接点击任意步骤跳转。未完成的步骤按钮标灰+⚠提示，完成的标绿✅。老用户可以跳过已熟悉的步骤，新用户有引导。第二次筹备会可能只需要修改几张卡片——直接跳⑤即可 |
| 每步可以"暂存"后退出 | 开会时间有限，不需要一次完成全部6步 |
| ②任务池可以从模板导入 | 流程手册提供预设任务池，避免从零开始 |
| ③合并可拆分 | 合并错了可以拆开，不丢失数据 |
| ⑤按角色筛选 | 宣传人员不需要看到统筹的卡片，聚焦自己的工作 |
| NT任务的链接字段 | 链接指向NT平台的具体任务页，展示时需要 |
| 食宿NT覆盖检查 | 统计共创人任务总NT是否≥食宿所需NT，不够时标红警告 |

## 六、两条时间线的关系

系统中存在两条时间线，它们展示的是同一件事，但职责不同：

| | 营地设置中的时间线 | 日历系统中的时间线 |
|------|-----------------|------------------|
| **定位** | 初始默认值 + 存储地 | 展示 + 日常修改 |
| **数据来源** | `data.camp_dates`（由开营日自动计算） | `data.schedule`（由营地设置同步过来） |
| **编辑权限** | 设置初始框架（13个里程碑节点的默认日期） | 日常增删改（营期中实际发生的日程调整） |
| **修改粒度** | 节点级别（改一个节点的日期） | 日级别（每天的每个时段的具体活动） |
| **版本历史** | 无 | 有——每次修改留痕，可以从历史记录恢复 |
| **同步方向** | → 初始同步到日历 | ← 修改后可回写到设置（如果用户选择"以日历为准"） |

### 6.1 使用场景

```
营地设置                        日历系统
────────                        ────────
①选定开营日                     ③自动显示生成的日期
②自动算出13个节点日期
[＝同步] ───────────→           ④营期中：展览日期调整了
                               ⑤在日历里拖动展览日到新日期
                               ⑥系统提示："设置中的日期与日历不一致
                                  [以日历为准] [以设置为准]"
                               ⑦点"以日历为准" → 设置的节点日期更新
                               ⑧如果改错了 → 查看历史记录 → 恢复
```

### 6.2 冲突处理

当两边的日期不一致时（比如设置里展览日是7/28、日历里改成了7/29）：

- 日历系统上方显示提示条：「📅 展览日与设置不一致。设置值：7/28 → 当前值：7/29  [以日历为准] [以设置为准]」
- 用户选择后，两边统一。历史记录保留修改痕迹
- 如果用户不改，两边各自保留自己的值，下次打开时再次提示

### 6.3 实现方式

- 同步时，营地设置的 `data.camp_dates` → 写入 `data.schedule.dates` 和 `data.schedule.weekdays`
- 日历中修改日期时 → 对比 `data.camp_dates` → 不一致则提示
- 日历的修改历史存储在 `data.schedule_history`（数组，每次修改 push 一条，保留最近50条）

### 7.2 三种视图的分工

| 视图 | 位置 | 用途 | 什么时候用 |
|------|------|------|-----------|
| **时间线卡片** | 上方 | 一眼看到全流程的时间跨度，直观感受筹备→营期→结项的节奏 | 初次设定、向团队展示整体时间规划 |
| **节点详情** | 中间 | 点击某个节点后展开，查看和编辑单个节点的详细信息 | 需要修改某个特定节点时 |
| **表格一览** | 下方 | 所有节点在一张表里，批量编辑日期、查看计算规则、增删节点 | 精细调整、检查遗漏、对比各节点日期 |

### 7.3 交互说明

| 操作 | 效果 |
|------|------|
| 修改开营日 | 所有依赖开营日的节点日期自动重算。🔒锁定的节点不重算 |
| 修改营期天数 | 结营日及之后的所有节点自动重算 |
| 点击时间线卡片上的 ✎ | 弹出日期选择器，手动改这个节点的日期 |
| 点击时间线卡片 | 下方展开节点详情面板 |
| "锁定此日期" 勾选 | 该节点不受开营日变化影响。适合"展览日已经定了场地，不能改"的场景 |
| [+添加节点] | 创建自定义节点（如"第二次展览"、"期中评估"） |
| 表格中的 🔒 | 开营日节点不可删除，但可以改日期 |
| 表格中的 🗑 | 删除自定义节点。系统预置的13个节点不可删除（灰色） |
| 表格中直接点 📅 | 行内弹出日期选择器，快速编辑日期 |
| 手动改过日期的节点 | 旁边显示 ✎ 标记，一眼看出哪些是自动算的、哪些是手动调的 |

### 7.4 计算规则与锁定

每个节点有两种状态：

| 状态 | 行为 | 适合场景 |
|------|------|---------|
| **自动计算**（默认） | 日期由公式决定（如"开营-30天"），改开营日时自动重算 | 筹备会、招募启动等内部节点 |
| **🔒 锁定** | 日期固定，不受开营日变化影响 | 展览场地已定、外部合作方已确认的日期 |

**切换方式**：点击节点详情中的"锁定此日期"复选框。锁定后，表格中该节点的计算规则列显示"已锁定"。

### 7.5 与日历系统的关系

营地设置的日历（本节）与展示页面的日历系统（📅 日历系统标签）的分工见第六章。本节日历 = 初始默认值 + 存储地；日历系统 = 展示 + 日常修改。两边通过 [＝ 同步] 按钮保持数据一致。

### 8.6 六步导航栏

所有6个步骤共用一个顶部导航：

```
┌──────────────────────────────────────────────────────────────┐
│  ① 统筹  ② 任务池  ③ 打包  ④ 角色  ⑤ 填卡片  ⑥ 同步      │

└──────────────────────────────────────────────────────────────┘
```

- 点击任意步骤名 → 直接跳转（不强制顺序）
- ✅ 已完成 / ⚠ 进行中 / ⬜ 未开始
- "暂存"保留当前步骤的填写进度，退出后回来继续
- 每个步骤底部统一三个按钮：`[← 上一步] [暂存] [保存并进入下一步 →]`
