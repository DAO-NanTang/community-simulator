## 核心变化

### 1. 包名 → 人名

| 旧 | 新 |
|----|-----|
| 包名 = "宣传工作包" | 认领人 = "小红" |
| 先取名再选人 | 选中任务→直接选人认领 |

### 2. 认领 = 创建工作人员卡片

勾选任务 → 下拉选人（已有）或输入新名字 → 确认认领 →
- 如果名字已存在：任务追加到该人名下
- 如果新名字：自动创建一张工作人员卡片（姓名必填，角色=所选类型，寄语选填）

**工作人员卡片字段**：

| 字段 | 必填 | 说明 |
|------|------|------|
| 姓名 | ✅ | 认领人名字 |
| 角色/类型 | 可修改 | 默认取任务类别，可改（统筹/宣传/NT/课程/…），可新增类型 |
| 寄语 | 选填 | 一句话，如"把筹备会开好" |

### 3. 类型可修改

角色/类型不再锁死为预设列表。认领时：
- 下拉框显示已有类型
- 底部有 `＋ 新增类型` 选项
- 新增后自动加入选项列表

### 4. 未认领兜底

所有没被勾选认领的任务，集中在底部「未认领任务」区，每条旁边有一个下拉框：
- 下拉列出所有已创建的工作人员
- 选人 → 确认 → 任务归属该人

这样确保没有遗漏的任务。

### 5. 右侧面板

右侧不再是「已合并的包列表」，而是「工作人员卡片列表」：
- 每张卡片显示：头像、姓名、角色、寄语
- 可修改（弹出编辑框）、可移除
- 底部 `＋ 新建工作人员卡片` 按钮（不绑定任务，只创建空白卡片）

```javascript
// data.staff_cards — 工作人员卡片
data.staff_cards = [
  { name:'砚仁', role:'统筹', note:'把筹备会开好', task_names:['召集筹备会','预算任务表'] },
  { name:'小红', role:'宣传', note:'多拍好照片', task_names:['设计招募文案','每日推文'] },

// 未认领任务：task_names 不在任何 staff_cards 里的任务
// 通过遍历 wizardTasks 并检查是否在 staff_cards 的 task_names 中来确定
```

## 交互流程

```
1. 左侧勾选任务（可全选）
2. 下拉选择认领人（已有工作人员）或输入新名字
3. 点击「确认认领」
   → 任务归入该人名下
   → 右侧卡片列表更新
4. 底部「未认领任务」自动刷新
5. 全部认领完毕 → 汇总确认 → 进入步骤④
```
