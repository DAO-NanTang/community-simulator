## 二、三个表的定位

### 2.1 📊 预算材料表（上）

**数据来源**：该人物名下所有任务的 `materials[]` 字段汇总。

```
物品      预算RMB  预算NT  来源任务
─────────────────────────────────────
笔记本×5    ¥50     —     召集筹备会
打印费      ¥30     —     动员会
颜料×3      —     NT60   准备画材清单
─────────────────────────────────────
合计        ¥80    NT60
```

- 同一物品名称自动合并数量（笔记本 3+2=5）
- 「来源任务」列可追溯每笔预算的出处
- 预算数字 = 步骤④角色填卡片时填入的预估值

### 2.2 💳 实际使用表（下）

**数据来源**：公会资金池流水表中该人物经手的记录。

```
物品      实花RMB  实花NT  日期    事由
────────────────────────────────────────
笔记本×3    ¥35     —     7/15   采购
打印费      ¥30     —     7/16   文印店
────────────────────────────────────────
合计        ¥65     —
```

- 从 `data.finance_cny.transactions` 中筛选 `person === 该人物` 的记录
- 按物品名称分组汇总
- 日期显示最近一笔

### 2.3 ⚖ 对照（下）

**数据来源**：预算表 vs 实际表的逐项对比。

```
物品      预算    实际    差额    状态
──────────────────────────────────────
笔记本×5  ¥50    ¥35    ¥15    ✅ 未超
打印费    ¥30    ¥30     ¥0    ✅ 持平
颜料×3   NT60     —    NT60    ⚠ 未花
──────────────────────────────────────
合计     ¥80+60  ¥65    剩余    ✅
```

- 按物品逐项对比
- 实际 > 预算 → 🔴 超支
- 实际 ≤ 预算 → 🟢 正常
- 预算有但实际没花 → ⚠ 未使用（提醒是否漏记）

## 三、左侧面板追加的统计项

选中人物后，左侧面板加 3 行：

```
📋 总任务 5     ← 已有
📦 材料项 3     ← 新增：该人物的材料条目数
💰 预算 ¥80    ← 新增：该人物材料预算合计
💳 实花 ¥65    ← 新增：该人物实际花费合计
```

## 四、切换机制

切换按钮放在左侧面板的进度条下方：

```
┌ 左 ────────────┐
│ [头像]          │
│ ♂️ 砚仁 ✎       │
│ 核心成员        │
│                 │
│ 📋 总任务  5    │
│ 📦 材料项  3    │
│ 💰 预算 ¥200   │
│ 💳 实花 ¥155   │
│                 │
│ ████░░ 78%     │ ← 进度条
│                 │
│ [📋 任务]       │ ← 切换按钮在这里
│ [📦 材料表]     │
└────────────────┘
```

- 两个按钮纵向排列，当前选中项高亮
- 默认选中「📋 任务」→ 右面板显示当前的任务列表
- 点「📦 材料表」→ 右面板切换为三段式材料视图
- 切换状态按人物独立记忆（看砚仁时切到材料表、切换到朝林时回到任务）

## 五、预算预填

步骤④角色填卡片时，每项任务的 `materials[]` 已经有预设模板（第 3698-3719 行，如"国画教学 → 毛笔×10 套 ¥200"）。这些预设值就是预算的**初始值**。

用户可以在步骤④调整数量/价格。调整后的值 → 写入 `data.tasks[].materials[]` → 汇总到个人材料预算表。

### 预填规则

| 任务关键词 | 预设材料 | 来源 |
|-----------|---------|------|
| 国画/白描/临摹 | 毛笔、宣纸、墨汁、颜料 | 第 3698 行 |
| 写生 | 写生本、炭笔 | 第 3700 行 |
| 装裱 | 装裱材料 | 第 3702 行 |
| 宣传/小红书 | 推广流量费 | 第 3705 行 |
| 安全检查 | 保险费 | 第 3709 行 |
| 住宿 | 住宿费 | 第 3712 行 |
| 展览 | 展览材料 | 第 3719 行 |

## 六、实施要点

### 6.1 改动范围（dashboard.html）

| 区域 | 改动 |
|------|------|
| 游戏策划组 `#memberDetail` | 右面板加切换按钮 + 材料视图渲染 |
| 游戏玩家 `#playerDetail` | 同上（共创人也可以看自己的材料） |
| `renderMemberDetailFor()` | 新增 `viewMode` 参数 + 材料表渲染分支 |
| 左侧面板 | 加 3 行统计（材料项/预算/实花）|

### 6.2 新增 JS 函数

```
getPersonMaterials(name)       → { budget:[], actual:[], comparison:[] }
  ├── 从 data.tasks 筛选 assignee===name → 汇总 materials[]
  ├── 从 data.finance_cny.transactions 筛选 person===name → 汇总
  └── 逐项对比 → 生成 comparison 数组

renderMaterialSheet(name)      → HTML
  ├── 渲染预算材料表
  ├── 渲染实际使用表
  └── 渲染对照表 + 超支/正常状态
```

### 6.3 数据流

```
步骤④角色填卡片
  │
  │  填写 materials[] → 写入 data.tasks
  │
  ▼
游戏策划组标签
  │
  │  选中人物 → getPersonMaterials(name)
  │
  ├── 预算表 ← data.tasks[].materials[]（按 assignee 筛选）
  ├── 实际表 ← data.finance_cny.transactions[]（按 person 筛选）
  └── 对照   ← 逐项 diff
```

## 七、超支提醒

对照表中：

- `实际 > 预算` → 行背景变红，末尾显示 `🔴 超支 ¥XX`
- `实际 ≤ 预算` → 行背景正常，末尾 `✅`
- `预算有、实际无` → 行背景灰黄，末尾 `⚠ 未记录`

底部汇总行：

```
🟢 全部未超预算    ←  所有项目 actual ≤ budget
🟡 2项未记录支出   ←  有 budget 但无 actual
🔴 1项超支 ¥35    ←  actual > budget
```
