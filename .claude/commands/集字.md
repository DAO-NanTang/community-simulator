从《大唐双龙传》语料库中检索最匹配的原著段落，供Claude手工套用——最小化改动（人名+代词），保留原著文笔质感。

## 三级匹配策略

### 第一级：精确匹配（首选）
工具返回的Top段落中有可套用的句子 → 直接套用。仅改人名、代词、地点。

**如果候选段落都不可用** → 换查询词重跑一次。查询越精准，命中率越高。

### 第二级：跨场景套用（备选）
找不到同场景的段落时，找**相同叙事功能**的段落：
- 找不到「少女从天坠落」→ 找「有人坠水/被击飞/落马」
- 保留动作结构和句式节奏，换核心名词

### 第三级：句式模板（兜底）
连跨场景都找不到时，用 `--templates` 输出句式模板。将新内容填入模板的 `[ ]` 中，保留其余部分不变。

模板文件：`集字工具/templates.json`（按叙事功能分类）

## 套用步骤

1. **写查询**：从timeline节点提取核心动作，30-50字
2. **跑工具**：`retrieve.py -s "查询" -n 5 --templates`
3. **选句子**：从候选段落中选出可套用的句子，丢弃不相关的
4. **套用改写**：只改三个东西——人名、代词、地点。不改句式、词汇、节奏
5. **标注来源**：每句后面注明候选段落编号
6. **自检**：对照检查清单

## 写作规则（必须遵守）

### 禁止
- **禁止使用破折号「——」**。黄易原文不使用破折号。用逗号、句号、分号替代。
- **禁止加入自己的描写**。不要添加候选句中没有的形容词、比喻、心理活动。
- **禁止改写句式**。候选句的句子长度、逗号密度、断句方式必须保留。

### 套用法则
- **最小改动**: 人名替换 + 地点/代词微调。其他一字不动。
- **段落节奏**: 黄易的段落结构是「同类型成块」——3-4句连续动作后接3-4句连续对话。不要一句动作一句对话地交替。
- **每句标注来源**: 输出时标注每句来自哪个候选段落，证明没有自己编。

### 检查清单
- [ ] 有没有「——」？→ 删除
- [ ] 有没有候选句中没出现的词？→ 替换
- [ ] 有没有一句动作一句对话交替？→ 合并成块
- [ ] 每句都能指出候选段落来源？→ 标注

## 技术信息

- 语料：黄易《大唐双龙传》，380万字，108,797句
- 检索：BGE-M3语义检索（1024维），预计算向量缓存，秒级响应
- 预计算缓存：`corpus_vectors.npy`（首次需运行 `build_vectors.py` 生成，约30分钟）
- 行号估算：`corpus_index + 14`，可用 `index_novel.py` 验证原文

## 注意事项

- 不要修改工具代码
- `build_vectors.py` 是离线任务，只运行一次
- 如果查询包含原著名（如"寇仲""徐子陵"），检索会自动匹配这些角色出场的句子

## 章节结构类型（8种积木）

所有章节都从以下8种类型中选取。同人章节可以自由组合，同一类型可以连用多章。

### A · 铺设型
**用途**: 引入新角色/新势力/新世界观
**节奏**: 慢→中，多线并行
**结构**:
```
环境开场（黄易式景物）→ 反派/第三方视角 → 主角视角切换 → 各方线索汇聚 → 悬念收尾
```
**心理密度**: 低。主角尚未进入核心行动
**系统**: 激活/首次展示

### B · 紧张推进型
**用途**: 危机升级、被迫逃亡、悬念断章
**节奏**: 中→急，单线推进
**结构**:
```
触发危机 → 行动块（逃/躲/追）→ 短暂喘息 → 危机再次升级 → 悬崖式收尾
```
**心理密度**: 中。即时反应为主（黄易式半句嵌动作）
**系统**: 静默或积分跳动

### C · 动作高潮型
**用途**: 多组动作连续、正面对抗、重大转折
**节奏**: 快→极快→骤停
**结构**:
```
突发事件 → 动作块1（遭遇）→ 动作块2（对抗）→ 动作块3（脱逃）→ 受伤/代价 → 收束句
```
**心理密度**: 中低。动作优先，心理极短
**系统**: 静默

### D · 过渡喘息型
**用途**: 两场大事件之间的缓冲、情感积累
**节奏**: 中→慢
**结构**:
```
上章余波 → 小型危机（非致命）→ 共同应对 → 情感互动（心理块展开）→ 新地点落脚
```
**心理密度**: 高。神豪式独立段，处境评估+对他人的观察
**系统**: 积分结算/轻微互动

### E · 计谋转折型
**用途**: 主角主动行动、计谋布局、人际关系变化
**节奏**: 中→急→缓→新方向
**结构**:
```
角色设局/伪装 → 计谋推进 → 意外危机 → 外力介入 → 救援/脱困 → 新盟友/新方向
```
**心理密度**: 高。决策分析+风险评估（神豪式选项分析）
**系统**: 可能触发关键提示

### F · 休息信息型
**用途**: 安全环境下的世界观扩展、角色关系深化、技能学习
**节奏**: 慢→中→末尾加速（危机预兆）
**结构**:
```
安全环境 → 背景铺陈 → 人物互动（对话块为主）→ 技能/信息传递 → 亲密时刻 → 危机预兆
```
**心理密度**: 中高。对他人的观察+对处境的反思
**系统**: 积分结算/新能力兑换

### G · 决战死亡型
**用途**: 全书情感核心、重要角色死亡、巨大转折
**节奏**: 急→骤停→极慢
**结构**:
```
正面冲突 → 逃亡/追踪 → 决斗 → 死亡 → 临终对话 → 葬别 → 悲痛（心理块展开）→ 决定/誓言
```
**心理密度**: 极高。丧恸+自责+决心（神豪式展开+黄易式收束）
**系统**: 可能与情感峰值共振

### H · 修炼成长型
**用途**: 时间跨度、内在转变、技能升级
**节奏**: 极慢，时间压缩
**结构**:
```
时间标记 → 重复性动作（修炼/劳作）→ 微小突破 → 时间跳跃 → 新状态 → 决定离开
```
**心理密度**: 中。阶段性反思（神豪式独立段）
**系统**: 积分积累期

### I · 征途遭遇型
**用途**: 新地图、新人物、新冲突的引入
**节奏**: 中→急
**结构**:
```
新旅程 → 环境描写（黄易式）→ 遇新人 → 新人困境 → 出手介入 → 新冲突爆发 → 悬念
```
**心理密度**: 中。对新人评估（类似神豪选项分析）
**系统**: 积分事件触发

## 同人卷结构设计原则

1. **8种积木自由组合**，不限于原著顺序
2. **同一类型可连用多章**（如连用两个F型来展开情感）
3. **整体叙事弧**（铺垫→危机→高潮→收束）保持不变
4. **同人比原著章节更多**是正常的——多出来的是情感空间和心理描写
5. **每章开头标注类型**，方便后续润色时套用对应的结构模板
