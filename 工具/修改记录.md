## 一、日程锁头修复

### 用户要求

每日安排表格中，末尾三列（布展、展览、结营）不应该显示 🔒 锁头。锁头给人一种"位置被锁定到固定索引"的误解，但实际上它们只是自然处于末尾——因为它们是最后的列，不需要锁来固定。同时，插入/删除列时这三列应始终保持在末尾。

### 计划修改（4 处）

**1. `getAnchorSet()` 精简**（第 4686 行）

- **当前代码**：锁定 7 列 `[1, photoDay, exhDay-3, exhDay-2, exhDay-1, exhDay, lastDay]`
- **改为**：仅保留 `[1]`（D1 开营日）
- **效果**：末尾列不再显示 🔒，视觉上不再暗示"被锁定"

**2. 新增常量 `RESERVED_TAIL_COLS`**

- **位置**：`getAnchorSet()` 上方，与日程相关常量放在一起
- **值**：`var RESERVED_TAIL_COLS = 3;`（布展、展览、结营共 3 列）
- **用途**：`addScheduleDay`、`removeScheduleDay`、`copyPrevDaySchedule` 三处共用这一个常量，避免各处散落魔法数字 `-3`

**3. `addScheduleDay()` 限制插入位置**（第 4841 行）

- **新增**：
  ```js
  var maxInsert = Math.max(0, data.schedule.dates.length - RESERVED_TAIL_COLS);
  if (insertAt > maxInsert) insertAt = maxInsert;
  ```
- **效果**：「＋天」新增的列永远插在末尾 3 列之前

**4. `removeScheduleDay()` 禁止删除末尾三列**（第 4857 行）

- **新增**：
  ```js
  var minReserved = Math.max(0, data.schedule.dates.length - RESERVED_TAIL_COLS);
  if (removeAt >= minReserved) { alert('最后 ' + RESERVED_TAIL_COLS + ' 天（布展/展览/结营）不可删除'); return; }
  ```
- **效果**：「−天」按钮无法删除末尾保留列

**5. `copyPrevDaySchedule()` 限制插入位置**（第 4870 行）

- **新增**：
  ```js
  var maxInsert = Math.max(0, data.schedule.dates.length - RESERVED_TAIL_COLS);
  if (insertAt > maxInsert) insertAt = maxInsert;
  ```
- **效果**：「📋 复制前一天」新列永远插在末尾保留列之前

## 二、预算框架改造

### 用户要求

1. 收入与支出分成两个独立卡片
2. 学费 = 早鸟价 × 比例 + 原价 × 剩余比例，自动计算
3. 住宿/吃饭有单价参数（单价 × 天数 × 共创人数，自动计算）
4. 预设固定支出项：娱乐资金、破冰活动经费、展览费用、日常经费、画材费用、开营晚餐、迎新晚会、结营聚会、保险费、备用金、推广费
5. 预设收入项：基金申请、社区募捐、DAO 申请
6. 每条目只需填 **RMB 金额** 和 **NT 金额** 两个数字
7. 不做薪资计算，不设分类下拉（薪/雇/费/激），不设计算方式下拉
8. 可自定义添加/删除条目
9. 导出 Excel 匹配模板 `D:\桌面\模板.xlsx` 的结构
10. 自动计算项（住宿费、餐费、画材费用、保险费）显示计算结果而非输入框

### 计划修改

**1. 引入 exceljs CDN**（`<head>` 末尾，第 337 行 Phosphor Icons 之后）

```html
<script src="https://cdn.jsdelivr.net/npm/exceljs@4.4.0/dist/exceljs.min.js"></script>
```

用于在浏览器端生成带合并单元格和样式的 `.xlsx` 文件。

**2. HTML 布局**（约第 585–635 行，替换原「💰 预算框架」卡片）

原卡片内容全部替换为三个区块：

- **📥 收入参数卡片**：
  - 基础参数行：共创人数 + 工作人员数 + 天数（只读）+ 早鸟价/比例/原价
  - 收入条目列表区（`#wizIncomeItems`）
  - 底部：「＋ 添加收入项」按钮 + 收入合计

- **📤 支出参数卡片**：
  - 单价参数行：住宿 ¥/NT /人/天 + 吃饭 ¥/NT /人/天
  - 支出条目列表区（`#wizExpenseItems`）
  - 底部：「＋ 添加支出项」按钮 + 支出合计

### 四、方案本身的风险点与建议

#### 4.1 Step 7 "隐藏旧元素"不可行

**问题**：方案建议"保留旧 HTML 元素不变，用 CSS `display:none` 隐藏"。但：
- `renderStep1`（4429-4441）每次调用都会 `.value = ...` 写入这些元素
- `saveStep1`（5072-5084）每次保存都会读这些元素的 `.value` 写进 `data.budget`
- 隐藏后新旧两套逻辑并行运行：新 UI 展示 items 计算，旧逻辑在隐藏元素上读写真值，数据不一致

**建议**：直接删除旧元素，同时修改三处 JS（见 3.2 表格）。不要"隐藏"。

#### 4.2 新 `updateWizBudgetCalc` 与旧调用点兼容

**问题**：旧 `updateWizBudgetCalc` 被多处调用（HTML oninput × 10+、renderStep1 × 2、saveStep1、syncStaffCount）。新函数需要：
- 不再读取已删除的旧元素
- 新 HTML 中的 oninput 也需要绑定到新函数

**建议**：新函数同名替换，旧 oninput 随 HTML 替换自然消失。renderStep1 和 saveStep1 中的调用点保留不变。

#### 4.3 budget_items 需要纳入 localStorage 持久化

**问题**：`saveStep1` 保存 `data` 到 localStorage，但如果 `budget_items` 只在 `updateWizBudgetCalc` 中被 `localStorage.setItem`，则保存时机不够明确。

**建议**：`updateWizBudgetCalc` 末尾已有 `localStorage.setItem`（新方案中），每次计算完自动保存。`saveStep1` 不再需要单独处理 budget_items。

#### 4.4 Excel 导出不能再引用 category

**问题**：`exportBudgetExcel` 中的 `fillAmtCells` 之前依赖 `item.category` 字段，精简后已删除该字段。

**建议**：所有金额统一填入经费列 K-L（方案中已写明）。
