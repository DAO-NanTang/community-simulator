# 监察面板 · 重构方案

> 目标：像微信红点一样做实时提醒——折叠时看数字，展开后看详情，看过就消红点
> 2026-07-07

---

## 〇、监察是什么

**实时提醒系统，不是仪表盘。** 管理员不需要在这里操作——操作仍在猎人公会、NT管理、资源金库里。

监察的工作方式像微信消息列表：

> 不展开 → 只冒红点数字，"有 3 件事需要关注"
> 展开 → 看到具体是谁、什么事、多久了
> 看过 → 红点消失，下次只冒新的

### 核心机制：折叠 + 红点 + 已读/未读

| 机制 | 说明 |
|------|------|
| 折叠 | 每层默认折叠（只显示摘要行），点击「展开」看完整内容 |
| 红点（●） | 标记"上次看过之后新出现"的条目 |
| 模式按钮徽章 | HUD 栏「📊 监察 ③」显示总未读数，不用打开监察就能感知 |
| 已读 | 展开某一项 → 该项被标记为已读 → 红点消失；下次刷新时不再算新 |
| 持久化 | 已读状态存 `localStorage.oversee_state`，刷新页面不丢 |

### 第一层：🔴 待处理（始终展开）

每条待审核/逾期都显示具体的人名和任务名——不给操作按钮，但**给足信息让管理员判断要不要切到 NT管理 去处理**。

**待审核列表**：每条 = 谁 → 什么任务 · 等了多久
**逾期列表**：按人聚合，每人一行 = 谁：哪些任务分别逾期多久

折叠时显示：`⏳ 待审核 ② +1新`（数字 = 总数，+N新 = 其中新增数）
展开后显示：完整列表，新增项带 ● 前缀

### 第二层：🟡 全员进度（默认折叠）

折叠时一行总结：`⚠️ 2人待审核 · 2人逾期 · 1人完成 · 1人空闲`
展开后显示完整进度条 + 状态标签，排序同前。

### 第三层：🟢 NT 池（始终展开）

一行数字，不含红点（纯数据展示）。

## 二、已读/未读追踪

### 数据结构

`localStorage.oversee_state`：

```json
{
  "seenSubmitted": ["陈若雨|用户访谈", "墨染|编写文档"],
  "seenOverdue": ["小红|系统架构", "fc|代码审查"],
  "collapsed": { "progress": true, "ntpool": false },
  "lastOpened": "2026-07-07T15:30:00"
}
```

### 判断逻辑

| 条目 | "新增"判断 |
|------|-----------|
| 待审核项 | `claimant.name + '|' + task.name` 不在 `seenSubmitted` 中 |
| 逾期项 | `assignee + '|' + task.name` 不在 `seenOverdue` 中 |

### 已读时机

- 展开某个折叠区 → 将该区内所有条目加入 `seenXxx` 数组
- 下次 `showOverseePanel()` 渲染时，这些条目不再显示 ●

### 模式按钮徽章更新

每次渲染监察面板时，计算总未读数（待审核新增 + 逾期新增），更新按钮徽章。退出监察时徽章不消失（保留在按钮上，持续提醒），只有展开对应区域才会消掉该区域的徽章数。

## 三、改动范围

### 修改

| 文件 | 改动 |
|------|------|
| `showOverseePanel()` | 全部重写，加入折叠/展开逻辑、已读追踪、详情展示 |
| `switchBuilderMode()` | 进入/退出监察时刷新按钮徽章 |
| 新增 `getOverseeState()` / `saveOverseeState()` | 读写 `localStorage.oversee_state` |
| 新增 `updateOverseeBadge()` | 更新「📊 监察」按钮上的红点数字 |

### 不改

- 其他 tab 不变
- 数据层不变（`data.tasks`、`data.finance` 等）
- CSS 不变（复用现有 `.card` 样式，新增少量内联样式）

## 四、验收

1. 模式按钮「📊 监察」上有红色数字徽章
2. 打开监察 → 🔴 待处理始终展开，显示具体人名 + 任务名 + 等待天数
3. 新增项带 ● 前缀，已读项不带
4. 🟡 全员进度默认折叠，显示一行摘要；点击展开看进度条
5. 展开某个区域后，红点消失，按钮徽章数减少
6. 刷新页面 → 已读状态保持
7. 无操作按钮
