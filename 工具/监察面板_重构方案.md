# 监察面板 · 重构方案

> 目标：管理员打开监察，2秒看清全局，不被信息轰炸
> 2026-07-07

---

## 〇、监察是什么

**仪表盘，不是操作面板。** 管理员不需要在这里做任何事——操作在猎人公会、NT管理、资源金库里。监察只回答一个问题：

> "现在有什么需要我知道的？"

## 一、三层信息结构

```
┌─ 📊 监察 ────────────────────────────────────────────┐
│                                                      │
│  🔴 待处理（2条，必须马上看到）                        │
│  ⏳ 待审核 3 项 · 最早已等 6 小时                      │
│  ⚠️ 逾期 2 项（小红 · fc）                            │
│                                                      │
│  🟡 全员进度（按优先级排序）                           │
│                                                      
│  陈若雨 ████████░░ 4/6  待审核                       │
│  墨染   ██████░░░░ 3/5  进行中                       │
│  念儿   ████░░░░░░ 2/4  进行中                       │
│  小红   ██░░░░░░░░ 1/6  ⚠️逾期                       │
│  王五   ░░░░░░░░░░ 0/1  空闲                         │
│                                                      
│  🟢 NT池 30,000 · 已发 12,500（12人）· NTA 砚仁      │
└──────────────────────────────────────────────────────┘
```

### 第一层：🔴 待处理

| 显示 | 数据来源 | 说明 |
|------|---------|------|
| 待审核 N 项 | `data.tasks[].claimants` 中 status='submitted' 的数量 | 超过5项或超过24h变红 |
| 最早已等 X 小时 | 最早 submitted_at 到现在的时间差 | 超过24h加「⚠️堆积」 |
| 逾期 N 项（人名） | `data.tasks` 中 deadline < today 且状态未完成 | 列出逾期人的名字 |

**规则**：每条一行，不加任务名、不给操作按钮。

### 第二层：🟡 全员进度

| 显示 | 说明 |
|------|------|
| 名字 | 只显示名字 |
| 进度条 | 完成/总数，百分比色（绿≥80% 黄≥40% 红<40%） |
| 状态标签 | 四种：`待审核` `有逾期` `进行中` `空闲` |

**排序规则**：
1. 有待审核 → 最上面（最近活跃）
2. 有逾期 → 第二
3. 正常进行中 → 第三
4. 空闲（0任务）→ 最后

**不显示**：任务名、截止日期、NT点数

### 第三层：🟢 数字

| 显示 | 数据来源 |
|------|---------|
| NT池总额 | `data.budget.nt_total_pool` |
| 已发放 | `data.finance` 中 type='task_reward' 的金额和 |
| 发放人数 | 去重计数 `to` 字段 |
| NT管理员 | `data.staff_cards` 中 role='NT' 的人名 |

**一行结束。**

## 二、和其他 Tab 的分工

| 信息 | 在哪看 | 监察看不看 |
|------|--------|:--:|
| 任务详情（名称/截止/描述） | 猎人公会 | ❌ |
| 审核操作（确认/退回） | NT管理 | ❌ |
| 成员信息（性别/角色/简介） | 成员档案 | ❌ |
| 财务明细（收入/支出流水） | 资源金库 | ❌ |
| 待审核有几项、堆积了没 | 监察 | ✅ |
| 谁进度正常、谁掉队了 | 监察 | ✅ |
| NT池还剩多少、发了多少人 | 监察 | ✅ |
