# 南塘云村 · 助手化交互方案

> 从「表单工具」变成「社区管家」——不只是提醒你，更是辅助你、建议你、帮你做决定
> 2026-07-07

---

## 〇、核心转变

```
旧思维：打开系统 → 找到功能 → 填表单 → 提交 → 完成
          ↑                        ↑
       你主动找                  你手动填

新思维：系统感知事件 → 主动推送 → 你一句话或点一下 → 系统自动填好 → 完成
          ↑               ↑           ↑                ↑
       系统主动        系统提醒    你只需确认        系统帮你填
```

**不只是提醒，更是辅助决策。** 系统不只是说"白菜快过期了"，而是说"白菜还剩2天，你可以和冰箱里的五花肉一起做个白菜炖肉，番茄还够用"。

| 能力 | 说明 | 技术实现 |
|------|------|---------|

## 二、助手入口

### 2.1 常驻助手条

所有页面底部或侧边固定一个助手入口：

```
┌──────────────────────────────────────────────────────────────┐
│  💬 [    说说你做了什么，或者拍张照...              ]  🎤 📷  │
└──────────────────────────────────────────────────────────────┘
```

- **输入文字**："买了三斤白菜两斤番茄"
- **语音输入**（点🎤）：直接说话
- **拍照**（点📷）：拍小票、拍打扫前后照、拍物品

输入后，助手解析 → 展示将要执行的操作 → 用户确认 → 自动执行。

### 2.2 对话式交互

用户输入一句话后，助手不是直接执行，而是**展示理解结果，让用户确认**：

```
用户: "刚买了白菜三斤番茄两斤放在冰箱了，花了45"

助手:
  ┌─────────────────────────────────────────┐
  │  我理解你要做以下操作：                   │
  │                                         │
  │  ✅ 入库：白菜 × 3斤 → 冰箱·上层冷藏    │
  │  ✅ 入库：番茄 × 2斤 → 冰箱·上层冷藏    │
  │  💰 记账：支出 45 元                    │
  │                                         │
  │  📝 提示：白菜建议2天内吃完。             │
  │         你现在可以做 白菜炖粉条、番茄炒蛋  │
  │                                         │
  │  [✓ 确认] [✗ 取消] [✎ 修改]            │
  └─────────────────────────────────────────┘
```

## 三、六个核心场景

### 场景1：购物入库（小票扫描 + 语音）

**触发方式**：扫小票、或说一句话、或手动输入。

**流程**：

```
① 用户拍照小票 → OCR 识别文本
② 本地模型解析：提取 {物品名, 数量, 单位, 价格}
③ 匹配已知物品库（"白菜"→蔬菜、"五花肉"→肉类）
④ 查冰箱当前空间 → 建议存放位置
⑤ 查保鲜天数 → 标注"建议X天内食用"
⑥ 查库存+菜谱库 → 推荐可组合的菜单
⑦ 生成确认卡片 → 用户一键确认
⑧ 自动写入冰箱库存 + 可选写入虚拟账本（如果涉及金额）
```

**用户侧体验**：拍照 → 看一眼确认 → 点确认。3 秒完成。

### 场景2：饭局编排

**触发方式**：点"攒饭局"、或说"今晚凑个局5个人"。

**流程**：

```
① 用户发起 → 设定人数、时间
② 系统根据人数和历史数据 → 推荐菜单
   例："5个人，建议3菜1汤。根据冰箱库存，可以做：
       番茄炒蛋（番茄有、鸡蛋有、缺葱）
       白菜炖粉条（白菜有、粉条有）
       红烧肉（五花肉有、土豆有）
       紫菜蛋花汤（紫菜有）"
③ 系统自动生成子任务：
   · 采购：买葱一把（缺的食材）
   · 帮厨：2人（洗菜、切菜、炒菜）
   · 洗碗：1人
   · 摆桌：1人
④ 推送给在地成员："今晚饭局缺帮厨1人、洗碗1人"
⑤ 大家点一下认领 → 任务分配完成
⑥ 饭后 → 系统自动记账（帮厨+贡献值、吃饭-NT）
```

**用户侧体验**：设人数 → 看推荐菜单 → 调整 → 确认 → 系统自动拆任务+通知+记账。

### 场景3：田间管理

**触发方式**：系统主动推送（基于生长周期）。

**流程**：

```
① 系统计算每块菜地的生长天数 → 预测成熟日期
② 成熟前1天推送："番茄预计明天可以摘了 🍅"
③ 用户去摘 → 对着助手说"番茄摘完了，大概5斤"
④ 系统：
   · 更新菜地状态（A区→已收割）
   · 查冰箱空间 → "冰箱上层还有位置，建议放上层"
   · 自动入库：番茄 × 5斤
   · 推荐："番茄可以做番茄炒蛋、番茄牛腩、凉拌番茄"
   · 记账：田间劳动 +贡献值
⑤ 如果没及时摘 → "番茄已经熟透了，建议今天内采摘"
⑥ 如果一直不摘 → "番茄过熟了，可以留种或做肥料"
```

**用户侧体验**：收到推送 → 去摘 → 说一句话 → 系统全部处理。

### 场景4：卫生打扫

**触发方式**：系统主动推送（基于脏污度 + 排班）。

**流程**：

```
① 系统计算每个空间的脏污度 → 达到阈值 → 触发打扫提醒
② 推送给本周值日人："厨房该打扫了（脏污度 72/100）🧹"
③ 用户去打扫 → 拍前后两张照
④ 对助手说"厨房打扫完了"
⑤ 系统：
   · 上传前后照
   · 自动提交审核 → 推送给管理员
   · 管理员看到推送 → 点"确认"
   · 自动记账：贡献值+40、经验值+40
   · 脏污度归零
⑥ 如果管理员24h未审核 → 系统自动通过
```

**用户侧体验**：收到提醒 → 打扫 → 拍照 → 说一句话 → 完。

### 场景5：冰箱管理（库存 + 过期预警）

**触发方式**：系统主动推送（基于保鲜天数）或用户手动查看。

**流程**：

```
① 系统每天（页面打开时）检查所有物品保鲜状态
② 提前1天警告：推送"白菜明天过期，建议今天食用 🥬"
③ 推荐用法："冰箱里有白菜+五花肉+粉条 → 可以做白菜炖粉条"
④ 用户吃了 → 对助手说"白菜吃完了"
⑤ 系统：
   · 标记物品已消耗
   · 记账：消耗物品 +贡献值
   · 如果是饭局中消耗 → 关联饭局记账
⑥ 如果过期了 → "白菜已过期，建议清理扔掉"
```

**用户侧体验**：收到提醒 → 决定做什么菜 → 说一句话标记消耗 → 完。

### 场景6：非标劳动申报（仲裁）

**触发方式**：用户主动申报。

**流程**：

```
① 用户对助手说："我整理了三个月水电账单，花了2小时"
② 系统：
   · 解析：操作=非标劳动申报、描述=整理水电账单、耗时=2小时
   · 推荐期望报酬：根据类似劳动历史 "建议申报 15-25 NT"
   · 生成申报卡 → 用户确认期望值 → 提交
③ 系统随机抽取 3 名经验值≥500 的陪审员
④ 推送给他们："小红申报了非标劳动，请评分 [0-30 NT]"
⑤ 三人评分 → 取均值 → 自动记账 → 申报者收到 NT+贡献值+经验值
⑥ 陪审员每人 +2 贡献值（劳务费）
```

## 四、AI 层的技术选择

### 两种路径

| | 本地小模型 | 轻量 API |
|------|----------|---------|
| 是什么 | 浏览器内运行的微型模型（如 Gemma 2B 量化版、ONNX Runtime） | 调用外部 API（如 Claude Haiku、GPT-4o-mini） |
| 优点 | 离线可用、隐私安全、零延迟、免费 | 理解力强、多语言好、不需要下载模型 |
| 缺点 | 需要首次下载模型（~500MB-1GB）、复杂语句可能误判 | 需要网络、有延迟、有调用成本 |
| 适合 | 简单解析（"买了X斤Y"）、物品分类、过期计算 | 菜单推荐、自然语言理解、复杂对话 |

### 推荐方案：本地优先 + API 兜底

```
简单操作（购物、打扫、采摘）→ 本地小模型处理（快、免费、离线）
复杂推荐（菜单搭配、饭局安排）→ API 调用（理解力强）
```

**本地模型技术栈**：
- 文本解析：使用 `transformers.js`（在浏览器里跑 BERT 级别的小模型）做意图识别和实体提取
- 无需 NLP 模型时可回退到**关键词匹配规则**——"买了X斤Y"这种固定句式用正则即可覆盖 80% 场景
- 物品分类、保鲜天数 → 内置知识库（JSON），不依赖 AI

**这意味着**：第一阶段可以不用任何 AI 模型——用规则+模板就能覆盖大部分场景。AI 模型是锦上添花，不是必需品。

## 五、界面改动

### 5.1 新增：助手输入条

所有主界面底部固定显示。输入后弹出对话卡片。

### 5.2 改造：物品入库

旧：表单页 → 逐字段填写。
新：① 拍照小票或语音 → ② 助手展示解析结果 → ③ 一键确认。

### 5.3 改造：任务（委托/大扫除/饭局）

旧：NT 管理面板 → 逐条手动创建任务。
新：点"发起饭局"→ 设人数/时间 → 系统自动拆子任务 → 推送 → 自动记账。

### 5.4 新增：提醒消息流

不是静态列表，是时间线形式的主动推送：
```
今天 14:00  🧹 厨房该打扫了（脏污度 72）
昨天 09:00  🍅 番茄成熟，可以摘了
前天 18:00  🥬 白菜还剩 1 天，建议尽快食用
```

### 5.5 保留：手动录入入口

所有自动流程都保留手动入口。有些人不习惯语音、有些复杂情况助手解析不对——保留"手动添加物品""手动创建任务"作为后备。

## 六、实施路径

| 阶段 | 内容 | 依赖 |
|------|------|------|
| **A: 规则引擎** | 关键词匹配 + 内置知识库（物品分类、保鲜天数、生长周期） | 无 |
| **B: 主动推送** | 过期预警、成熟提醒、脏污度提醒 | A |
| **C: 自动化流程** | 饭局编排、小票扫描（OCR）、一键记账 | A |
| **D: AI 增强** | 本地小模型（意图识别+实体提取）、菜单推荐 API | C |

**A 和 B 可以立刻做，不依赖任何 AI。** 规则引擎 + 推送就能让系统从"表单工具"变成"主动助手"。
