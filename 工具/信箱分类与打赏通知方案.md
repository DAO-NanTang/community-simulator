# 信箱分类 + 打赏通知 方案

> 2026-07-10 | 待讨论，不执行

---

## 〇、现状

信箱有 18 种消息类型，全部混在一个列表里，只分「新消息」和「更早」。消息多了以后，想找「审批结果」得翻半天，打赏更是**根本不进信箱**——收发双方都没有通知。

---

## 一、打赏进信箱

### 1.1 在 `computeInboxMessages` 中新增打赏扫描

信箱消息是 `computeInboxMessages()` **动态计算**的（不是持久存储的消息表）。所有消息类型（`submitted`、`overdue`、`review_approved` 等）都在这个函数里通过遍历 `data.tasks`、`data.finance` 等数据源实时生成。

打赏通知也走同样的模式：在 `computeInboxMessages` 中新增一段，扫描 `data.tips` 数组。

每条发给当前用户的打赏生成一条消息：

```
type: 'tip_received'
category: 'tip'
icon: 💬

内容：「砚仁 送了 🌹小花 · 1NT」（免费鼓励不显示金额）
createdAt: tip.at（打赏时间）
id: 'tip_' + index（在 data.tips 数组中的索引，支持已读追踪）
跳转：无（留在当前页）
```

### 1.2 变动

- `sendTip` 函数：**不改**（它只负责写入 `data.tips`，打赏数据已经在那了）
- `computeInboxMessages`：新增 ~15 行，扫描 `data.tips` 生成 `tip_received` 消息
- `TYPE_ICON` / `TYPE_LABEL` 新增 `tip_received` 条目

## 二、信箱分类

### 2.1 五个分类

把现有 18 种消息类型 + 新增的 `tip_received`，归入 5 个分类：

| 分类 | 图标 | 包含的消息类型 |
|------|:--:|------|
| 📋 任务 | `task` | `submitted`（待审核）、`overdue`（逾期）、`progress_low`（进度掉队）、`deadline_close`（截止临近） |
| ✅ 审批 | `review` | `review_approved`（审核通过）、`review_rejected`（审核退回） |
| 💬 打赏 | `tip` | `tip_received`（收到打赏）📌新增 |
| 💰 财务 | `finance` | `nt_received`（NT到账）、`pool_low`（NT池预警） |
| 📅 活动 | `event` | `meal_*`（订餐）、`game_soon`（桌游）、`meeting_*`（开会）、`auction_*`（拍卖）、`inn_*`（客栈） |

### 2.2 信箱界面变化

**之前**：

```
┌─ 📬 信箱 ──────────────────┐
│ 🔴 新消息（5）              │
│  🆕 宏心 提交了 3 项任务…    │
│  ⚠️ 朝林 2 项任务逾期…      │
│  ✅ 「采购」已通过审核…       │
│  ⏰ 「布置」今天截止…        │
│  💰 收到 NT 50…            │
│                             │
│ 📋 更早（12）▾              │
│ [全部标为已读]               │
└─────────────────────────────┘
```

**之后**：

```
┌─ 📬 信箱 ──────────────────────────┐
│ [📋任务 2] [✅审批 1] [💬打赏 0] [💰财务 1] [📅活动 0] │
│                                     │
│ 🔴 新消息（4）                       │
│  🆕 宏心 提交了 3 项任务…            │
│  ⚠️ 朝林 2 项任务逾期…              │
│  ⏰ 「布置」今天截止…                │
│                                     │
│ 📋 更早（8）▾                       │
│ [全部标为已读]                       │
└─────────────────────────────────────┘
```

- 顶部一行分类标签，显示每类未读数
- 默认显示「全部」
- 点击某个分类 → 只显示该分类的消息
- 点击「全部标为已读」→ 只标当前分类的已读（还是全部？建议全部标）

### 2.3 实现

在 `renderInbox()` 函数内：

1. 给每个消息类型一个 `category` 字段
2. 加一个分类变量 `inboxCategory = 'all'`
3. 顶部渲染分类标签行
4. 过滤逻辑：`if (inboxCategory !== 'all') msgs = msgs.filter(m => m.category === inboxCategory)`

改动量：~30 行（纯 inbox 渲染函数内部）。

```
A：sendTip 加打赏通知（~8 行）

```

## 五、自查

- **少即是多**：5 个分类够用，不建二级菜单。分类标签一行显示，不增加面板高度。
- **复用优先**：所有消息类型已定义，只需加 category 字段。打赏通知复用现有消息渲染函数。
- **需求对齐**：打赏缺通知 → sendTip 加。信箱消息多看不清 → 分类过滤。
- **全局最优**：分类字段加在消息对象上，后续新增的消息类型自动归入某个分类。
- **长期主义**：分类体系是平的（不是嵌套的），以后要加新分类只需在标签行加一个按钮。
