# 个人工作台 · 设计方案

> "我的资料"改名 + 侧边栏布局 + 营队/私人双轨制
> 2026-07-07

---

## 〇、为什么要改

### 现在的问题

1. "我的资料"听起来像档案库，实际上它是个人工作台
2. 营队发的任务和私人委托混在一起，NT 收支也混在一起——分不清哪些是组织经费、哪些是个人收入
3. UI 是平铺的，任务、材料、流水全挤在一个页面，没有结构

### 改什么

1. **改名**："我的资料" → 用户自己的名字（如「砚仁」），作为个人工作台入口
2. **侧边栏布局**：左侧导航，右侧内容，结构清晰
3. **双轨制**：营队工作和私人委托分离，NT 池分离，流水分离
4. **新字段**：任务加 `scope`（camp/personal），finance 加 `scope`

---

## 一、双轨制——核心逻辑

### 两套 NT 池

```
营队 NT（camp_nt）
├─ 来源：营队任务奖励（组织发放）
├─ 用途：不能用于个人消费（是"工资"，记录性质）
└─ 管理：管理员/NT管理发放

个人 NT（personal_nt）
├─ 来源：私人委托收入、转账、其他
├─ 用途：自由支配
└─ 管理：自己管理
```

### 两套任务

```
🏕️ 营队工作
├─ 来源：管理员/共建者在 NT管理 或 猎人公会 发布
├─ 经费：从营队 NT 池支出
├─ 约束：需要审核（发布者审核）
└─ 展示：在个人工作台的"营队工作"标签下

✏️ 私人委托
├─ 来源：任何人可以发布私人委托
├─ 经费：从发起人的个人 NT 支出
├─ 约束：可设置是否需要审核
└─ 展示：在个人工作台的"私人委托"标签下
```

### 两套流水

```
💰 流水
├─ 🏕️ 营队流水：营队任务奖励、营队材料支出
└─ ✏️ 私人流水：私人委托收入、个人转账、个人消费
```

## 二、UI 布局

### 整体结构

```
┌─ 砚仁 · #A3F2 ─────────────────────────────────────────────┐
│ ┌─ Sidebar (180px) ─┐ ┌─ Main Content ───────────────────┐ │
│ │                    │ │                                  │ │
│ │  👤 个人总览       │ │  [选中后展示对应内容]            │ │
│ │  🏕️ 营队工作  [3] │ │                                  │ │
│ │  ✏️ 私人委托  [1] │ │                                  │ │
│ │  💰 流水          │ │                                  │ │
│ │  ⚙️ 设置          │ │                                  │ │
│ │                    │ │                                  │ │
│ │ ─────────────────  │ │                                  │ │
│ │ 💎 营队 12,500     │ │                                  │ │
│ │ 🔗 个人 3,200      │ │                                  │ │
│ │                    │ │                                  │ │
│ └────────────────────┘ └──────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 侧边栏

```
👤 个人总览         ← 头像、ID、简介、NT 汇总
🏕️ 营队工作  [3]    ← 数字 = 进行中的营队任务数
✏️ 私人委托  [1]    ← 数字 = 进行中的私人委托数
💰 流水             ← 收支记录（可切换营队/私人）
⚙️ 设置             ← 改头像/名字/密码/钱包
────────────────────
💎 营队 NT 12,500   ← 营队 NT 余额
🔗 个人 NT 3,200    ← 个人 NT 余额
```

### 各板块内容

| 侧边栏项 | 主内容区展示 |
|---------|------------|
| 👤 个人总览 | 头像、ID、简介、NT 余额合计、最近 5 条流水、进行中的任务预览 |
| 🏕️ 营队工作 | 筛选 tabs：[进行中][待审核][已完成] → 任务卡片列表，每张卡：任务名、截止日期、NT奖励、状态、[提交]按钮 |
| ✏️ 私人委托 | 同上结构，但 scope=personal |
| 💰 流水 | 筛选 tabs：[全部][营队][私人] → 时间线列表：日期、金额、来源/去向、备注 |
| ⚙️ 设置 | 头像选择器、名字修改、密码修改（旧密码+新密码+确认）、钱包地址绑定 |

## 三、数据字段扩展

### data.tasks 新增字段

```js
{
  scope: 'camp',           // 'camp' | 'personal'   — 营队工作 or 私人委托
  visibility: 'all',       // 'all' | 'adventurer' | 'npc' | 'visitor' | 'season_4' | …
  reviewer: '',            // 审核人名字（空 = 发布者自己审）
  // … 现有字段不变
}
```

### data.finance 新增字段

```js
{
  scope: 'camp',           // 'camp' | 'personal'
  // … 现有字段不变
}
```

### data.members 新增字段

```js
{
  id: '#A3F2',             // 系统唯一 ID，注册时自动生成，不可改
  wallet: '',              // web3 钱包地址
  personal_nt: 0,          // 个人 NT 余额
  camp_nt: 0,              // 营队 NT 余额
  badges: [],              // 勋章列表
  // … 现有字段：name, role, avatar_seed, bio 等不变
}
```

### nt_users 新增字段

```js
{
  id: '#A3F2',             // 系统唯一 ID
  pwd_hash: '',            // 密码（加密存储）
  season: '',              // 所属期数，如 '4'
  // … 现有字段：name, role 等不变
}
```

## 四、和现有系统怎么衔接

### 和任务公告栏的关系

```
任务公告栏（全局广场）
├─ 显示所有可领任务（根据 visibility 过滤）
├─ 🏕️ 营队任务 → 领取后进入领取者的"营队工作"
├─ ✏️ 私人委托 → 领取后进入领取者的"私人委托"
└─ 每条任务标注 scope 标签（🏕️营队 / ✏️私人）

个人工作台
├─ 🏕️ 营队工作 → 只显示 scope=camp 且自己领取的
├─ ✏️ 私人委托 → 只显示 scope=personal 且自己领取的
├─ 发布的任务 → 显示自己发布的（不管 scope）
└─ 待审核 → 显示 scope=camp 且 reviewer 是自己的
```

### 和 NT管理的关系

```
NT管理（不变）
├─ 发布营队任务（scope 自动设为 camp）
├─ 审核提交
├─ 管理营队 NT 池
└─ 新增：可选设置任务 visibility（谁能领）
```

### 和信箱系统的关系

```
📬 信箱（不变）
├─ 营队任务审核通过 → 通知
├─ 私人委托完成 → 通知
├─ NT 到账（区分营队/私人）→ 通知
└─ 只在渲染时加 scope 标签，不改变消息结构
```
