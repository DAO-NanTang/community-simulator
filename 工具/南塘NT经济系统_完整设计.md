# 南塘 NT 经济系统 · 完整设计 v8

> **平台 = 虚拟账本 + P2P 清算引擎。** 不铸币、不持币、不做银行。
> **贡献值 = 社区真正的价值尺度。** NT 只是结算工具。
> **NFT = 贡献值的链上固化。** 劳动不再消失，它成为你永久的凭证。
> 2026-07-07 · 经多轮对谈确认

---

## 〇、平台定位（最终确认）

```
平台是什么：
  ✅ 一个虚拟记账系统——记录每个人在社区内的劳动和消费
  ✅ 一个 P2P 清算引擎——月度结算时算出谁该给谁多少 NT
  ✅ 一个贡献值计算器——衡量「你的 NT 是劳动挣的还是外部灌的」
  ✅ 跳转到 web3 钱包执行转账

平台不是什么：
  ✗ 不铸造 NT（铸币权在南塘 DAO，不在平台）
  ✗ 不销毁 NT（平台不控制任何人的钱包）
  ✗ 不持有 NT 池（平台没有「社区钱包」，只有虚拟账本）
  ✗ 不做 NT ↔ RMB 兑换
  ✗ 不是银行——是清算所

平台的唯一目的：
  防止南塘 DAO 乱发 NT 造成的输入性通胀冲击社区劳动价值。
  手段不是「堵住外部 NT」，而是「贡献值门槛」——
  你可以在钱包里有一百万 NT，但没有贡献值，你在社区里买不了画。
```

## 一、三层价值体系（v7 版）

```
┌────────────────────────────────────────────────────────────┐
│                    南塘社区 · 三层价值                       │
│                                                            │
│  ① NT（web3 钱包余额）                                      │
│     链上真币，南塘 DAO 发行。平台不铸不毁。                   │
│     来源：DAO 分发 / 他人转账 / 社区劳动结算收入              │
│     只在月度结算时实际转账                                    │
│                                                            │
│  ② 贡献值（劳动承诺，可转移）                                 │
│     平台虚拟记账——衡量你对社区的净贡献                        │
│     = 劳动所得 − 消费支出 + 接收的转移                       │
│     消费时转移给卖家（75%）+ 公共池（25% 磨损）               │
│     贡献值决定你能在社区做什么：买画、发委托、预定优先          │
│                                                            │
│  ③ 经验值（资历，只涨不跌）                                   │
│     = 所有劳动所得贡献值的永久累计                            │
│     解锁权限、展示资历等级                                    │
└────────────────────────────────────────────────────────────┘
```

### NT 和贡献值的本质区别

```
NT = 结算工具。你有多少 NT，取决于你钱包里有多少。
     外部可以灌进来，DAO 可以乱发——平台管不了。

贡献值 = 社区价值。你有多少贡献值，取决于你在社区做了多少劳动。
       外部 NT 灌不进来，DAO 乱发影响不了——平台管得住。

所以：
  在社区内，贡献值 > NT。
  你有 100 万 NT 但贡献值为 0 → 买不了画。
  你有 500 NT 但贡献值 2000 → 你的每一分 NT 都有劳动背书。
```

## 二、虚拟账本：日常怎么记账

### 2.1 所有操作都是加减法

```
平台不持有任何 NT。
平台只维护一个虚拟账本。

  小红打扫画室 → 平台记账：
    小红 | +40 NT（应收）| +40 贡献值 | +40 经验值
    系统 | -40 NT（应付）|           |

  朝林吃晚饭 → 平台记账：
    朝林 | -15 NT（应付）| -15 贡献值 |
    系统 | +15 NT（应收）|           |

  砚仁买小红的画（200 NT + 200 贡献值）→ 平台记账：
    砚仁 | -200 NT（应付）| -200 贡献值 |
    小红 | +200 NT（应收）| +150 贡献值（75%）|
    公共贡献池 |         | +50 贡献值（25% 磨损）|

  到了月底：
    小红应收：40 + 200 = 240 NT
    朝林应付：15 NT
    砚仁应付：200 NT

    系统生成结算单：
      朝林 → 小红 15 NT
      砚仁 → 小红 200 NT
      （砚仁另外还有一笔 25 NT 的应收，从 fc 那里来……）

    所有人确认 → 批量执行转账 → 链上结算
```

### 2.2 虚拟账本结构

```js
// 这是平台维护的唯一「账本」——不是钱包，不是池子
nt_virtual_ledger: {
  month: "2026-07",
  entries: [
    {
      id: "entry-001",
      date: "2026-07-11",
      type: "cleaning",          // cleaning | meal_labor | meal_dine | farming | task | sale | accommodation | booking
      description: "大扫除·画室",
      parties: {
        "小红": { nt: +40, cv: +40, xp: +40 },    // NT变动 / 贡献值变动 / 经验值变动
        "公共贡献池": { cv: +0 }                      // 公共池只计贡献值，不计 NT
      }
    },
    {
      id: "entry-002",
      date: "2026-07-11",
      type: "meal_dine",
      description: "饭局·晚餐·用餐",
      parties: {
        "朝林": { nt: -15, cv: -15 },
        "王五": { nt: -15, cv: -15 }
      }
    },
    {
      id: "entry-003",
      date: "2026-07-11",
      type: "sale",
      description: "购买画作「南塘的夏天」",
      parties: {
        "砚仁": { nt: -200, cv: -200 },
        "小红": { nt: +200, cv: +150 },    // 75%
        "公共贡献池": { cv: +50 }             // 25% 磨损
      }
    }
  ],
  settled: false,            // 是否已链上结算
  settlementTxHash: null     // 结算交易哈希
}
```

### 2.3 月底结算

```
每月最后一天 23:59：

  ① 平台汇总本月所有虚拟账目
  ② 计算每个人的净头寸：
     小红：+240 NT（应收）
     朝林：-110 NT（应付）
     砚仁：-180 NT（应付）
     fc：+50 NT（应收）

  ③ 系统生成 P2P 清算方案：
     朝林 → 小红 110 NT
     砚仁 → 小红 130 NT
     砚仁 → fc 50 NT

  ④ 生成批量转账 → 用户确认 → 钱包签名 → 链上结算

  ⑤ 结算完成后 → 每个人的贡献值和经验值正式归档
     → 虚拟账本清零，下月重新开始
```

### 2.4 云村任务 → 虚拟账本（自动数据通路）

```
云村任务完成时（claimer 标记完成，poster 确认）：

  → 触发 event: "task_completed"
  → 自动调用记账函数，写入 nt_virtual_ledger：

     // 委托任务（P2P）
     nt_virtual_ledger.entries.push({
       date: today,
       type: "task",
       description: task.name,
       parties: {
         [claimer]: { nt: +task.points, cv: +task.points, xp: +task.points },
         [poster]:   { nt: -task.points, cv: -task.points }
       }
     })

     // 大扫除任务
     parties: {
       [claimer]: { nt: +finalNT, cv: +finalNT, xp: +finalNT }
     }

     // 饭局劳动任务
     parties: {
       [claimer]: { nt: +finalNT, cv: +finalNT, xp: +finalNT }
     }

  同时更新个人三栏数据（即时到账，非月底一次性）：
     nt_users[claimer].contributionValue += task.points
     nt_users[claimer].experienceValue += task.points
     （NT 余额在月底 P2P 结算时才实际转账，
       但贡献值和经验值即时更新）

效果：
  · 任务完成 → 账本自动更新，无需人工双录
  · 贡献值和经验值即时反馈（个人仪表盘马上看到变化）
  · NT 月底统一结算（减少链上 Gas 费）
  · 数据一致性由系统保证，不会出现「两张皮」
```

## 三、贡献值：社区真正的价值尺度

### 3.1 贡献值转移（不是销毁）

```
甲买乙的画：200 NT + 200 贡献值

  甲的贡献值 -200（失去劳动承诺）
  乙的贡献值 +150（收到劳动承诺，扣除 25% 公共磨损）
  公共贡献池 +50（磨损 = 社区的「流转税」）

  贡献值没有消失。
  它从买家转移到了卖家，一部分给了社区。
  卖家收到贡献值 → 有动力继续创作。
  社区收到磨损 → 用于补贴新人。
```

### 3.2 贡献值累进税（P2 · 延后启用）

```
每季度末，贡献值 > 2000 的部分，征收 5% → 转入公共贡献池。

  小红贡献值 3000 → (3000-2000) × 5% = 50 → 贡献值变为 2950
  砚仁贡献值 500 → 不到 2000 → 不征税

  目的：防止老成员无限囤积贡献值，保持社区流动性。

已知漏洞（P2 修复）：
  按「季度末瞬时值」征税 → 突击消费可避税，突击劳动反被税。
  → 方案：改为按「季度内日均贡献值」征收（需每日快照）。
  → 决定：先上线跑一个季度，拿到真实数据后再启用。
    在此之前累进税暂不执行。
```

### 3.3 贡献值为负的处理

```
贡献值 < 0 → 🟡 在消耗社区资源

连续 2 周为负 → 🟠 提醒
连续 4 周为负 → 🔴 互评会讨论

兜底机制（按顺序）：
  ① 红牌保护期（最长 1 个月）——疾病/家庭急事可申请
  ② 零工驿站——推送双倍贡献值救助任务
  ③ 信用借贷——借 NT 还「NT + 10% 贡献值利息」
  ④ 劝退——以上都用过仍无效 → 社区投票
```

### 3.4 贡献值决定你能做什么

```
贡献值门槛（平台自动检查）：

  预定热门空间时段 → 按贡献值排序
  发布委托任务 → 需要贡献值 ≥ 0
  购买社区作品 → 卖家设定贡献值门槛（如 200 贡献值）
  发起公约修订提案 → 需要贡献值 ≥ 500
  参与陪审团 → 需要经验值 ≥ 500

外部 NT 再多，贡献值不够 → 受限。
```

## 四、住宿费和空间费

### 4.1 住宿费

```
固定价格，对外公开，不浮动。

定价：RMB 30 / 天 或 NT 30 / 天（任选其一）

  选 RMB → 线下付现金/转账
  选 NT  → 虚拟记账，月底 P2P 结算时转给劳动者

为什么 RMB 和 NT 二选一？
  · 有的人 NT 多 → 用 NT 付
  · 有的人 NT 少 → 用 RMB 付
  · 两种方式都支持社区运转
  · RMB 收到的钱用于社区公共开支

修订方式：
  季度公约会议投票 → 多数通过 → 下季度生效
```

### 4.2 水电费

```
水电费单独计算，每月按人头均摊。

线下收取 RMB → 用于缴纳实际水电账单。

优先级：
  社区活动收入 → 优先覆盖水电费
  若活动收入足够覆盖 → 成员免摊
  若不足 → 差额部分由在地成员均摊
```

### 4.3 空间预定费

```
固定费率。如画室 2 NT/小时。
虚拟记账，月底 P2P 结算。

社区活动使用空间 → 免预定费
（鼓励办活动，活动本身为社区创造价值）
```

### 4.4 访客住宿

```
短期访客（≤4 天）：
  同样 RMB 30 或 NT 30 / 天
  若没有 NT → 通过新手任务获得（见 §七）
```

## 五、非标劳动仲裁庭（陪审团）

```
① 申报
   任何人可申报「非标劳动」：
   「整理了社区三个月水电账单，花了 2 小时，期望 20 NT」

② 陪审团
   系统随机抽取 3 名经验值 > 500 的成员
   匿名打分：0 ~ 30 NT

③ 记账
   取三人均值 → 申报者应收 NT +N
   公共贡献池 → 应付 NT -N（月底从净消费者那里收）

④ 陪审团激励
   每位陪审团成员 +2 NT（固定劳务费，也从公共贡献池记账）
```

## 六、社区公共贡献池（虚拟）

```
公共贡献池不持有 NT，只持有「贡献值」和「虚拟 NT 债权」。

收入（贡献值）：
  · 交易磨损（25%）→ 贡献值
  · 累进税（贡献值 > 2000 部分的 5%）→ 贡献值

收入（虚拟 NT 债权，月底从净消费者回收）：
  · 红牌保护期垫付
  · 陪审团发放
  · 零工驿站双倍结算

### 6.1 公共池的月底清算（债权优先级）

```
公共池不持有 NT 钱包。但它在月底 P2P 清算时有「优先级债权」：

  ① 汇总本月所有公共池支出：
     陪审团发放 + 新手任务 + 红牌垫付 + 零工驿站双倍结算
     = X NT（本月公共池总债权）

  ② 月底 P2P 清算时：
     Step 1：从「净应付者」的应付总额中，优先划出 X NT → 分配给公共池债权人
     Step 2：剩余的应付 NT → 按正常 P2P 分配给净劳动者

  ③ 如果净应付总额 < X NT（消费不够覆盖公共支出）：
     → 差额从「住宿费/空间费应收 NT」中补足
     → 再不够 → 滚入下月公共池债权（延迟支付，不计利息）

  ④ 如果净应付总额 > X NT（消费充足）：
     → 公共池债权全部兑付
     → 多余 NT 按正常 P2P 分配给净劳动者

公共池本身没有钱包。
它只是一个「清算优先级」——在月底 P2P 结算时，
公共池的债权人排在个人债权人前面。
```

### 6.2 公共池收支一览

```
收入（贡献值）：
  · 交易磨损（25%）→ 贡献值
  · 累进税（贡献值 > 2000 部分的 5%）→ 贡献值（P2 延后启用）

支出（虚拟 NT 债权，月底清算时优先兑付）：
  · 陪审团发放
  · 新手任务 NT
  · 红牌保护期垫付
  · 零工驿站双倍结算
```

## 七、新成员冷启动（新手任务制）

### 7.1 原则

```
住宿费从第一天起就要付（RMB 30 或 NT 30/天）。
没有免费住宿，没有社区垫付。

初始 NT 怎么来？
  → 通过「新手任务」获得。
  → 每个新手任务都有明确的价值。
  → 完成即获得 NT——不施舍，不白给。

为什么要这样？
  → NT 发的每一分都有价值。
  → 认识社区、了解公约、和人建立连接——这些事情本身就是有价值的。
  → 防止「贪小便宜」——没有免费午餐。
```

### 7.2 新手任务清单

```
新成员到达后，系统自动推送新手任务：

  ┌────────────────────────────────────────────┐
  │         🌱 欢迎来到南塘 · 新手任务           │
  │                                            │
  │  □ 签署社区公约                     +5 NT   │
  │    阅读并签署南塘在地社区公约                │
  │                                            │
  │  □ 认识 3 位在地成员               +15 NT   │
  │    和 3 位在地成员交换名字和一句话介绍        │
  │    （对方在平台上确认即可）                   │
  │                                            │
  │  □ 社区空间打卡                     +20 NT   │
  │    参观并打卡 5 个社区空间：                 │
  │    · 画室 · 厨房 · 菜地 · 展览厅 · 宿舍     │
  │    （在每个空间扫码或标记位置）              │
  │                                            │
  │  □ 参加 1 次饭局                    +10 NT   │
  │    报名参加一次社区饭局                      │
  │                                            │
  │  □ 参加 1 次大扫除                  +15 NT   │
  │    参加第一次周六大扫除                      │
  │                                            │
  │  全部完成额外奖励：                +10 NT   │
  │  ─────────────────────────                 │
  │  总计可获得：                      75 NT    │
  └────────────────────────────────────────────┘

75 NT = 约 2.5 天的住宿费（NT 部分）
新成员有足够的时间通过劳动和饭局继续挣 NT。
```

### 7.3 贡献值抵押机制（防止「白嫖」）

```
新手完成任务 → NT 立即到账（可以付住宿费，解决生存问题）

但贡献值不是一次性全部生效：
  新手获得的 75 NT 对应 75 贡献值
  → 先进入「新手冻结池」，暂不生效
  → 每周大扫除后，自动解冻 50%（37.5，四舍五入）
  → 两周后全部解冻

如果新人两周内离开（未参加任何大扫除）：
  → 冻结池中的贡献值 → 自动转入社区公共贡献池
  → 相当于新人「白嫖」的部分被社区回收

效果：
  · 老成员看到新人也在持续劳动 → 公平感上升
  · 新人需要通过持续参与才能完全「激活」自己的贡献值
  · 防止「拿了 75 NT 就走」的投机行为
```

### 7.4 新手任务是一次性的

```
每个成员只能完成一次新手任务。
这不是「低保」，这是「入学引导」。

任务做完 → NT 到账 → 进入正常社区经济循环。
之后的 NT 全部来自：大扫除、饭局劳动、农活、委托、陪审团。
```

### 7.4 短期访客（≤4 天）

```
同样付 RMB 30 或 NT 30 / 天。

若没有 NT：
  → 同样可以做新手任务获得初始 NT（如果还没做过）
  → 或者：帮厨/洗碗 1 小时 → 记账 10 NT → 付住宿费
  → 或者：直接用 RMB 付住宿费
```

## 九、NFT：贡献值的链上固化（v8 新增）

### 9.1 问题：劳动没被看见

```
志愿者在社区做了很多事：
  打扫、做饭、陪护、修东西、整理账目、接待新人……

但这些劳动：
  · 只存在于大家的记忆里，时间久了就模糊
  · 被「人情」和「关系」包裹，没有固化下来
  · 离开了社区，没有人知道你做过什么
  · 无法在简历上写「我在南塘打扫了 48 次画室」
  · 因为这些劳动没有凭证——没有文凭、没有证书、没有记录

结果：
  做了很多实事的人，对外「无法证明」自己做过。
  这是对劳动者最大的不公平。
```

### 9.2 NFT 的定位

```
NFT ≠ 投机品
NFT ≠ 炒币
NFT = 链上永久凭证

南塘 NFT = 你的社区劳动履历，被固化在链上。

  它不是凭空铸造的——每一枚 NFT 背后都有真实的劳动记录。
  它不可篡改——一旦上链，任何人都无法抹去你的贡献。
  它有 RMB 价值——因为它证明了一个人曾经真实地为社区付出过。

  这不是「投资品」。
  这是「劳动文凭」。
```

### 9.3 NFT 类型

#### 类型 A：贡献里程碑 NFT（展示型 · 不可转移 · 自动铸造）

```
触发条件：贡献值累计达到特定阈值

  🌱 贡献值达 100  → 铸造「新芽」NFT
  🌿 贡献值达 500  → 铸造「小树」NFT
  🌳 贡献值达 1000 → 铸造「大树」NFT
  🏔️ 贡献值达 2000 → 铸造「山峰」NFT
  ⭐ 贡献值达 5000 → 铸造「星辰」NFT

每个 NFT 包含元数据：
  {
    "type": "贡献里程碑",
    "level": "大树",
    "cvAtMint": 1000,
    "member": "小红",
    "period": "2026-07 ~ 2026-12",
    "highlights": [
      "打扫画室 48 次",
      "做饭 23 次",
      "接待新成员 5 人"
    ],
    "mintedAt": "2026-12-31"
  }
```

#### 类型 B：年度贡献凭证 NFT（展示型 · 不可转移 · 年度铸造）

```
每年末自动生成。像一份「年度劳动报告」被铸造成 NFT。

  「2026 年度 · 南塘社区贡献者 · 小红」
   共计劳动 186 次
   打扫画室 48 次、厨房 12 次
   参与饭局 34 场（主厨 8 场、备菜 15 场）
   农业劳动 23 次（种植 5、浇水 12、收割 6）
   接待新成员 5 人
   陪审团参与 12 次
   累计贡献值增长：+1,860
   经验值累计：2,450

  → 铸造成 1 枚年度 NFT
```

#### 类型 C：作品创作 NFT（流通型 · 可转移 · 创作者自主铸造）

```
画作、手工艺品、文字创作——创作者可以将其铸造成 NFT。

  NFT 元数据除了作品信息外，附带：
    · 创作者的当前贡献值（作为「创作者信誉背书」）
    · 创作者的当前经验值
    · 创作所使用的空间（画室/厨房/菜地 → 有故事）

  买家的购买行为 → 贡献值转移 → 也记录在链上
  → NFT 的交易历史本身就是社区贡献的流转史
```

#### 类型 D：特殊贡献 NFT（展示型 · 不可转移 · 陪审团认定后可选铸造）

```
非标劳动通过陪审团认定后，可选择铸造成 NFT。

  「法律咨询服务 · 帮助社区审阅租赁合同 · 2026-07-11」
  陪审团评分：20/30 NT
  → 铸造成 NFT（可选，不强制）
  → 永久记录在链上
```

### 9.4 NFT 的价值来源

```
为什么南塘 NFT 值钱？

  不是因为炒作。
  不是因为稀缺性（人为制造的）。
  不是因为「社区名气」。

  值钱 = 因为它证明了一个真实的人做了真实的劳动。

  对劳动者自己：
    · 它是一份「社区劳动文凭」
    · 可以在面试时展示：「我在南塘生活了 6 个月，这是我的链上记录」
    · 可以用于申请下一家社区/合作社/DAO 的成员资格

  对社区外的观察者：
    · 它是一份不可篡改的信任证明
    · 「这个人不是嘴上说说的志愿者，这个人是实打实做了事的」

  对社区本身：
    · 它是社区的「价值输出」
    · 南塘不仅是一个物理空间——它能够把「劳动」固化为「凭证」
    · 这个凭证本身就有社会价值
```

### 9.5 NFT 两类：展示型 vs 流通型

```
                ┌──────────────────────────────────────────┐
                │          南塘 NFT 分类                     │
                │                                          │
                │  展示型 NFT（不可转移）                      │
                │    · 贡献里程碑（🌱🌿🌳🏔️⭐）              │
                │    · 年度贡献凭证                           │
                │    · 特殊贡献凭证                           │
                │    · 绑定钱包，不可出售                      │
                │    · Gas 费：社区支付（Polygon L2）          │
                │    · 用途：个人简历、社区荣誉、对外证明        │
                │                                          │
                │  流通型 NFT（可转移）                        │
                │    · 作品创作（画作/手工艺品/文字）           │
                │    · 可出售、可转让                          │
                │    · Gas 费：创作者自行承担                  │
                │    · 用途：艺术品交易、收藏                   │
                │    · 价值来源：艺术品本身，非劳动凭证         │
                └──────────────────────────────────────────┘

为什么分两类？
  · 展示型 =「劳动文凭」→ 不可交易 → 杜绝炒作 → 社会价值
  · 流通型 =「艺术品」→ 可交易 → 价值来自作品本身 → 投机风险可控
  · 界限清晰：谁都能看出「这是一张劳动证书」vs「这是一幅画」
```

### 9.6 L2 与 Gas 费

```
链选择：Polygon（Matic）
  · Gas 费极低（每笔 $0.01-0.05）
  · 生态成熟，与 Ethereum 完全兼容
  · 国内用户可通过 MetaMask 等钱包接入

Gas 费承担：
  展示型 NFT → 社区支付
    来源：住宿费 RMB 部分拨出少量预算
    或：社区成员自愿捐赠 Gas 基金
    估算：全年全部成员铸造展示型 NFT ≈ $2-5（几十元 RMB）

  流通型 NFT → 创作者自行承担
    每笔约 $0.01-0.05 ≈ 几毛钱 RMB
    创作者自主决定是否铸造

铸造流程：
  展示型 → 触发条件满足 → 推送「是否铸造？」→ 确认
    → 社区 Gas 钱包支付 → 链上铸造 → 进入用户钱包
  流通型 → 创作者主动发起 → 填写作品信息
    → 跳转钱包 → 创作者支付 Gas → 链上铸造
```

### 9.7 NFT 和贡献值的关系

```
贡献值 = 平台内部的虚拟记账单位
  可以增减（劳动 +、消费 -）
  社区内部使用

NFT = 贡献值的链上固化
  不可篡改
  永不消失
  可以对外展示
  社区外部也认可

关系：
  贡献值是「活的」→ 每天都在变
  NFT 是「凝固的」→ 把某个时刻的贡献值快照永久保存

  就像：
  银行卡余额是「活的」→ 每天进出
  工资条是「凝固的」→ 证明了某段时间你挣了多少钱
```
