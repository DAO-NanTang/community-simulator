# 行动方案：dashboard.html → 南塘世界·双面终端

> 原始归档：`dashboard_archive_20260706.html`
> 设计依据：`方案/世界终端_设计方案.md` + `方案/世界终端_完整愿景.md`

---

## 核心理念

现在 dashboard.html 是一个管理工具——所有人看到差不多的东西，共建者和玩家混在一起。

要把它变成**一个网页，两张脸**：
- **共建者**登录 → 看到「世界终端」（深色终端风格，能看到所有人的数据，能创建世界）
- **冒险者**登录 → 看到「副本游戏」（暖色冒险风格，只能看到自己的数据和排行榜）

同一份数据，两种完全不同的体验。共建者可以随时切到冒险者视角预览效果，但冒险者不能反过来偷看管理后台。

---

## 分三个阶段做

### 第一阶段（这次做）：搭骨架 + 登录页

**这是最关键的阶段——把"两张脸"的框架搭出来。**

#### 1. 登录页（新的入口）

打开网页不再直接看到仪表盘，而是先看到一个登录页：

- 页面上方：世界名称「南塘·第四期」+ 营期时间
- 三个入口按钮：
  - 「我是新来的冒险者」→ 选头像 → 填名字 → 设密码 → 进入冒险者界面
  - 「我是世界建造者」→ 选头像 → 填名字 → 设密码 → 填邀请码 → 进入共建者界面
  - 「我上次来过」→ 看到一排头像，点自己的头像 → 输密码 → 进入
- 已注册的用户头像分两排显示：🧙 共建者一排，⚔️ 冒险者一排
- **账号密码存在 localStorage 里**（不需要后端服务器，纯前端实现）

#### 2. 两套界面

登录后根据身份展示不同的界面：

**共建者看到——世界终端（深色底 + 金色字，像游戏控制台）：**

顶部有一个模式切换栏：
```
[创世] [监察] [预测] [整理] [结算]    [⚔️ 切到冒险者视角]
```

- **创世模式**：就是现在 5 步向导的内容（①创世设定 → ②任务之书 → ③委托分配 → ④封印指定 → ⑤世界启动）
- **监察模式**：开营后看全局数据——所有冒险者的进度条、今日任务状态、资源消耗
- **预测模式**：结营前几天看完成率预测、NT 结算预测
- **整理模式**：结营后的数据归档、结项清单
- **结算模式**：全局成就展示

共建者能看到**所有人的所有数据**——每个冒险者的任务、NT、进度。

顶部有「切到冒险者视角」按钮，点了就能预览冒险者看到的样子。

**冒险者看到——副本游戏（暖色底 + 羊皮纸纹理）：**

- 一个简洁的导航栏：[我的委托] [南塘地图] [冒险任务] [排行榜] [世界时间线] [通关结算]
- **只能看到自己的数据**——自己的任务、自己的 NT、自己的进度
- 排行榜可以看到所有人的名字和排名（但看不到别人的详细数据）
- 没有切换按钮，不能偷看共建者后台

#### 3. 世界状态条

两个界面顶部都有一条状态栏：
```
🌍 南塘·第四期 | ⏳ 距启动 X天 | 👥 冒险者 X/XX | 💰 ¥XX,XXX | 💎 NT XX,XXX
```

#### 4. 名字全部改掉

| 现在的名字 | 改成 |
|-----------|------|
| 📝 营地设置 | 合并到创世模式里 |
| 📋 任务大厅 | 📋 猎人公会 |
| 🗺️ 攻略地图 | 🗺️ 冒险任务 |
| 👥 南塘NPC | 👥 冒险者档案 |
| ⚔️ 游戏玩家 | 合并到冒险者档案 |
| 📅 日历系统 | 📅 世界时间线 |
| 💰 公会资金池 | 💰 资源金库 |
| 🧾 结算界面 | 🧾 通关结算 |
| "新增任务" | "发布委托" |
| "激励点" | "NT 奖励" |

#### 5. 任务变成卡片

在猎人公会和冒险任务里，任务不再是表格行，而是一张张游戏卡片：
- 宣传类 = 金色边框（传令卡）
- 课程类 = 绿色边框（修行卡）
- 生活类 = 棕色边框（生活卡）
- 财务类 = 紫色边框（财富卡）
- 管理类 = 蓝色边框（秩序卡）
- NT 奖励用金色大字
- 完成的卡片盖上半透明印章

### 第二阶段（下次做）：内容充实

骨架搭好之后，往里面填内容。

#### 1. 冒险者欢迎页

冒险者第一次登录时，先看到一个欢迎页：
- 世界介绍（这个营是干什么的）
- 自己的主线任务数量
- 食宿规则说明
- 「⚔️ 进入世界」按钮

#### 2. 个人状态卡片

冒险者主界面上方有一个自己的状态卡：
- 头像 + 名字
- NT 余额 + 食宿还能住几天
- 主线/支线/共创完成进度条
- 下一个截止的闸口是什么

#### 3. 南塘地图

把「冒险任务」里的空间地图换掉，改成真实的南塘实景地图：
- 用实际的素社、画室、农田、南塘村照片或卫星图
- 标注每个地点正在进行的任务
- 点击地点能看到相关委托

#### 4. 今日焦点面板（监察模式）

共建者的监察模式里，系统自动列出今天需要关注的事：
- 🔴 谁的任务逾期了
- 🟡 什么还没做
- 🟢 什么已经完成

#### 5. 排行榜

冒险者能看到：
- 主线进度榜（谁完成了多少）
- NT 积分榜（谁赚了多少）
- 称号墙（每个人获得的独特称号）

#### 6. 个人通关证书

每个冒险者结营时看到自己的证书：
- 完成了多少任务
- 获得了什么称号
- 带出了什么能力
- 结营感言

#### 7. 日历热力图 + 心流曲线

世界时间线里加上：
- 每天任务密度标记（绿/黄/红）
- 难度曲线可视化

### 第三阶段（远期，可以不做完）：高级功能

这些是锦上添花的东西，看时间做。

#### 1. 交易市场

冒险者之间可以发布交易：
- 「我用颜料×3 交换 NT 30」
- 「求帮厨一次，报酬 NT 50」

#### 2. 同伴动态

显示其他冒险者刚刚做了什么：
- 「墨染完成了 M5 闸口①」
- 「念儿发布了一条交易」

#### 3. 成长记录（旅程日记）

每个冒险者有一页日记：
- 每天一条记录（自动生成 + 手动添加）
- 展示这个人在南塘的完整旅程

#### 4. 资源消耗预测

共建者监察模式里，根据当前消耗速度预测：
- 结营时 NT 剩余多少
- 赏金池够不够用

#### 5. 全局成就结算

共建者结算模式里展示：
- 每个共建者贡献了什么
- 本期数据摘要
- 下期改进清单

#### 6. 后端登录（如果有需要）

目前的登录是 localStorage 实现的纯前端方案（够用）。如果以后需要真正的账号系统，可以接后端。

## 这次能做的 vs 留给后面的

### 这次（第一阶段）全部能做

因为都是前端改 HTML/CSS/JS，不需要服务器，不需要数据库：
- 登录页 ✅
- 两套界面框架 ✅
- 世界状态条 ✅
- 全部改名 ✅
- 卡片样式 ✅
- 权限控制（共建者看全部，冒险者看自己）✅
- 深色/暖色两套主题 ✅

### 留给第二阶段

需要更多设计细节和内容填充：
- 南塘地图（需要真实照片素材）
- 排行榜（需要设计展示逻辑）
- 通关证书（需要设计证书模板）

### 留给第三阶段

比较复杂或者暂时不急：
- 交易市场（交互逻辑复杂）
- 同伴动态（需要事件系统）
- 后端登录（需要服务器）

## 具体怎么改（技术层面）

**不动的东西：**
- 还是那个 dashboard.html 一个文件
- 数据模型照旧（tasks、members、budget、schedule）
- localStorage 结构兼容（已存的数据不会丢）
- 导出导入 JSON 照常用

**怎么改：**
- 在 HTML 最前面加登录页的代码
- CSS 加两套主题变量（`.builder-mode` 和 `.adventurer-mode`）
- JS 加登录/注册逻辑、权限判断、模式切换
- 原有的渲染函数加上权限过滤（冒险者只看自己的数据）
