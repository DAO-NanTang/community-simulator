## 一、任务全生命周期 & 数据流

```
① 创建（来源三选一）
   ├─ 创世Step② → wizardTasks[] → Step④ → executeSync() → data.tasks[]
   ├─ NT管理 showPostTask() → prompt收集字段 → data.tasks.push()
   └─ 猎人公会 saveTask() → 表单 → data.tasks.push/替换

② 分配（assignee）
   创世Step③：wizardTasks[idx]._claimChecked → 转为 data.tasks[].assignee

③ 展示（公告栏） — renderQuestBoard()
   数据源：data.tasks，过滤 status==='active' && claimants存在
   按 questFilter 过滤（type/claimable/claimed）
   ⚠️ 当前不按 scope/visibility 过滤

④ 领取 — claimTask(idx)
   t.claimants.push({name, claimed_at, status:'in_progress'})
   → localStorage.setItem('camp_data', …)
   → renderQuestBoard() + renderWorkspace()

⑤ 提交 — submitTask(idx)
   claimant.status → 'submitted'；submitted_at ← 当前日期
   → renderQuestBoard() + renderWorkspace()

⑥ 审核 — reviewTask(idx, name, action)
   approve: claimant.status → 'completed'
     → data.finance.push({type:'task_reward', to:name, amount:t.points, scope:t.scope})
   reject: claimant.status → 'rejected'
   → renderNTAdmin()

⑦ 结算 — renderSettlementAll()
   汇总：tasks + finance + members → 每人NT余额、任务完成统计、材料费

⑧ 归档
   手动改 status → '已确认'（无自动归档流程）
```

## 二、NT 余额——三个计算点，逻辑不一致

| 位置 | 函数 | 计算方式 | scope区分 |
|------|------|---------|:--:|
| HUD栏 | `updateWorldHUD()` | finance中 `to===user`之和 − `from===user`之和 | ❌ 不区分 |
| 个人工作台 | `renderWorkspace()` | 同上，但按 `tx.scope` 分组累加 | ✅ 区分 |
| 我的资料(旧) | `renderMyProfile()` | 同HUD，不区分scope | ❌ 不区分 |
| NT管理 | `renderNTAdmin()` | `poolTotal - totalIssued - pendingAmount` | ❌ 营队池 |

**问题**：三个地方三个算法。用户在不同页面看到的NT数字可能不一样。

## 三、已知 Bug（按严重程度排序）

### 🔴 P0 — 数据/逻辑错误

| # | Bug | 根因 | 影响 |
|---|-----|------|------|
| 1 | **信箱一次性消红点** | `inbox_lastOpened` 是单时间戳。打开信箱 → 全部"已读"。应该逐条核销 | 管理员错过重要提醒 |
| 2 | **创世任务不同步到工作台** | `renderWorkspace()` camp只查`claimants`（主动领的），不查`assignee`（分配的）。创世Step③分配走assignee | 活动任务页空白 |
| 3 | **预算天数变，餐费不变** | `updateWizBudgetCalc()` 的 DOM 绑定可能不完整 | 预算数字错误 |
| 4 | **NT余额三处不一样** | HUD/workspace/profile 三个函数各自算NT，逻辑不同 | 用户困惑 |

### 🟡 P1 — 功能缺失

| # | Bug | 根因 | 影响 |
|---|-----|------|------|
| 5 | **公告栏不过滤scope/visibility** | 字段已加，`renderQuestBoard()` 没用上 | 私人委托和营队任务混在一起 |
| 6 | **loadData() 可能丢字段** | normalize 时只列了部分字段 | 导入JSON后claimants等丢失 |
| 7 | **reviewTask NT池检查用错字段** | 使用`data.budget.nt_pool`但`updateWizBudgetCalc()`会删掉它 | NT池检查失效 |

### 🟢 P2 — 体验

| # | Bug | 根因 |
|---|-----|------|
| 8 | 任务卡片无法展开看详情 | 无展开/折叠功能 |
| 9 | 头像在设置中提示"点击编辑"但无入口 | 文字有，功能无 |
| 10 | `currentUser` 只有 name+role | 无法显示ID、期数等信息 |

## 五、修复方案

### 总体策略

先修数据流（让数据对）→ 再修功能（让功能全）→ 最后体验（让体验好）。

### Phase A：数据修正（4项，互不依赖可并行）

| A1 | 统一NT余额计算 | 三处统一为一个函数 `calcNTBalance(user, scope?)` |
| A2 | 工作台camp加assignee匹配 | `if (t.assignee===name) myCampTasks.push(…)` |
| A3 | 预算联动修复 | 检查 `updateWizBudgetCalc` 的DOM绑定链 |
| A4 | 信箱逐条核销 | `inbox_lastOpened` → `inbox_seen_ids`（已读消息ID集合） |

### Phase B：功能补全（依赖A完成）

| B1 | 公告栏加scope/visibility过滤 | renderQuestBoard加过滤维度 |
| B2 | NT管理发布时加scope/visibility UI | showPostTask改表单 |
| B3 | 任务卡片加展开/折叠 | 显示详情、材料、子任务 |

### Phase C：清理（依赖A+B验证通过）

| C1 | 模板提取 | PRESET_TEMPLATES → templates/*.json |
| C2 | 加重置功能 | "重置所有数据"按钮 |
| C3 | 清理死代码 | showAddMember、旧renderMyProfile引用等 |
