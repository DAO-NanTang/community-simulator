# 阶段一：身份 + 住宿 + NT 基础

> **目标**：人进来了，住下了，NT 开始流转。
> **状态**：📝 待审查
> **依赖**：无（一切的基础）

---

## 一、本阶段要解决什么问题

1. **谁是谁？** —— 在地的、云端的、管理的，身份要分清
2. **谁住哪？** —— 在地的人住哪个房间、住多久
3. **NT从哪来、到哪去？** —— 最基础的积分流转：注册赠送 → 住宿扣除

一句话验收标准：**在地的人能用来管理谁住哪、住多久，NT开始流转。**

## 二、身份系统

### 2.1 三种身份

```
身份层级（权限从高到低）：

  🧙 管理员 ───── 最高权限
   │  创建邀请码、确认打扫、分配NT池、编辑地图
   │
  🏕️ 在地成员 ─── 核心操作者
   │  通过NPC邀请码注册、操作物品、打扫、发委托、写记录
   │
  ☁️ 云村民 ───── 远端观察者
      自由注册、看地图、看时间线、留言、弹幕、打赏
```

### 2.2 注册流程

```
【在地成员注册】
  进入云村 → 点"注册" → 选头像 → 填名字 → 填密码
  → 选"在地成员" → 输入NPC邀请码 → 验证通过
  → ✅ 注册成功 → +100NT（注册赠送）
  → 进入云村主页

【云村民注册】
  进入云村 → 点"注册" → 选头像 → 填名字 → 填密码
  → 选"云村民" → 无需邀请码
  → ✅ 注册成功 → +0NT（云村民不参与NT经济，但可以接收打赏）
  → 进入云村主页

【管理员】
  预设账号，不可通过注册页面创建
  首次使用时系统检测 nt_users 为空 → 引导创建管理员账号
```

### 2.3 邀请码机制

```
管理员生成邀请码：
  [管理面板] → [邀请码管理] → [生成新码]

  生成时可设：
  - 数量：一次生成几个（默认1个）
  - 用途备注："新来的共建者""短期义工""访客"
  - 是否限制使用次数：一次性 / 多次（默认一次性）

  生成后显示：NT-XXXX-XXXX（可复制发送给在地的人）

  邀请码列表：
  ┌────────────────────────────────────────────┐
  │ 邀请码            │ 状态    │ 使用者  │ 生成时间 │
  ├────────────────────────────────────────────┤
  │ NT-A1B2-C3D4      │ 已使用   │ 王五    │ 7/3     │
  │ NT-E5F6-G7H8      │ 未使用   │ -       │ 7/5     │
  │ NT-I9J0-K1L2      │ 已过期   │ -       │ 6/20    │
  └────────────────────────────────────────────┘
```

## 三、住宿管理

### 3.1 核心模型

```
房间 = 宿舍里的一个可住人的区域
住宿记录 = 一个人 + 一个房间 + 一段时间 + NT费用

      ┌──────────────────┐
      │   宿舍 (dorm)     │
      │  ┌───┐ ┌───┐    │
      │  │101│ │102│    │
      │  │2人│ │2人│    │
      │  └───┘ └───┘    │
      │  ┌───┐ ┌───┐    │
      │  │103│ │104│    │
      │  │1人│ │2人│    │
      │  └───┘ └───┘    │
      └──────────────────┘

每个房间：名称、容量（床位数）、当前入住人数
每条记录：谁、哪个房间、入住日、退宿日、天数、费率、总额、已付/未付
```

│       │ 各种操作奖励                │
│       ▼                             │

│  │ 用户余额  │────→│ 住宿扣除  │   │

|------|:---:|--------|------|

## 五、NT 计算逻辑详解

### 5.1 住宿费计算

```
日费率：不同房间可以不同价
  标准间：70 NT/天
  大房间：100 NT/天
  小单间：50 NT/天

计算方式：
  days = checkOut - checkIn（按日期差，不满一天算一天）
  total = days × dailyRate

退宿时：
  if already paid (paidAmount >= total):
    不扣
  else:
    从用户余额扣 (total - paidAmount)
    如果余额不足 → 标记为"欠费"，允许退宿但记录欠款
```

### 5.2 欠费处理

```
如果退宿时余额不足：
  ┌────────────────────────────────┐
  │ ⚠️ 王五 NT余额不足              │
  │                                 │
  │ 应付: 210 NT                    │
  │ 余额: 30 NT                     │
  │ 欠费: 180 NT                    │
  │                                 │
  │ [确认退宿（记录欠费）] [取消]    │
  └────────────────────────────────┘

  确认后：
  → 扣除全部现有余额（30 NT）
  → 记录欠费 180 NT
  → 用户状态标记 "in_debt"
  → 下次该用户获得NT时自动抵扣欠费
```
