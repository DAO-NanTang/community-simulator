# 阶段一：身份 + 住宿 + NT 基础

> **目标**：人进来了，住下了，NT 开始流转。
> **状态**：📝 待审查
> **依赖**：无（一切的基础）

---

## 一、本阶段要解决什么问题

1. **谁是谁？** —— 在地的、云端的、管理的，身份要分清
2. **谁住哪？** —— 在地的人住哪个房间、住多久
3. **NT从哪来、到哪去？** —— 最基础的积分流转：注册赠送 → 住宿扣除

一句话验收标准：**在地的人能用来管理谁住哪、住多久，NT开始流转。**

## 二、身份系统

### 2.1 三种身份

```

   │  通过NPC邀请码注册、操作物品、打扫、发委托、写记录

  ☁️ 云村民 ───── 远端观察者
      自由注册、看地图、看时间线、留言、弹幕、打赏
```

### 2.2 注册流程

```
【在地成员注册】
  进入云村 → 点"注册" → 选头像 → 填名字 → 填密码

  进入云村 → 点"注册" → 选头像 → 填名字 → 填密码

  → ✅ 注册成功 → NT 余额 0
  → 进入云村主页

【管理员】
  预设账号，不可通过注册页面创建
  首次使用时系统检测 nt_users 为空 → 引导创建管理员账号
```

## 四、NT 经济基础：三层价值体系

> ⚠️ **本节是 NT 经济系统的核心设计，替代了旧版的「NT 余额 + NT 池」简单模型。**
> 完整设计见 [南塘NT经济系统_完整设计.md](../南塘NT经济系统_完整设计.md)。

### 4.0 平台定位

平台是一个**虚拟记账系统 + P2P 清算引擎**。不铸币、不持币、不做银行。

```
平台是什么：
  ✅ 虚拟记账——记录每个人在社区内的劳动和消费
  ✅ P2P 清算引擎——月度结算时算出谁该给谁多少 NT
  ✅ 贡献值计算器——衡量「你的 NT 是劳动挣的还是外部灌的」

平台不是什么：
  ✗ 不铸造 NT（铸币权在南塘 DAO）
  ✗ 不持有 NT 池（平台没有「社区钱包」）
  ✗ 不是银行——是清算所
```

### 4.1 三层价值

```
┌────────────────────────────────────────────────────────────┐
│                    南塘社区 · 三层价值                       │
│                                                            │
│  ① NT（链上代币，南塘 DAO 发行）                             │
│     平台不铸不毁。来源：DAO分发/劳动结算/他人转账。          │
│     只在月度结算时实际转账。                                  │
│                                                            │
│  ② 贡献值（劳动承诺，可转移）                                │
│     = 劳动所得 − 消费支出 + 接收的转移                       │
│     决定你能在社区做什么：买画、发委托、预定优先。            │
│     消费时 75% 转移给卖家，25% 磨损进入公共贡献池。           │
│                                                            │
│  ③ 经验值（资历，只涨不跌）                                   │
│     = 所有劳动所得贡献值的永久累计                            │
│     解锁权限（如陪审团资格需 ≥500 经验值）。                  │
└────────────────────────────────────────────────────────────┘
```

### 4.2 虚拟账本（不是钱包）

平台不持有任何 NT。只维护一个虚拟账本，记录每个人的应收/应付。月底统一 P2P 清算。

```js
// nt_virtual_ledger —— 平台维护的唯一「账本」
{
  month: "2026-07",
  entries: [
    {
      id: "entry-001",
      type: "cleaning",           // cleaning | meal_labor | meal_dine | task | sale | accommodation
      description: "大扫除·画室",
      parties: {
        "小红": { nt: +40, cv: +40, xp: +40 },
      }
    },
    {
      type: "meal_dine",
      description: "饭局·晚餐·用餐",
      parties: {
        "朝林": { nt: -15, cv: -15 },
        "王五": { nt: -15, cv: -15 }
      }
    }
  ],
  settled: false
}
```

**记账原则**：
- 劳动 → NT +N（应收）、贡献值 +N、经验值 +N
- 消费 → NT -N（应付）、贡献值 -N
- 交易 → NT 转移、贡献值转移（75% 给卖家 + 25% 公共磨损）
- 月底结算 → 净应收者从净应付者收款 → 链上转账

### 4.3 阶段一的 NT 规则

| 事件 | NT变动 | 贡献值 | 经验值 | 说明 |
|------|:---:|:---:|:---:|------|
| 新手任务完成 | +5~20/项 | +同等 | +同等 | 替代旧版「注册赠送 100NT」|
| 大扫除劳动 | +40 | +40 | +40 | 阶段三定义 |
| 饭局帮厨 | +15~40 | +同等 | +同等 | 阶段二/四定义 |
| 饭局用餐 | -15/餐 | -15 | — | 阶段二/四定义 |
| 住宿费 | -30/天 | 0 | — | RMB 30 或 NT 30，二选一 |
| 空间预定 | -2/小时 | 0 | — | 如画室 |

> ⚠️ **取消注册赠送 100NT**。新成员通过「新手任务」获得初始 NT（总计 75 NT），不施舍、不白给。详见 §4.6 新手任务。

### 4.4 个人三栏显示

```
个人资料卡中：

  ┌───────────────────┐
  │  👤 王五           │
  │  🏕️ 在地成员       │
  │                    │
  │  🪙 NT: 30        │  ← 链上余额（月底结算后更新）
  │  💠 贡献值: 120   │  ← 劳动净值（决定社区权限）
  │  ⭐ 经验值: 350   │  ← 劳动累计（决定资历等级）
  │                    │
```

### 4.5 月底 P2P 清算（管理员操作）

每月最后一天，系统汇总虚拟账本，生成 P2P 清算方案。管理员确认后，各人通过 web3 钱包执行转账。

详见 [南塘NT经济系统_完整设计.md §二](../南塘NT经济系统_完整设计.md)。

### 4.6 新手任务（替代注册赠送）

新成员注册后 NT 余额为 0。通过完成「新手任务」获得初始 NT——不施舍、不白给。

```
┌────────────────────────────────────────────┐
│         🌱 欢迎来到南塘 · 新手任务           │
│                                            │
│  □ 签署社区公约                     +5 NT   │
│  □ 认识 3 位在地成员               +15 NT   │
│  □ 社区空间打卡（5个空间）          +20 NT   │
│  □ 参加 1 次饭局                   +10 NT   │
│  □ 参加 1 次大扫除                 +15 NT   │
│  全部完成额外奖励：                 +10 NT   │
│  ─────────────────────────                 │
│  总计可获得：                      75 NT    │
└────────────────────────────────────────────┘
```

75 NT ≈ 2.5 天的住宿费。新成员有足够时间通过劳动和饭局继续挣 NT。

**贡献值抵押机制**：新手获得的 75 贡献值先冻结。每周大扫除后解冻 50%，两周后全部解冻。若两周内离开 → 冻结贡献值转入公共池。

### 4.7 虚拟账本数据结构

```js
// nt_virtual_ledger —— 平台维护的唯一「账本」，不是钱包
{
  month: "2026-07",
  entries: [
    {
      id: "entry-001",
      date: "2026-07-11",
      type: "cleaning",          // cleaning | meal_labor | meal_dine | task | sale | accommodation | booking
      description: "大扫除·画室",
      parties: {
        "小红": { nt: +40, cv: +40, xp: +40 }
      }
    }
  ],
  settled: false,
  settlementTxHash: null
}
```

> ⚠️ **NT 变动统一走虚拟账本**。不再使用 `nt_nt_pool` 结构。每个人的 NT 应收/应付、贡献值变动、经验值变动全部记录在 `nt_virtual_ledger.entries[].parties` 中。月底清算时将净 NT 头寸通过 P2P 转账在链上结算。贡献值和经验值即时生效（不等月底）。

## 五、住宿费计算（统一价）

```
日费率：RMB 30 或 NT 30/天（二选一，不按房间分级）

计算方式：
  days = checkOut - checkIn
  total = days × 30

选 RMB → 线下付现金/转账
选 NT  → 虚拟记账，月底 P2P 结算

退宿时：
  已付 → 不扣
  未付 → 标记为欠费，允许退宿，欠款记入虚拟账本（月底结算）
```

### 5.2 欠费处理

```
如果退宿时余额不足：
  ┌────────────────────────────────┐
  │ ⚠️ 王五 NT余额不足              │
  │                                 │
  │ 应付: 210 NT                    │
  │ 余额: 30 NT                     │
  │ 欠费: 180 NT                    │
  │                                 │
  │ [确认退宿（记录欠费）] [取消]    │
  └────────────────────────────────┘

  确认后：
  → 扣除全部现有余额（30 NT）
  → 记录欠费 180 NT
  → 用户状态标记 "in_debt"
  → 下次该用户获得NT时自动抵扣欠费
```

```
  ├── 💰 NT池管理：查看总池、注资、查看流水
  └── ⚙️ 系统设置：日费率、房间定义（阶段2完善）
```

## 七、交互细节补充

### 7.1 头像选择器

**简化决策**：初期只保留一种风格（`avataaars`，卡通人物风格），使用已有的 DiceBear API。6 种风格的选择器在 MVP 阶段过度设计——增加 API 请求次数和选择复杂度，但对用户体验的提升有限。后续阶段如需扩展，再加风格切换。

```
注册时头像选择：
  ┌────────────────────────────────┐
  │  选择你的头像                   │
  │                                 │
  │        (🤖 随机头像)            │
  │                                 │
  │  [🎲 随机换一个]               │
  │                                 │
  │  [← 返回] [✓ 确认头像]         │
  └────────────────────────────────┘
```

头像通过 DiceBear API（`avataaars` 风格）生成，seed 随机分配。用户可多次点击「随机换一个」直到满意。

### 7.2 会话管理

```
- 登录后设置 nt_session，包含 name 和 role
- 所有页面读取 nt_session 判断登录状态和权限
- 退出时清除 nt_session（不清除 nt_users）
- 支持"记住我"（把 session 持久化）
- 如果有 nt_session 且未过期 → 自动跳过登录页
```
