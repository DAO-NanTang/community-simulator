# 阶段五：在地记录 + NT 激励机制

> **目标**：拍照+一行字就能发记录，每条记录自动获得NT，形成完整的时间线。
> **状态**：📝 待审查
> **依赖**：阶段一（身份）、阶段二（空间）、阶段三（卫生）、阶段四（委托）

---

## 一、本阶段要解决什么问题

1. **日常值得被记录** —— 做饭、收菜、修缮、画画……每条记录都是数字孪生的血肉
2. **记录本身产生价值** —— 不是只有"完成任务"才获得NT，日常记录也有NT奖励
3. **时间线串起一切** —— 从物品进出到打扫到委托到日常，形成完整的故事

一句话验收标准：**在地的人随手拍照+一行字发记录，自动获得NT，按空间分类，云村民能看到完整的时间线（阶段六）。**

## 二、记录模型

### 2.1 什么是"在地记录"

```
在地记录 = 一张照片（可选）+ 一行文字 + 空间标签 + 心情标记

和"委托完成"的区别：
  - 委托完成：有人发布、有人接、有人确认，NT在任务中流转
  - 在地记录：没有发布人/接单人，就是日常分享，NT是系统奖励

和"物品操作"的区别：
  - 物品操作：对已有物品的状态变更（放入/吃掉/扔掉）
  - 在地记录：可以是没有对应物品的日常（"今天修好了篱笆""画了一幅画"）
  
但物品操作也可以自动生成一条在地记录（见下文）
```

### 2.2 记录类型

| 类型 | 说明 | 示例 | NT基础奖励 |
|------|------|------|:---:|
| 🍳 厨房记录 | 做饭相关 | "今天包了饺子，猪肉白菜馅" | +3 |
| 🌿 田间记录 | 种植/收获 | "B区番茄红了，收了8斤" | +3 |
| 🔧 修缮记录 | 修理/维护 | "修好了合作社2楼的水龙头" | +5 |
| 🎨 创作记录 | 画画/手工 | "画了一下午，这幅快完成了" | +3 |
| 🧹 打扫记录 | 打扫（非任务） | "顺手把走廊扫了" | +2 |
| 📋 日常记录 | 其他日常 | "今天南塘来了三位客人" | +2 |
| 📦 入库记录 | 物品入库 | "赶集买了鸡蛋30个放冰箱" | +2 |
| 🗑 出库记录 | 物品消耗 | "冰箱里的白菜终于吃完了" | +1 |

### 3.2 物品操作自动生成记录

阶段二的物品操作（放入/吃掉/扔掉）可以自动生成一条简要记录，不需要用户额外操作：

```
物品操作时，勾选"同时发布一条记录"：

  [✓ 同时发布一条记录]
    → 自动填好类型（入库/出库）
    → 自动填好空间
    → 自动关联物品
    → 自动生成文字："放入白菜×2到冰箱上层"
    → 用户可选改文字、加照片
    → 同样获得 +2NT（物品记录NT） + 可选的照片NT加成
```

### 3.3 外站联动（远期）

远期可以联动其他工具自动生成记录：

```
- 集字法练习完成 → 自动发一条"📝 今日集字完成"
- 其他应用 Webhook → 自动生成记录
```

## 五、NT 计算逻辑

### 5.1 记录NT规则

| 记录类型 | 基础NT | 有照片加成 | 说明 |
|----------|:---:|:---:|------|
| 🍳 厨房记录 | +3 | +1 | 做饭有照片更生动 |
| 🌿 田间记录 | +3 | +1 | 收菜拍照很自然 |
| 🔧 修缮记录 | +5 | +1 | 修东西有前后对比有意义 |
| 🎨 创作记录 | +3 | +2 | 创作有照片额外加分 |
| 🧹 打扫记录 | +2 | +1 | 随手打扫 |
| 📋 日常记录 | +2 | 0 | 日常分享 |
| 📦 入库记录 | +2 | 0 | 物品操作自带 |
| 🗑 出库记录 | +1 | 0 | 物品操作自带 |

### 5.2 NT每日上限

```
防止刷NT：
  每人每天通过"发记录"获得NT的上限：30 NT

  例如：
  - 发了3条厨房记录（3×3=9）+ 1条修缮记录（5）= 14NT（正常）
  - 发了15条日常记录 → 达到上限30NT后不再获得（防止灌水）

  物品操作和委托的NT不受此限制（那是真实劳动）
```

### 5.3 NT总览（阶段五累计）

到阶段五结束时，NT的来源和去向汇总：

| 来源 | 单次NT | 触发方式 | 阶段 |
|------|:---:|------|:---:|
| 注册赠送 | +100 | 一次性 | 1 |
| 标记物品 | +2 | 每件物品 | 2 |
| 消耗物品 | +1 | 每次消耗 | 2 |
| 打扫审核通过 | +10 | 每次打扫 | 3 |
| 完成委托 | 按委托额 | 每次确认 | 4 |
| 发在地记录 | +2~+7 | 每条（日上限30） | 5 |

| 去向 | 单次NT | 触发方式 | 阶段 |
|------|:---:|------|:---:|
| 住宿费 | -70/天 | 退宿结算 | 1 |
| 发布委托 | 按委托额 | 预扣 | 4 |
| 打赏他人 | 自由金额 | 手动 | 6 |

## 八、与其他阶段的衔接

### 本阶段是如何整合前面的

```
阶段五的"时间线"是前面所有阶段事件的统一展示层：

  阶段一事件：用户注册
  阶段二事件：物品放入/消耗/过期
  阶段三事件：打扫开始/提交/审核通过
  阶段四事件：委托发布/接取/完成/确认
  阶段五事件：日常记录/创作记录/修缮记录
  
  → 全部汇入同一条时间线
  → 按空间/按人/按时间筛选
  → 云村民（阶段六）看的就是这条时间线
```

## 九、待确认问题

1. **NT每日上限**：30 NT/天合适吗？相当于每天发10条基础记录。是限制"发记录"获得NT，还是所有来源NT都设上限？
2. **记录类型**：8种够用吗？是否需要"聚会记录""访客记录"等？
3. **心情标记**：目前8个emoji，是否需要自定义？
4. **照片存储**：如果照片用base64存localStorage，很快就会超5MB限制。是否考虑：缩略图+压缩、限制照片数量、或者后续迁移到IndexedDB/服务器？
5. **物品操作自动生成记录**：默认开启还是默认关闭？建议开启但允许手动关闭。
