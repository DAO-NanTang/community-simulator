## 二、审核显示位置（修订）

关键决策：**工作台加「营队审批」入口，但 NT 管理保留全局审核列表**。

| 审核类型 | 谁看 | 第一入口 | 第二入口 |
|---------|------|---------|---------|
| 营队审批（发布后） | 管理员 | **工作台**「🏛️ 营队审批」← 本次新增 | NT管理（保留） |
| 提交审核（完成后） | 发布者/共建者 | **工作台**「⏳ 待审核」← 已有 | NT管理（保留全局视图） |

> **为什么不删除 NT 管理的审核列表？** NT 管理的审核列表是全局视图——管理员/NT 角色可以看到所有人的待审核项。工作台是个人视图。删除全局视图意味着多管理员场景下无法快速了解"当前有多少待审核"。两个入口并存：个人工作台是日常入口，NT 管理是全局概览。

## 三、具体改动

### Step 1：管理员工作台新增「🏛️ 营队审批」

**改哪里**：`renderWorkspace()`，在现有「⏳ 待审核」分区上方

**做什么**：
1. 收集 `data.tasks` 中 `status === 'pending_approval'` 的任务
2. 仅对 `currentUser.role === 'admin'` 显示
3. 展示：任务名、scope 标签、NT、发布者、日期
4. 按钮：`[✅ 通过]` `[❌ 退回]`
5. 点击后调用现有的 `approveCampTask(idx, approve)`

**复用**：`approveCampTask()` 已存在（L5397），无需新写。只需在调用后追加 `renderWorkspace()` 刷新：

```
// approveCampTask 末尾原为 renderNTAdmin()
// 改为同时刷新工作台 + NT管理
localStorage.setItem('camp_data', JSON.stringify(data));
renderNTAdmin();
if (!document.getElementById('tab-workspace').classList.contains('hidden')) renderWorkspace();
```

### Step 2：工作台「⏳ 待审核」加类型标签

**改哪里**：`renderWorkspace()` 中现有待审核区（L3192 附近）

**做什么**：在每个待审核条目上加 scope 标签显示：

```
当前：  宏心 → 工笔画临摹 · NT100  [✅通过] [❌退回]
改后：  宏心 → 工笔画临摹 · 🏕️营地 · NT100  [✅通过] [❌退回]
```

标签复用现有 `renderQuestCard` 中的 `scopeTag` 逻辑（L4473）：
- `scope === 'personal'` → ✏️私人
- 否则 → 🏕️营地

### Step 3：提交弹窗显示要求清单 + 校验

**改哪里**：`showSubmitModal()`（L4751）+ `confirmSubmit()`

**做什么**：
1. 打开提交弹窗时，读取 `t.submit_requirements`，在弹窗中动态显示要求：

```
📋 发布者要求提交：
  ✅ 📝 文字说明（必填）
  ✅ 📷 至少 1 张照片（必填）
  ⬜ 📎 文件（可选）
```

2. 确认提交时校验：
   - 如果 requirements 要求 note → textarea 不能为空
   - 如果 requirements 要求 photo → images 至少 N 行
   - 不满足 → alert 提示，不提交

3. 每条要求旁加 `✅/⬜` 图标，提交者填写后自动变为 ✅

**复用**：现有 `submitTaskModal` HTML（L1242）和 `showSubmitModal`（L4751），在上面增量加校验逻辑，不重建。

### Step 4：NT 管理审核列表保留 + 标注

**改哪里**：`renderNTAdmin()` L4720-4723

**做什么**：保持现有审核列表不变，只加一行注释标注"全局视图"：

```js
// NT管理审核列表：全局视图，供管理员/NT 查看所有待审核项
// 个人日常审核入口在工作台「⏳ 待审核」分区
```
