## 一、发布表单 —— visibility 新增（不改 scope）

### 现状

- scope 下拉：🏕️营地 / ✏️私人（用户手动选，控制谁出 NT）
- visibility：不存在，`confirmPostTask()` 硬编码 `visibility: 'all'`（L4793）

### 改动

在现有 scope 下拉**旁边**新增 visibility 下拉：

```
范围：[🏕️ 营地 ▾]    谁能领：[🌐 所有人 ▾]    ← 新增
```

visibility 选项：

| 值 | 显示 | 说明 |
|----|------|------|
| `all` | 🌐 所有人 | 默认 |
| `adventurer` | ⚔️ 冒险者 | — |
| `builder` | 🧱 共建者 | 管理员也可领 |
| `npc` | 👥 在地伙伴 | — |
| `visitor` | 🏕️ 云村民 | — |
| `specific` | 🎯 指定某人 | 选后弹出选人器 |

说明：
- scope 和 visibility 是**两个独立维度**：scope = 谁出钱，visibility = 谁能领
- scope 保持手动选择，不从入口推断（用户可能需要跨场景发布）
- visibility 选 `specific` 时弹出**复用现有的 pickerDropdown**（L3279 `showReviewerPicker`），选人结果写入 `target_claimant` 字段（与 `reviewer` 字段区分）

### 实现位置

- HTML：在 `postTaskScope` 下拉旁边加 `<select id="postTaskVisibility">`
- JS `showPostTask()`：初始化 `postTaskVisibility.value = 'all'`
- JS `confirmPostTask()`：读取 `visibility` 和可选的 `target_claimant`

## 二、类型 —— 加自定义标签按钮

### 改动

在类型下拉框旁加 `＋` 按钮：

```
类型：[支线 ▾]  [＋]
```

交互：点击 `＋` → 内联输入框出现 → 输入新类型名 → 回车确认 → 写入 `data.custom_tags` → 下拉框自动追加新选项 → 选中新选项。

### 复用说明

此模式与 questboard 自定义标签（L2969-2974）完全一致：
- 同一个 `data.custom_tags` 数组
- 同一个 Enter 确认 + localStorage 持久化模式
- 区别：questboard 的标签用于**筛选任务**，发布表单的标签用于**标记任务类型**——两者共享 `custom_tags` 命名空间是**有意为之**：发布时创建的类型自动出现在筛选区，无需二次配置

## 五、撤回 —— 完整设计

### 需求

发布者可以撤回自己发布的任务。撤回需要审核（防止滥用），审核通过后 NT 退回。

### 入口

在发布者视角的任务卡片上（仅在任务尚未有人完成时显示）：

```
[🏳️ 撤回]
```

### 限制条件

- ✅ 可撤回：任务无人提交 / 有人提交但无人审核通过
- ❌ 不可撤回：已有 claimant 状态为 `completed`（审核通过）

### 撤回流程

```
发布者点 [🏳️撤回]
  → 弹确认框："撤回「任务名」？审核通过后 NT 将退回。"
  → 确认 → 任务状态变为 'withdrawing'（撤回中）
  → NT 管理员看到待审核撤回 → 审核
    → 通过：
      - 任务状态变为 'withdrawn'
      - 所有 claimant 状态变为 'withdrawn_by_poster'（未完成的领取者）
      - data.finance 新增 type='task_refund' 记录（退款审计）
      - NT 退回发布者账户
    → 拒绝：
      - 任务状态恢复为 'active'
      - 任务继续正常流转
```

### 新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `task.status` | `'withdrawing'` / `'withdrawn'` | 撤回状态（现有值不变：active/in_progress/已完成/已确认） |
| `claimant.status` | `'withdrawn_by_poster'` | 被发布者撤回（现有值不变：in_progress/submitted/completed/rejected/dropped） |
| `finance.type` | `'task_refund'` | NT 退款记录（现有 type 不变：task_reward/…） |

### 审计追踪

撤回 → 审核通过后：

```js
data.finance.push({
  type: 'task_refund',
  from: '系统',        // NT 池
  to: posterName,      // 退回发布者
  amount: task.points,
  task_name: task.name,
  date: new Date().toISOString().slice(0,10),
  scope: task.scope,
  confirmed_by: reviewerName,  // 审核撤回的管理员
  refund_reason: '发布者撤回'
});
```

> 不删除原有 finance 记录。退款是新增一条正向记录，保持完整审计链。
