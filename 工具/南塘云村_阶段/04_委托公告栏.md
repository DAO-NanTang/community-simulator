# 阶段四：委托公告栏

> **目标**：在地成员之间可以互相委托、接单、确认，NT在任务中流转。
> **状态**：📝 待审查
> **依赖**：阶段一（身份+NT）、阶段二（空间定义）

---

## 一、本阶段要解决什么问题

1. **"我需要帮忙"** —— 有人需要翻地、帮厨、修东西，可以发布委托
2. **"我能帮忙"** —— 有人看到委托，接单，完成后获得NT
3. **NT有了明确的交换场景** —— 委托 = 用NT换劳动/服务/物品

一句话验收标准：**在地成员能发委托、接委托、完成确认，NT自动流转。**

---

## 二、委托类型

| 类型 | 说明 | 示例 | NT建议范围 |
|------|------|------|:---:|
| 🛠️ 劳动委托 | 需要出力的事 | "帮我翻一下菜地B区" | 30-100 |
| 🎨 创作委托 | 画画、写字、设计 | "画一幅南塘风景，300NT" | 50-500 |
| 📦 物品委托 | 需要某样东西 | "谁有多余的番茄苗？" | 10-50 |
| 🍳 帮厨委托 | 做饭相关 | "今晚帮忙切菜备料" | 20-50 |
| 🔧 维修委托 | 修东西 | "合作社2楼水龙头漏水" | 20-100 |
| 📋 杂务委托 | 其他零碎事务 | "帮我去镇上取快递" | 10-80 |

## 三、委托全流程

### 3.1 状态流转

```
  [发布] ──→ [待接] ──→ [已接/进行中] ──→ [已完成] ──→ [已确认]
    │           │            │               │            │
    └── 可取消 ─┘            └── 可放弃 ────┘            │
                             └── 可退回 ────────────────→ 退回给接单人
                             
  超过截止日期无人接 → [已过期]
  发布人取消 → [已取消]
```

### 3.2 发布委托

```
[公告栏] → [+ 发布委托]

  ┌─────────────────────────────────────┐
  │  📋 发布委托                         │
  │                                      │
  │  标题: [________________________]   │
  │        "帮忙翻一下菜地B区"          │
  │                                      │
  │  类型: [劳动委托 ▼]                 │
  │                                      │
  │  详细描述:                           │
  │  [________________________]         │
  │  [________________________]         │
  │                                      │
  │  关联空间（可选）:                   │
  │  [菜地B区 ▼]                        │
  │                                      │
  │  NT奖励: [50] NT                    │
  │  （发布时从你的余额中预扣）          │
  │                                      │
  │  截止日期: [2026-07-10] \[可选]     │
  │                                      │
  │  需要拍照确认: ○ 是 ○ 否            │
  │                                      │
  │  你的余额: 250 NT                    │
  │  发布后余额: 200 NT (预扣50)         │
  │                                      │
  │  [✓ 发布委托]  [取消]               │
  └─────────────────────────────────────┘

发布时：
  → 从发布者NT余额预扣委托金额
  → 委托状态: "pending"（待接）
  → 记录事件: task_post
  → 在公告栏显示
```

### 3.3 接委托

```
[公告栏] → 点击委托 → [接取委托]

  ┌─────────────────────────────────────┐
  │  📋 委托详情                         │
  │                                      │
  │  🛠️ 劳动委托                        │
  │  帮忙翻一下菜地B区                   │
  │                                      │
  │  发布人: 阿楠                        │
  │  奖励: 50 NT                         │
  │  截止: 2026-07-10                    │
  │  关联空间: 菜地B区                   │
  │                                      │
  │  描述:                               │
  │  "B区的土需要翻一下，大概2小时工作量 │
  │   工具在工具间。"                    │
  │                                      │
  │  [✋ 我来接！]  [← 返回]            │
  └─────────────────────────────────────┘

接取后：
  → 委托状态: "accepted"（已接）
  → 接单人: 大飞
  → 记录事件: task_claim
  → 委托从"待接列表"移到"进行中列表"
  → 发布人和接单人都会收到通知
```

### 3.4 完成与确认

```
接单人完成后 → 点击 [标记完成]

  ┌─────────────────────────────────────┐
  │  ✅ 完成任务                         │
  │                                      │
  │  委托: 帮忙翻一下菜地B区             │
  │                                      │
  │  📷 完成照片（可选）:                │
  │  [拍照] 或 [上传]                   │
  │                                      │
  │  备注: [________________________]   │
  │        "翻好了，土很松"              │
  │                                      │
  │  [✓ 提交完成]  [取消]               │
  └─────────────────────────────────────┘

提交后 → 委托状态: "completed"
       → 通知发布人确认
       → 记录事件: task_done

发布人确认：

  ┌─────────────────────────────────────┐
  │  👀 确认完成                         │
  │                                      │
  │  委托: 帮忙翻一下菜地B区             │
  │  完成人: 大飞                        │
  │                                      │
  │  完成照片:                           │
  │  ┌─────────────────────┐            │
  │  │     (翻好的菜地)      │            │
  │  └─────────────────────┘            │
  │                                      │
  │  备注: "翻好了，土很松"              │
  │                                      │
  │  [✓ 确认 → 50NT发给大飞]            │
  │  [✗ 退回 → 需要修改]               │
  └─────────────────────────────────────┘

确认后：
  → NT从预扣中转给接单人
  → 委托状态: "confirmed"
  → 记录事件: task_confirm
  → 双方各自收到通知
```

### 3.5 取消与过期

```
【发布人取消】（委托还在"待接"状态时）
  → 预扣的NT退回发布人
  → 委托状态: "cancelled"
  → 记录事件: task_cancel

【接单人放弃】（已接但没完成）
  → 委托回到"待接"状态
  → 记录事件: task_unclaim

【委托过期】（超过截止日期无人接）
  → 预扣的NT退回发布人
  → 委托状态: "expired"
  → 记录事件: task_expire
```

## 五、NT 计算逻辑

### 5.1 委托NT流转

```
发布时：
  发布人余额 -= 委托金额（预扣到系统托管）
  接单人余额不变

确认完成时：
  系统托管 NT → 接单人余额
  如果退回：
    系统托管 NT → 退回发布人余额
```

### 5.2 委托相关NT事件

| 事件 | NT变动 | 发布人 | 接单人 | 系统托管 |
|------|:---:|:---:|:---:|:---:|
| 发布委托 | -委托金额 | 余额减少 | — | +委托金额 |
| 取消委托 | +委托金额 | 退还 | — | -委托金额 |
| 过期未接 | +委托金额 | 退还 | — | -委托金额 |
| 确认完成 | 0 | 不变 | +委托金额 | -委托金额 |
| 退回重做 | 0 | 不变 | 不变 | 不变 |

### 5.3 余额不足处理

```
如果发布时余额不足：
  ┌────────────────────────────────┐
  │ ⚠️ NT余额不足                   │
  │                                 │
  │ 需要: 50 NT                     │
  │ 余额: 30 NT                     │
  │ 缺少: 20 NT                     │
  │                                 │
  │ [取消] 或 [赚更多NT再来]        │
  └────────────────────────────────┘
```

## 九、待确认问题

1. **委托的NT金额**：是否完全由发布人自由设定？还是系统给个建议范围？
2. **"承诺类"委托**：比如"我承诺画一幅画，300NT"——这种是发布人承诺自己会做，还是悬赏让别人做？两种逻辑不同。
   - 悬赏型："我需要一幅画，谁能画？300NT"
   - 承诺型："我承诺本周画一幅画，画完发出来"（类似自我约束，不需要接单人）
3. **委托争议**：如果发布人不满意但不确认，NT一直卡在系统托管怎么办？是否需要"管理员仲裁"机制？
4. **委托模板**：是否需要预设一些常用委托模板？（"打扫XX""翻XX菜地""帮厨今晚"）
5. **和阶段三打扫的关系**：打扫任务由系统自动生成（阶段三），还是也放在委托公告栏？建议：系统生成的打扫和手动发布的委托分开显示，但都算任务。
