# 阶段二：空间地图 + 物品标记

> **目标**：地图活了，每件东西都有数字记录。在地的人开始填这个数字孪生。
> **状态**：📝 待审查
> **依赖**：阶段一（身份系统）、地图编辑器（画好网格地图）

---

## 一、本阶段要解决什么问题

1. **地图从静态变可交互** —— 点击空间能看到里面的东西
2. **每件实物有数字记录** —— 冰箱里放了什么、仓库有什么、谁放的、什么时候放的
3. **物品有生命周期** —— 从放入到消耗/过期，全程追踪

一句话验收标准：**在地的人打开云村，点进冰箱，能看到每层有什么、谁放的、还有几天过期。**

---

## 二、核心理念：地图 + 账本

```
┌──────────────────────┐     ┌──────────────────────┐
│    地图（空间索引）    │     │    账本（时间索引）    │
│                      │     │                      │
│  点厨房 → 看到冰箱    │     │  查"白菜"→ D1放入     │
│  点冰箱 → 看到每层    │     │         → D3被评论    │
│  点白菜 → 看到详情    │     │         → D6被吃掉    │
│                      │     │                      │
│  "现在有什么？"       │     │  "发生了什么？"       │
└──────────────────────┘     └──────────────────────┘
         ↕ 互相联通 ↕
  从地图点进去看到的是当前状态
  从账本查过来看到的是完整历史
```

## 三、空间语义层

### 3.1 概念

地图编辑器画了网格（哪个格子是路、哪个是墙、哪个是田），但网格本身不知道"这一片区域叫厨房"。**空间语义层**就是给网格区域赋予名字和意义。

```
地图编辑器输出 (nt_map_data)：
  grid[3][5] = { fill: "#8B4513", border: "wall" }
  grid[3][6] = { fill: "#8B4513", border: "wall" }
  ...（一堆格子）

云村叠加 (nt_spaces)：
  "coop-1F-kitchen" = {
    name: "厨房",
    gridArea: { x1:2, y1:3, x2:6, y2:8 },  ← 框选这片格子
    type: "kitchen"
  }
```

| 卧室 | bedroom | 床、衣柜、书桌 | ✓ | 0.5× |
| 卫生间 | bathroom | 淋浴区、马桶、洗手台 | ✓ | 2.0× |
| 走廊 | hallway | - | - | 1.0× |
| 储物间 | storage | 货架A、货架B | ✓ | 0.3× |
| 菜地 | field | 地块1、地块2... | ✓ | — |
| 画室 | studio | 画架区、材料区 | ✓ | 0.8× |
| 洗衣房 | laundry | 洗衣机、晾衣区 | ✓ | 0.5× |
| 户外 | outdoor | - | - | — |

### 4.3 消耗物品流程

```
在物品上点 [✓吃] / [🗑扔] / [📦移]：

  ┌─────────────────────────────────────┐
  │  🥬 白菜 ×2 — 消耗/处理             │
  │                                      │
  │  操作类型:                           │
  │    ○ ✓ 吃掉/用掉                    │
  │    ○ 🗑 扔掉（过期/变质）           │
  │    ○ 📦 移入堆肥（有机废物）        │
  │    ○ 📋 移入其他位置                │
  │                                      │
  │  数量: [2]（可部分消耗，如2棵吃1棵）│
  │                                      │
  │  谁操作: [下拉选在地成员 ▼]         │
  │                                      │
  │  📷 拍照（可选）: [拍照]            │
  │                                      │
  │  备注: [______________]             │
  │                                      │
  │  [✓ 确认 → +1NT]  [取消]            │
  └─────────────────────────────────────┘

确认后：
  → 物品数量减少（如果减到0则标记为"已消耗"）
  → 记录事件：evt type=item_consume / item_spoil / item_move
  → 操作者获得 +1NT（消耗记录奖励）
  → 如果拍了照 → 照片存入事件记录
```

### 4.4 过期提醒系统

```
系统每天自动检查（打开页面时触发）：

  对每件有保质期的物品：
    daysLeft = 到期日 - 今天
    
    if daysLeft <= 0:
      → 🔴 已过期！标记为"变质"
      → 在首页/物品列表显示红色警告
      → 发送提醒："冰箱里的白菜已过期2天，请处理"
    
    else if daysLeft <= 1:
      → 🟡 即将过期！显示黄色提醒
      → "冰箱里的白菜还有1天过期"
    
    else if daysLeft <= 3:
      → 🔵 注意：显示蓝色提示
      → 不弹窗，仅在物品旁显示剩余天数

提醒展示位置：
  1. 首页顶部横幅（醒目）
  2. 该物品所在空间入口处（点进去能看到）
  3. 可选的浏览器通知（需要授权）
```

## 七、NT 计算逻辑（本阶段新增）

### 7.1 物品相关NT

| 操作 | NT变动 | 获得者 | 说明 |
|------|:---:|--------|------|
| 添加物品 | +2 | 添加者 | 每件物品一条记录 |
| 消耗物品（吃掉/用掉） | +1 | 操作者 | 鼓励及时消耗记录 |
| 物品过期未处理 | 0 | — | 不扣NT，但显示警告 |
| 物品变质丢弃 | 0 | — | 已过期的丢弃不获得NT |

### 7.2 NT计算防刷机制

```
防止恶意刷NT：
  1. 同一人在同一位置、同一物品名、短时间内重复添加 → 系统提示"你刚刚放过类似物品，确认？"
  2. 添加和消耗必须是对应的人（不能自己添加自己消耗来刷NT——其实是允许的，因为这就是"自己放的东西自己用"的正常场景）
  3. 真正的防刷靠的是：每件物品都有照片为证（鼓励拍照），以及在地成员互相看到实物不一致时可以纠错
```
