## 四、共建者入口路径确认

**用户提问**：共建者进入广场后，和冒险者是一样的路径吗？

**确认**：是的，逻辑正确。

```
登录 → 南塘村口（所有人统一入口）
         │
         └─ 点击活动卡片"南塘艺术共创营·第四期"
              │
              ├─ 系统检查：该用户在该活动中是什么身份？
              │
              ├─ 有 builder 卡片 → 进共建者界面
              │     （builderTabs：砚仁、公告栏、猎人公会、NT管理…）
              │
              ├─ 是 adventurer 且交了学费 → 进冒险者界面
              │     （adventurerTabs：砚仁、公告栏、时间线、排行榜）
              │
              ├─ 是在地NPC → 进营队展示页（三视图）
              │
              └─ 是云村民 → 进营队展示页（三视图）+ 只能浏览
```

同一入口、同一张活动卡片、同一个点击动作。系统根据身份自动路由到不同界面。共建者和冒险者进的是同一个活动，但看到的管理界面不同。

## 六、最终结论

**方案可以执行，先把三项必须修改的调完。**

最大的风险点不是技术实现，而是执行顺序——先建村口再统一术语会导致返工。调换顺序后，整体逻辑链是：

```
术语统一 → 村口落地 → 重置干净 → 日期+提交完善 → UID → 任务大厅 → 后续
```

共建者的入口路径逻辑是正确的——同一入口、自动路由到不同界面。不需要给共建者单独建入口。
