- **目标**：让**所有参与者视角**（共建者/冒险者/南塘NPC/云村民）的内容在手机上好看好用；管理员"世界端"保持桌面专用

## 一、总原则（贯穿所有批次）

1. **手机覆盖层，不碰桌面**：所有改动放进 `@media (max-width:768px)`（必要时 480px），桌面（>768px）一行不受影响。
2. **只改布局，不改功能/内容**：同一份数据、同一套功能，只换手机上的排布方式。不删任何参与者功能。
3. **属两版共用核心，不进 NT-TEST 围栏**：手机 CSS 生产版也要，所以不加围栏；每批改完**重跑 `build_server.py`** 让 `dashboard.server.html` 同步。
4. **对付内联样式的手法**：用 `max-width:100%`（只封顶、不改小宽度）+ 必要处 `!important` 压过内联固定像素宽；实在压不住的（JS 写死的定位）用媒体查询覆盖或加 `isMobile()` 判断。
5. **逐屏迭代、真机验收**：桌面优先的老代码，响应式必须在真机上反复看，一批一验。

## 三、B0 · 全局护栏（可直接执行版）

### 3.1 做什么
在 `<style>` 末尾（现有媒体查询之后、`</style>` 之前）新增一段：

```css
/* ══════ 移动端全局护栏（B0）· 仅 ≤768px 生效，桌面不受影响 ══════ */
@media (max-width: 768px) {
  /* 1. 禁止页面横向溢出 */
  html, body { max-width: 100%; overflow-x: hidden; }
  main { max-width: 100%; padding: 12px 8px; box-sizing: border-box; }

  /* 2. 卡片/弹窗/登录卡流式 */
  .card, .modal, .login-card, .village-modal .vm-inner { max-width: 100% !important; box-sizing: border-box; }

  /* 3. 输入/下拉/文本域不溢出（全页含弹窗/侧栏内；max-width 只封顶，width:45px 的小框仍是 45px） */
  input:not([type=checkbox]):not([type=radio]),
  select, textarea { max-width: 100% !important; box-sizing: border-box; }

  /* 4. 表格改横向滚动，不撑破整页（全页含弹窗内） */
  table { display: block; overflow-x: auto; -webkit-overflow-scrolling: touch; max-width: 100%; }

  /* 5. 图片不溢出（全页） */
  img { max-width: 100%; height: auto; }

  /* 6. 固定侧栏最小钳制：宽度不超视口，避免溢出横拖（完整全宽抽屉改造留 B2） */
  .invite-sidebar, .hub-sub-panel { max-width: 100vw !important; box-sizing: border-box; }
}
```

### 3.2 为什么安全
- 整段只在 **≤768px** 生效，桌面完全不进。
- 输入/表格/图片规则**不限定 `<main>`**（弹窗、侧栏、登录卡都在 body 层，`<main>` 之外），全页覆盖才是真正的"全局护栏"；但因整段仍在 `@media (max-width:768px)` 内，桌面照样零影响。
- `max-width` **只封顶不强改**：小宽度的输入框、按钮保持原样，只阻止超出屏幕。
- 不动任何 HTML / JS，纯新增 CSS，可随时删掉回退。
- 放在现有媒体查询**之后**，同名规则以本段为准。

### 3.3 已知边界（B0 不解决、留给后续批）
- 表格 `display:block` 是"能横滑不撑破"的托底，**可读性**留 B3 用卡片式优化。
- 侧栏（邀请码/云村/内测）：B0 只做**宽度钳制**（不超视口，rule 6）；JS 坐标定位与完整**全宽抽屉**留 **B2**。
- 多列布局（登录双栏、时间线双列）B0 不拆，留 **B1/B3**。

### 3.4 B0 验收（手机真机）
- 打开首页/各标签，**正常内容页面不能再左右横拖**（无横向溢出）；固定侧栏（邀请码/云村/内测）因 JS 坐标定位偏移可能仍有横拖，**随 B2 全宽抽屉改造一并解决**（B0 只做宽度钳制，rule 6）。
- 输入框、卡片不超出屏幕宽。
- 宽表格变成"表格内部可横滑"，不再把整页撑宽。
- 桌面浏览器看：**布局与现在完全一致**（无回归）。

## 四、B1–B4 做法（审查用，执行时细化）

### B1 · 高频参与者屏
- **登录/注册**：`.login-split` 双栏（135px+240px）→ 手机上 `flex-direction:column` 竖排；头像列表改横向滑动或网格。
- **我的任务/提交/任务大厅**：任务卡 `quest-grid` 已单列；确保筛选栏、认领条、按钮 `flex-wrap`；提交弹窗全屏（现有 768 规则已做）。
- **余额/通知**：统计块、列表项竖向堆叠，数字不挤。

### B2 · 南塘云村
- `.invite-sidebar` / `.hub-sub-panel` / 内测侧栏 写死 280px 且 JS 按坐标定位 → 手机上用媒体查询 `!important` 覆盖成**全宽底部抽屉**（`left:0;right:0;width:100%;bottom:0;top:auto`），必要时加 `isMobile()` 让 JS 跳过坐标定位。
- **⚠️ 面板定位必须处理"切换/旋转"**：这些面板的坐标是**开面板那一刻**用 `getBoundingClientRect` 算出并写成内联 `left:xxxpx` 钉死的（如 `hub-sub-panel` 的 `left = 菜单.left - 336`）。若只在开面板时判断 `isMobile()`，用户在桌面开了面板再切到手机宽度/旋转屏幕，内联坐标会**残留错位**。因此二选一或都做：(a) 用媒体查询 `!important` 强制覆盖手机下的 `left/right/width`，让残留的 JS 坐标失效；(b) 加 `resize` 监听，切换/旋转时**重新定位**。**不能只在开面板时判断一次。**
- 云村各页内容卡片单列化。

### B3 · 时间线/日历/成员/数据
- 时间线 `.tl-day-box`（`flex:0 0 38%` 双列）→ 手机 100% 单列竖排。
- 财务/成员/数据宽表：在手机上改**卡片式**（每行变一张小卡，字段纵向排）——比横滑更好读；按表逐个做。
- 日历表：B0 的横滑托底，如需更好体验再单独设计手机视图。

### B4 · 世界端
- 管理员创世向导/设置：手机上该标签显示"⚠️ 此功能请在电脑上操作"，不强行适配（操作频繁、不适合手指）。
