### 2.4 逻辑规则

| 规则 | 说明 |
|------|------|
| 床位冲突检测 | 同一床位同一时间段只能一人 |
| 申请→确认 | 房客申请后状态 `pending`，摊主确认后变 `confirmed` |
| 自动归档 | 离开日期过后自动标记 `checked_out` |
| 费用 | 暂不涉及 NT 扣费（社区公益），后续可加 |
| 记录同步 | 入住记录显示在房客个人页「我的住宿」 |

### 6.3 交易流程

```
买家点「购买」
  → 确认对话框：「确定购买「工笔花鸟画」？将从你的账户扣除 200 NT」
  → 确认 → NT 从买家 → 卖家
  → 物品状态变为「已售」
  → 双方流水记录更新
  → 卖家收到通知（信箱）
  → 线下交付（由买卖双方自行协调）
```

## 八、NPC 摊主系统

### 8.1 摊主角色

| 摊位 | NPC | 角色 | 邀请码前缀 | 管理权限 |
|------|-----|------|-----------|---------|
| 🥟 早餐铺 | 王婶 | npc | NT-NBC-早餐 | 编辑菜单、查看订单 |
| 🥬 素食 | 李姨 | npc | NT-NBC-素食 | 编辑菜单、查看订单 |
| 🍵 茶馆 | 老李 | npc | NT-NBC-茶馆 | 发布/删除八卦 |
| 🏨 客栈 | 老陈 | npc | NT-NBC-客栈 | 确认/拒绝入住 |

### 8.2 摊主工作台

NPC 摊主的个人工作台比普通用户多一个管理面板：

```
NPC「老陈」的工作台：

  头像卡片（同普通用户）
  
  右侧 ┌─ 📋 任务（同普通用户）
       ├─ 💰 流水（同普通用户）
       ├─ 🎒 我的物品（同普通用户）
       ├─ 🏨 客栈管理（摊主专属）  ← 新增
       └─ ⚙️ 设置
```

### 8.3 摊主权限判断

```js
function getVendorType(user) {
  // 根据用户的世界码判断摊主类型
  var wid = user.world_id || '';
  if (wid.indexOf('早餐') !== -1) return 'breakfast';
  if (wid.indexOf('素食') !== -1) return 'canteen';
  if (wid.indexOf('茶馆') !== -1) return 'teahouse';
  if (wid.indexOf('客栈') !== -1) return 'inn';
  return null;
}
```

### 8.4 摊主邀请码

与现有邀请码系统一致：
- 由管理员在邀请码侧边栏生成
- 标签选择「南塘NBC」→ 输入摊位类型 → 生成 `NT-XXXX-早餐`
- NPC 使用该码升级角色后自动获得摊主权限

### 10.3 实现方式

四个子页的内容可以直接复用各个独立页面的渲染函数。例如：
- 客栈子页 → 调用 `renderInnPage()`
- 素食子页 → 调用 `renderCanteenPage()`
- 集市子页 → 调用 `renderBazaarPage()`

这样可以避免代码重复，同时允许从实景地图 Hub 或独立标签进入。

## 十二、技术备忘

- 所有交易记录统一走 `data.finance`，类型标记 `bazaar_trade` / `canteen_order` 等
- 摊位管理权限基于用户 `vendor_type` 字段，由邀请码激活时写入
- 个人物品状态机：`unlisted ↔ listed → sold`
- 客栈床位冲突检测：同床位 + 同时间段 = 拒绝
- 茶馆八卦 7 天自动清理（`expires_at` 判断）
- 订餐截止时间：当日 9:00（早餐）/ 10:30（午餐）
