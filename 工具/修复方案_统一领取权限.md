# 统一领取权限方案

> 2026-07-10 | 目标：所有"谁能领"的判断走同一个函数

---

## 〇、现状问题

任务领取权限检查散落在 3 层，各缺各的：

| 层 | 位置 | 检查了 | 漏了 |
|---|------|--------|------|
| 过滤（看不看得到） | `renderVillageQuestHall` L2482 | visibility ✅ | admin 豁免 ❌ |
| 过滤（看不看得到） | `renderQuestBoard` L4812 | visibility ✅ | admin 豁免 ❌ |
| 卡片（显示锁定） | `renderQuestCard` L4636 | scope ✅ | visibility 不看 ❌ |
| 领取（实际执行） | `claimTask` L4938 | 重复、满额 ✅ | visibility + scope 不看 ❌ |

**两层过滤的 visibility 判断是完全相同的重复代码**（L2482 和 L4812 各写一遍），且都包含 admin 豁免。

## 一、新增 `canUserClaim(user, task)` 统一函数

### 位置

`ROLE_CAPABILITIES` 区域（L1888 附近），和 `userCan()`、`userTabs()` 同级。

### 函数签名

```javascript
function canUserClaim(user, task) {
  // 返回 { ok: true } 或 { ok: false, reason: '中文原因' }
}
```

### 检查链（按顺序短路）

```
① 用户不存在 → { ok: false, reason: '请先登录' }
② 任务不存在 → { ok: false, reason: '任务不存在' }
③ 任务不是 active → { ok: false, reason: '任务不可领取' }
    （草稿、已撤回、已完结的任务不能领）
④ 名额已满 → { ok: false, reason: '名额已满' }
⑤ scope 权限 → 营地任务且用户不是 isMember → { ok: false, reason: '需加入营队' }
⑥ visibility 权限：
    - 'all' → 放行
    - 'specific' → 用户不是被指定的人 → { ok: false, reason: '仅指定领取人可领' }
    - 'adventurer'/'builder'/'npc'/'visitor' → 角色不匹配
      → { ok: false, reason: '仅「角色中文名」可领' }
```

**注意**：「已领取过」的检查**不放在这里**。它属于分组逻辑（myClaim → myTasks/doneTasks），不是领取权限问题。放在这里会导致已领取的任务被错误归入锁定组。

### 关键设计：看全部，领受限

| | 看（过滤层显示） | 领（canUserClaim） |
|---|:---:|:---:|
| 管理员 | 全部可见 | 必须匹配 visibility |
| 冒险者 | 全部可见 | 必须匹配 visibility |
| NPC | 全部可见 | 必须匹配 visibility |
| 云村民 | 全部可见 | 必须匹配 visibility |

**所有人看到所有任务，卡片上区分可领和锁定。点领取时统一过 `canUserClaim`，不是目标对象就拒绝。没有任何人豁免。**

### 实现方式

`canUserClaim` 函数不做角色豁免。过滤层两处 filter **不做 visibility 隐藏**（删除旧代码），visibility 判断只存在于 `canUserClaim` 内部。卡片层和领取层所有人都调 `canUserClaim` 来决定锁定状态和领取结果。

## 二、三个调用点改造

### 2.1 领取层（`claimTask`）

**现在**：检查登录 → 重复 → 满额 → 直接 push

**改为**：最前面加一行守卫，其余不变

```
canUserClaim(currentUser, t) → !ok → alert(reason) → return
```

### 2.2 卡片层（`renderQuestCard`）

**现在**：`locked = ctx==='village' && scope==='camp' && !isMember`

**改为**：`var cr = canUserClaim(user, t); var locked = !cr.ok;`

显示「🔒」时直接用 `cr.reason` 作为提示文字。

### 2.3 过滤层（两处）

**现在**：filter 函数末尾各写了一遍 visibility 检查（重复代码），visibility 不匹配就隐藏。

**改为**：**删除** filter 中的 visibility 检查。任务全部进入列表，不再按 visibility 隐藏。visibility 判断完全交给 `canUserClaim` 在卡片层和领取层处理。

```
filter 中：visibility 检查全部删除
→ 所有人看到所有任务
→ 卡片层用 canUserClaim 区分「可领取」和「🔒 锁定」
```
