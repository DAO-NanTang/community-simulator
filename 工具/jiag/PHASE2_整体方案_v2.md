## 二、身份定义

### 2.1 五种身份

| 身份 | 怎么注册 | 默认去哪 | 说明 |
|------|---------|---------|------|
| **管理员** | 系统预置（砚仁/1500） | 上次活跃世界 | 全局权限，创建世界 |
| **共建者** | 不可直接注册，由 NPC/冒险者 被管理员在 Step3 认领角色后转化 | 村口 或 被分配的世界 | 管自己的任务和材料 |
| **冒险者** | 注册时填世界码（或游客/NPC 输入世界码升级） | 码对应的世界 | 副本玩家 |
| **南塘NPC** | 直接注册（或游客输入 NPC 邀请码升级） | 村口 | 在地常驻伙伴 |
| **游客** | 直接注册，无需任何码 | 村口 | 临时来访 |

### 2.2 身份升级规则

**核心原则：有码就能升级。码是邀请，不是锁。**

```
                          ┌─→ 管理员在Step3认领 ──→ 🧙 共建者
                          │
 🏕️ 游客 ──→ 拿到NPC邀请码 ──→ 👥 南塘NPC ──┤
    │                    │                    │
    │                    │                    └─→ 拿到世界码 ──→ ⚔️ 冒险者
    │                    │
    └──→ 拿到世界码 ──→ ⚔️ 冒险者 ──→ 管理员在Step3认领 ──→ 🧙 共建者
```

**三种邀请码：**

| 码类型 | 谁发 | 作用 | 说明 |
|--------|------|------|------|
| **世界码**（如 1100） | 管理员创建世界时生成 | 进入该期的副本世界，成为冒险者 | 游客和 NPC 都可以用 |
| **NPC 邀请码** | 管理员/共建者在村口生成 | 游客升级为南塘NPC | 意味着「欢迎你留下来」 |
| **角色认领** | 管理员在 Step3 操作 | NPC/冒险者 升级为共建者 | 意味着「你来帮忙运营」 |

### 2.3 升级操作

用户的「个人页面」里有一个按钮：

```
┌─ 👤 王五（游客）────────────────────────────┐
│                                              │
│  到访次数：3次  ·  完成委托：5项  ·  NT：450  │
│                                              │
│  ┌─ 升级身份 ────────────────────────────┐   │
│  │                                        │   │
│  │  输入邀请码：[________]  [确认]         │   │
│  │                                        │   │
│  │  · NPC邀请码 → 升级为南塘NPC           │   │
│  │  · 世界码 → 加入该期，成为冒险者        │   │
│  └────────────────────────────────────────┘   │
└──────────────────────────────────────────────┘
```

NPC 同样有这个面板——NPC 可以输入世界码升级为冒险者。

### 2.4 身份不可降级

- 升级后保留旧身份的所有数据（NT、记录、痕迹）
- 共建者同时也是冒险者（如果他之前是冒险者转化来的）
- 一个人可以有多重身份的历史，但当前活跃身份是最高的那个

## 三、登录后的路由逻辑

```
登录成功
    │
    ├─ 管理员
    │     → 进入「上次活跃的世界」（有则进，无则进村口）
    │     → 顶部切换栏：[🏕️ 村口] [🔮 第四期] [🔮 第五期] [+ 创建世界]
    │
    ├─ 共建者
    │     → 进入「自己被分配的世界」（有则进，无则进村口）
    │     → 顶部切换栏：[🏕️ 村口] [🔮 第四期]
    │
    ├─ 冒险者
    │     → 进入「注册时的世界」
    │     → 顶部切换栏：[🏕️ 村口] [🔮 第四期]
    │
    ├─ 南塘NPC（未被认领）
    │     → 进入村口
    │     → 无切换栏（只有村口）
    │
    └─ 游客
          → 进入村口
          → 无切换栏（只有村口）
```

## 六、村口管理员

### 6.1 怎么产生

- 第一个注册的共建者（砚仁）自动成为村口管理员
- 管理员可以在村口设置里转让给另一个人

### 6.2 管理员独有的功能

| 功能 | 操作 |
|------|------|
| 编辑公告 | 修改村口顶部的公告文字 |
| 确认交付 | 委托完成后，管理员确认 → NT 从委托方划给接单人 |
| 登记住宿 | 录入「谁 + 入住日 + 离开日 + 日费率」 |
| 标记已付 | 住宿费已交给管理员 → 标记 |
| 转让管理权 | 指定另一个共建者为新管理员 |

### 6.3 管理员面板

管理员在村口界面看到一个「⚙️ 管理」按钮，点击打开：

```
┌─ ⚙️ 村口管理 ────────────────────────────────────────┐
```

## 八、逻辑校验：走一遍四个典型场景

### 场景1：砚仁（管理员）登录

```
打开页面 → 点「我有账号」→ 看到砚仁头像 → 点 → 输密码1500

→ 上次活跃世界 = 第四期
→ 进入第四期 → 看到创世终端
→ 顶部切换栏：[🏕️ 村口] [🔮 第四期] [🔮 第五期] [+ 创建]
→ 点「村口」→ 切到村口，看到实景地图
→ 回到第四期 → 点 Step3 → 把小红认领为共建者（宣传角色）
✓ 逻辑通
```

### 场景2：陈若雨（新冒险者）注册

```
打开页面 → 点「我是新来的」→ 选「冒险者」
→ 填名字：陈若雨 → 设密码 → 选头像 → 填世界码：1100
→ 注册成功 → 自动进入第四期
→ 看到：[我的委托] [冒险任务] [世界时间线] [排行榜]
→ 顶部切换栏：[🏕️ 村口] [🔮 第四期]
→ 点村口 → 看到南塘实景地图 → 接了一个打扫画室的委托
→ 回到第四期 → 继续做主线任务
✓ 逻辑通
```

### 场景3：小红的完整升级路径（游客 → NPC → 共建者）

```
第一天：小红注册为游客 → 进入村口
       → 在村口帮忙打扫、帮厨 → 攒了 NT 120
       → 砚仁觉得她靠谱，给了她 NPC 邀请码

第二天：小红在个人页面输入 NPC 邀请码
       → 身份升级：游客 → 南塘NPC
       → 界面不变（还是村口），但身份标记变了
       → 她现在是「在地伙伴」，不再是临时游客

第三天：砚仁在第四期的 Step3 认领小红 → 角色=宣传
       → 小红再次登录
       → 系统识别：身份已变为共建者
       → 顶部出现世界切换栏：[🏕️ 村口] [🔮 第四期]
       → 进入第四期 → 看到：[我的委托] [我的材料] [世界时间线]
       → 她有了宣传相关的任务和 NT 预算
✓ 逻辑通
```

### 场景4：王五（游客 → 冒险者）

```
王五注册为游客 → 在村口住了几天 → 看到第四期的海报
→ 朋友给了他世界码 1100
→ 在个人页面输入世界码
→ 身份升级：游客 → 冒险者（第四期）
→ 界面从村口切换到第四期副本
→ 看到：[我的委托] [冒险任务] [排行榜]
→ 开始做主线任务、赚 NT
→ 顶部切换栏：[🏕️ 村口] [🔮 第四期]
→ 随时可以回村口接日常委托
✓ 逻辑通
```

### 场景5：朝林（NPC → 冒险者 → 共建者）

```
朝林已注册为南塘NPC → 在村口活跃
→ 想参加第四期 → 输入世界码 1100
→ 身份升级：NPC → 冒险者（第四期）
→ 同时保留 NPC 身份的所有记录

→ 砚仁在 Step3 认领朝林 → 角色=后勤
→ 身份再升级：冒险者 → 共建者
→ 他现在是：第四期共建者 + 第四期冒险者 + 南塘NPC
→ 三种身份的 NT、记录、痕迹全部保留
✓ 逻辑通
```

## 九、实施步骤

### 阶段 2A：统一登录 + 路由

- 登录页简化为两个入口
- 注册页三种身份选择
