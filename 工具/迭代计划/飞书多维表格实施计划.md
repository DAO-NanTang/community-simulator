# 飞书多维表格实施计划

## 整体架构

```
┌──────────────────┐      ┌─────────────────┐      ┌──────────────────┐
│  dashboard.html  │◀────▶│ feishu_server.py │◀────▶│  飞书多维表格      │
│  (浏览器)         │      │ (localhost:8765) │      │  (7 张表, 云端)   │
└──────────────────┘      └─────────────────┘      └──────────────────┘
        │                                                    │
        │  临时缓存                                           │  持久存储
        ▼                                                    ▼
   localStorage                                       Feishu Bitable
   (页面刷新不丢)                                       (永久, 云端)
```

---

## 数据存储位置

| 存储层 | 位置 | 作用 | 生命周期 |
|--------|------|------|---------|
| **主数据源** | 飞书多维表格（7 张表） | 持久化存储，数据唯一真值 | 永久 |
| **运行时缓存** | 浏览器 localStorage | 页面快速读写，离线降级 | 浏览器关闭后保留 |
| **备份** | data.json（可选导出） | 离线存档 | 手动导出 |

---

## 数据同步方式

### 页面启动时

```
dashboard.html 打开
    │
    ▼
fetch('localhost:8765/api/load-all')
    │
    ├── 成功 → 数据填入页面 + 写入 localStorage（缓存）
    │
    └── 失败（服务器未启动）→ 从 localStorage 加载上次缓存
```

### 用户编辑时（自动保存）

```
用户编辑数据（新增任务、改状态、改成员…）
    │
    ▼
updateAll() 函数（现有逻辑）
    ├── 更新页面 DOM
    ├── localStorage.setItem('camp_data', ...)  ← 现有，保留
    └── debouncedSaveToFeishu()                 ← 新增，自动
            │
            ▼
        延迟 2 秒（防抖：连续编辑时只保存最后一次）
            │
            ▼
        POST 'localhost:8765/api/save'  →  写入飞书
```

**不需要「保存到飞书」按钮**，不需要「从飞书加载」按钮。编辑即保存，打开即加载。

## 实施步骤

### 第一步：飞书建表

**目标**：在飞书 Base 中创建 7 张空表，设置所有字段。

**新建文件**：`工具/setup_feishu_tables.py`

**脚本行为**：
1. 用 App ID/Secret 获取 tenant_access_token
2. 在 Base `你的飞书多维表格 Base Token` 中逐一创建 7 张表
3. 为每张表创建字段（文本/单选/数字/日期/复选框/多行文本）
4. 为单选字段设置选项列表
5. 将每张表的 table_id、每个字段的 field_id 写入 `feishu_config.json`

**产出**：
- 飞书 Base 中 7 张有字段的空表
- `工具/feishu_config.json`（表 ID + 字段 ID 映射）

**依赖**：`pip install requests`

---

### 第二步：API 封装 + 本地代理

**目标**：构建数据读写的桥梁。

**新建文件**：

#### `工具/feishu_api.py` — 飞书 API 封装

```
FeishuClient 类
├── __init__(config_path)
│   加载 feishu_config.json
│   维护 token 缓存（自动刷新，2h 过期）
│
├── _get_token()
│   POST /auth/v3/tenant_access_token/internal
│   返回 access_token，缓存到过期前 5 分钟
│
├── _request(method, url, json)
│   统一 HTTP 请求
│   自动注入 Authorization header
│   429（限流）→ 等待重试
│   401（token 过期）→ 刷新 token 重试
│
├── get_records(table_name) → list[dict]
│   从飞书拉取指定表全部记录
│   分页自动处理（page_token 循环）
│   字段名转换：飞书 field_name → 项目 field_name（用 config 中的映射）
│
├── upsert_records(table_name, records) → dict
│   按「名称」字段匹配（任务名称/姓名/预算项目/事由…）
│   存在 → 更新（update）
│   不存在 → 新建（create）
│   批量处理，自动拆分 500 条一批
│
└── 字段映射逻辑
    config 中存 { 项目字段名: 飞书 field_id }
    读写时自动互转
```

#### `工具/feishu_server.py` — 本地 HTTP 代理

```
启动：python feishu_server.py
监听：http://localhost:8765

端点：
  GET  /api/load-all
    → 依次读取 7 张表
    → 组装为 Dashboard 的 data 对象格式
    → 返回 JSON

  POST /api/save
    → 接收 Dashboard 的完整 data 对象
    → 拆分为 7 张表的数据
    → 逐表调用 upsert_records()
    → 返回 { success: true, stats: { 用户表: {created:N, updated:M}, ... } }

  GET  /api/health
    → 返回 { status: "ok", token_valid: true }
    → Dashboard 用于检测服务器是否可用
```

**技术要点**：
- 用 Python 标准库 `http.server` 或轻量框架（Flask/FastAPI）
- 必须设置 CORS headers：`Access-Control-Allow-Origin: *`
- 飞书 API 返回的字段格式是 `{field_id: value}`，需要转换为 `{字段名: value}`

**依赖**：`pip install requests flask`（如果用 Flask）

#### `工具/feishu_config.json` — 配置（第一步生成）

```json
{
  "app_id": "cli_xxxxxxxxxxxxx",
  "app_secret": "你的飞书应用 Secret",
  "app_token": "你的飞书多维表格 Base Token",
  "tables": {
    "用户表": {
      "table_id": "tbl...",
      "key_field": "姓名",
      "fields": {
        "姓名": "fld...",
        "用户类型": "fld...",
        ...
      }
    },
    ...
  }
}
```

> 此文件含 App Secret，加入 `.gitignore`。

### 第三步：上传现有数据 + 修改 Dashboard

#### `工具/upload_data.py` — 一次性的数据上传

**行为**：
1. 读取 `data.json`
2. 提取 dashboard.html 中的 `PRESET_TEMPLATES`
3. 通过 `FeishuClient` 写入飞书：
   - tasks[] → 任务表（`记录类型=第三期`）
   - members{} → 用户表
   - decisions[] → 决策表
   - budget{} → 预算表（拆为逐行，`项目分类=初始参数`）
   - finance_cny.transactions[] → 流水表
   - schedule{} → 日程表（`记录类型=第三期`）
   - PRESET_TEMPLATES 中的日程模板 → 日程表（`记录类型=模板`）
   - PRESET_TEMPLATES 中的任务模板 → 任务表（`记录类型=模板`）
4. 打印统计：每张表创建了多少条记录

#### `dashboard.html` — 改动

**改动范围**：仅修改数据加载和保存的入口函数，不改动任何 UI 渲染逻辑。

**改动 1**：页面启动时自动加载飞书数据

```javascript
// 现有代码（页面加载时）：
const saved = localStorage.getItem('camp_data');
if (saved) { data = JSON.parse(saved); updateAll(); }

// 改为：
async function initData() {
  try {
    const resp = await fetch('http://localhost:8765/api/load-all');
    if (resp.ok) {
      data = await resp.json();
      localStorage.setItem('camp_data', JSON.stringify(data));
      updateAll();
      return;
    }
  } catch(e) { console.log('飞书不可用，使用本地缓存'); }

  // fallback：飞书不可用时从 localStorage 加载
  const saved = localStorage.getItem('camp_data');
  if (saved) { data = JSON.parse(saved); updateAll(); }
}
initData();
```

**改动 2**：编辑时自动保存到飞书

```javascript
// 新增防抖保存函数
let saveTimer = null;
function autoSaveToFeishu() {
  clearTimeout(saveTimer);
  saveTimer = setTimeout(async () => {
    try {
      await fetch('http://localhost:8765/api/save', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(data)
      });
    } catch(e) { console.log('飞书保存失败', e); }
  }, 2000);  // 2秒防抖
}

// 修改 updateAll() — 函数末尾加一行：
function updateAll() {
  // ... 现有所有逻辑不变 ...
  localStorage.setItem('camp_data', JSON.stringify(data));
  autoSaveToFeishu();  // ← 仅加这一行
}
```

**改动总结**：
- 新增 ~40 行 JS（initData + autoSaveToFeishu）
- 修改 ~2 行（页面启动逻辑 + updateAll 尾部）
- **零 UI 改动**（无新按钮，无新弹窗）
- **现有功能 100% 保留**（localStorage、JSON 导入导出、离线模式）

## 实施后：完整数据流

```
用户打开 dashboard.html
        │
        ▼
   fetch /api/load-all ──→ feishu_server ──→ 飞书 API ──→ 7 张表
        │                                              （云端）
        ▼
   数据填入页面
        │
   用户编辑（改任务/成员/流水…）
        │
        ▼
   updateAll()
     ├── 更新 DOM
     ├── localStorage 缓存
     └── 2秒防抖
           │
           ▼
     POST /api/save ──→ feishu_server ──→ 飞书 API ──→ 7 张表
                                                   （自动写入）
```

## 用户操作流程

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | `python feishu_server.py` | 启动本地代理（保持运行） |
| 2 | 打开 `dashboard.html` | 浏览器自动从飞书加载数据 |
| 3 | 正常使用（编辑任务/成员/流水…） | 每次修改 2 秒后自动保存到飞书 |
| 4 | 关闭页面 | localStorage 保留缓存，飞书数据已同步 |
| 5 | 下次打开 | 自动从飞书加载最新数据（包括其他人通过 API 做的修改） |
