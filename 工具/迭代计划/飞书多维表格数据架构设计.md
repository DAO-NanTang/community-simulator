# 飞书多维表格数据架构设计（v6 — 完整版）

## 设计原则

1. **用户表只存身份信息** — NT 收入从流水汇总
2. **模板与实例共表** — 用「记录类型」字段区分
3. **财务精简** — 预算 + 流水 2 张表，金额 + 货币类型合并
4. **每个字段有明确来源** — 可追溯到 data.json 或 Dashboard 源码
5. **字段尽量少、利用率尽量高** — 审查移除低利用率字段
6. **参数与结果分离** — 初始参数独立成表，预算结果另存；改参数自动重算预算，互不污染
7. **定价与执行分离** — NT 任务定价规则独立成表，与任务实例表分开管理
8. **编辑即保存** — 用户在 Dashboard 修改数据后 2 秒自动同步到飞书表格，浏览器本地存储作为离线兜底

## 表 4：NT 任务设计表

> 存的是 NT 任务的**定价规则**，不是任务执行记录。改规则 → NT 总池自动重算 → 结果写进预算表。

| # | 字段名 | 飞书类型 | 必填 | 示例 | 来源 |
|---|--------|----------|------|------|------|
| 1 | 所属营期 | 文本 | ✅ | `第三期` | 关联营地表 |
| 2 | 任务系列 | 单选 | ✅ | `主线` | 主线 / 支线 / 共创 / 隐藏 |
| 3 | 任务名称 | 文本 | ✅ | `M1 开营破冰` | Dashboard `budget.nt_task_design` |
| 4 | 任务分类 | 单选 | | `课程` | 与任务表分类一致 |
| 5 | 每人 NT | 数字 | ✅ | `5` | Dashboard `budget.nt_task_design` |
| 6 | 任务说明 | 多行文本 | | `营期第一天破冰活动` | NT 系统的任务描述 |
| 7 | 状态 | 单选 | | `启用` | 启用 / 备选 / 停用 |
| 8 | 备注 | 多行文本 | | | |

**NT 总池计算公式**：

> 共创人任务 NT 池 = Σ（每个任务的 每人NT × 预计共创人数）
> 加上 食宿 NT + 回购 NT + 赏金池 NT + 工作人员 NT
> = 当期 NT 总发行量

**与任务表的区别**：

| | NT 任务设计表 | 任务表 |
|---|---|---|
| 存什么 | 定价规则：「主线任务，每人 5 NT」 | 执行记录：「M1 开营破冰，砚仁，5 NT，已确认」 |
| 谁在用 | Wizard Step 1 做 NT 预算时用 | 营期每天 NT 结算用 |
| 改动影响 | 改规则 → 任务表中对应任务的积分联动更新 | 改实例 → 只影响那一条任务 |
| 每期条数 | 几十条（一个任务一条规则） | 几十条（每个任务实例一条） |
| 跨期复用 | 是（定价规则复制到下一期） | 否（每期重新创建任务实例） |

**审查**：8 字段。定价规则独立后：

- 修改定价，关联任务表的 NT 积分字段联动更新
- NT 总池自动重算，避免超发或不够发
- 定价规则可跨期复用，不需要每期重新手填

**数据分层说明**：

- **配置层**（初始参数表 + NT 任务设计表）：每期营地开始时设定，改参数自动重算预算。跨期可复制复用。
- **执行层**（其余 7 张表）：营期运行过程中产生的实际数据。任务完成记录、财务流水、成员信息等。

| 任务物料的结构化存储 | 多行文本，无法做预算对比 | 改为关联物料清单子表 |
| 个人材料表（预算 vs 实际对比） | 无法实现 | 通过物料子表 + 流水表关联查询实现 |

## 数据流说明

### 配置层 → 执行层的自动联动

```
初始参数表（改学费 5990→6990）
    ↓ 飞书公式自动重算
预算表（学费收入行：59900→69900）

NT 任务设计表（改主线任务 5→8 NT/人）
    ↓ 飞书公式自动重算
预算表（NT 总池行：联动更新）
    ↓ 关联更新
任务表（对应任务的 NT积分字段：5→8）
```

### 编辑即保存（Dashboard ↔ 飞书表格）

```
用户在 Dashboard 修改数据
    → 浏览器内存立即更新（界面即时响应）
    → 2 秒防抖后自动 POST 到飞书表格
    → 飞书表格存储成功
    → 其他打开 Dashboard 的人自动拉取最新数据

如果 2 秒内断网或关闭浏览器：
    → 浏览器本地存储（localStorage）保留副本
    → 下次打开页面时恢复并补同步到飞书
```
