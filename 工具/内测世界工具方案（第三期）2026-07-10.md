## 一、目标

给管理员一个"内测世界"工具：**让全系统认为现在是任意一天，且所有人、所有页面统一到这一天**，可冻结、跳转、步进推进时间，用来验证"提交方式 / 提交时间 / 逾期 / 催单 / 里程碑"等时间相关逻辑是否有 bug。要求：

1. **管理员专属**，入口放设置齿轮菜单，做成二级菜单。
2. **不污染真实数据**——测试跑在沙盒上，退出即还原。
3. **复用**第一期 `Clock` 接缝与现有 UI 范式，改动集中、低风险。
4. **低频工具**（预计仅一两次使用）——一切从简，不为它建重型机制。

## 四、总体架构（三块）

### 4.1 时间注入（唯一改 Clock 处，冻结模型）
`Clock` 内加**一个变量** `_frozenMs`：为空返回真实时间（等于现状），有值则全系统时间冻结在该时刻。`_wall()` 保持真实（ID 安全）。

- **跳到某天**：设 `_frozenMs` = 该日某时刻。
- **步进**：`_frozenMs` ± 增量（+1 小时 / +1 天 / +1 周 / −1 天）。
- **恢复真实**：清空 `_frozenMs`。
- **持久化**：`_frozenMs` 存 localStorage（`nt_virtual_ms`）；加载时自恢复，故刷新、切换模拟用户都保持同一虚拟日。

对外新增：`_setFrozen(ms)`、`_step(deltaMs)`、`_reset()`、`_state()`。

### 4.2 沙盒数据（快照+还原，复用现有存取）
- **进入内测**（幂等）：若当前不在内测，才把真实 `camp_data` 备份到 `camp_data__real_backup`，置标志 `nt_test_mode='1'`。此后写入照常落 `camp_data` → 成沙盒数据。
- **退出内测**：备份还原 `camp_data`，删备份与标志，`Clock._reset()`，刷新。
- **常驻横幅兜底**（替代独立崩溃提示条）：只要 `nt_test_mode='1'`（含刷新/崩溃后重开），顶部横幅常驻，横幅上"退出内测并还原"按钮**不分角色可见可用**——即安全出口。
- **范围**：首版仅隔离 `camp_data`（任务/提交/订单/预订/拍卖等时间数据都在此），前提是用现有账号登录提交。其余次要键（inbox 已读态、`LAST_PAGES` 等）**有意不隔离**，影响可忽略。若日后要在内测中新建测试账号模拟多人，再纳入 `NT_USERS_KEY`。

### 4.3 界面刷新
每次改时间后调用现有渲染分发（如 `switchTab(当前tab)`）重渲染当前视图。**无定时器**，只在点击时刷新一次。

### 5.2 面板（新增 devtoolsSidebar，复用现有 CSS 类）
右侧滑出栏，**复用 `.invite-sidebar` / `.invite-backdrop` / `.is-header` / `.is-body` 类，只加新 id、不新增 CSS**。含：
- **状态区**：当前虚拟时间、是否处于内测/沙盒。
- **进入 / 退出内测世界**。
- **跳到某天**：日期+时间选择器 →「跳转并冻结」。
- **步进**：−1 天 / +1 小时 / +1 天 / +1 周。
- **恢复真实时间**。

### 5.3 全局横幅（复用 previewBanner，收编退出出口）
处于内测时顶部常驻：`🧪 内测世界 · 虚拟时间 YYYY-MM-DD HH:mm · 沙盒数据 [退出并还原]`。仿 `previewBanner` 创建与 display 开关；**退出按钮不分角色**，是崩溃残留后的唯一安全出口。

## 七、影响面与风险
- **对现有功能**：真实模式下 `_now()` 等于现状，非内测/生产零感知。
- **主要风险**：内测期数据落沙盒——由快照+还原+幂等+横幅退出出口兜底。
- **回滚**：每批独立，单批出错只回退该批；整体可一键关闭入口。
- **已消除的债**：无实时定时器；退出出口人人可见，不会困死。
