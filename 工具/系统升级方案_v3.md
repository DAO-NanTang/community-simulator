## 一、会计系统升级

### 1.1 当前问题

```
data.finance = [...]   ← 所有 NT 流水
calcNtByScope(name)    ← 每次全量遍历 O(n)

风险：
  ❌ 无交易ID → 无法去重
  ❌ 多个 push 点各写各的 → 无统一入口
  ❌ 无 ref_id → 无法回溯业务
  ❌ 删除任务用 removeFromFinance → 可能遗漏
  ❌ 类型字符串硬编码 → 易拼错
```

### 1.2 升级方案：五条铁律

#### 铁律 1：统一记账入口 `recordTransaction()`

```js
function recordTransaction(opts) {
  // opts: { type, from, to, amount, scope, ref_id?, task_name?, item_name?, note? }
  if (!opts.amount || opts.amount <= 0) return null;
  
  var tx = {
    id: 'tx_' + Date.now().toString(36) + Math.random().toString(36).slice(2, 6),
    type: opts.type,
    from: opts.from || '系统',
    to: opts.to,
    amount: opts.amount,
    scope: opts.scope || 'camp',
    date: new Date().toISOString().slice(0, 10),
    created_at: new Date().toISOString(),
    ref_id: opts.ref_id || '',
    task_name: opts.task_name || '',
    item_name: opts.item_name || '',
    confirmed_by: opts.confirmed_by || '',
    note: opts.note || '',
    tx_hash: ''              // Web3 预留
  };
  
  if (!data.finance) data.finance = [];
  data.finance.push(tx);
  localStorage.setItem('camp_data', JSON.stringify(data));
  return tx;
}
```

**改动范围**：现有 2 处 `data.finance.push()` 改为调用此函数。

#### 铁律 2：交易类型枚举 `TX_TYPES`

```js
var TX_TYPES = {
  TASK_REWARD:      'task_reward',
  TASK_POST:        'task_post',
  CANTEEN_PREORDER: 'canteen_preorder',
  CANTEEN_WALKIN:   'canteen_walkin',
  BAZAAR_TRADE:     'bazaar_trade',
  TEAHOUSE_POST:    'teahouse_post',
  GAME_FEE:         'game_fee',
  ADMIN_ADJUST:     'admin_adjust',
  SEASON_BONUS:     'season_bonus'
};
```

#### 铁律 3：业务与记账原子绑定

状态变更 + 记账 + 持久化 = 同一步完成，不可拆分。

```js
// ✅ 正确示例
function purchaseBazaarItem(itemId, buyer) {
  var item = findItem(itemId);
  item.status = 'sold';           // 状态
  recordTransaction({...});       // 记账
  save();                         // 持久化
}

// ❌ 禁止：先改状态，再记账（中间崩溃会丢账）
```

#### 铁律 4：余额缓存

```js
var _balanceCache = {};
function getBalance(name) { /* 缓存 + O(n)兜底 */ }
function invalidateBalance(name) { delete _balanceCache[name]; }
```

#### 铁律 5：对账报表

NT 管理新增「📊 对账」面板：
- 营地池总额 / 已发放 / 待审核 / 待审批 / 剩余
- 分人统计：收入 / 支出 / 净余额
- 异常检测：出入不平衡自动标记
- 「🔄 全量校验」按钮

### 1.3 Web3 预留

```
当前阶段：Dashboard = 记账本，计算正确即可
后续阶段：recordTransaction 中加 tx_hash 字段
          → 钱包支付后回填
          → verifyTx(tx_hash) 核对链上数据
          → 不一致标记 disputed
```

## 六、交易类型与记账点

```
recordTransaction()  ← 唯一入口
  │
  ├─ 任务系统 ─────────────────
  │   TASK_REWARD   (审核通过)  from=系统 to=完成者 scope=camp
  │   TASK_POST     (发布委托)  from=发布者 to=任务池 scope=personal
  │
  ├─ 素社食堂 ─────────────────
  │   CANTEEN_PREORDER (预定)  from=食客 to=高琳姐 scope=personal
  │   CANTEEN_WALKIN   (现场)  from=食客 to=高琳姐 scope=personal
  │
  ├─ 集市 ────────────────────
  │   BAZAAR_TRADE (购买)      from=买家 to=卖家 scope=personal
  │
  ├─ 茶馆 ────────────────────
  │   TEAHOUSE_POST (发八卦)   from=作者 to=系统 scope=personal
  │
  ├─ 桌游 ────────────────────
  │   GAME_FEE (报名费)        from=玩家 to=发起人 scope=personal
  │
  └─ 管理 ────────────────────
      ADMIN_ADJUST (手动调账)
      SEASON_BONUS (结营奖励)
```
