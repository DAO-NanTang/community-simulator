## 一、会计系统升级

### 1.1 当前状态

```
data.finance = [...]   ← 所有 NT 流水，当前约 2 个 push 点
calcNtByScope(name)    ← 计算余额，每次全量遍历 O(n)
```

**风险**：无交易 ID、无统一入口、无业务关联字段、无类型枚举。

### 1.2 recordTransaction() — 统一记账入口

```js
function recordTransaction(opts) {
  // opts: { type, from, to, amount, scope, ref_id?, task_name?, item_name?, note? }
  if (!opts.amount || opts.amount <= 0) return null;

  var tx = {
    id: 'tx_' + Date.now().toString(36) + '_' + Math.random().toString(36).slice(2, 10),
    type: opts.type,
    from: opts.from || '系统',
    to: opts.to,
    amount: opts.amount,
    scope: opts.scope || 'camp',
    date: new Date().toISOString().slice(0, 10),
    created_at: new Date().toISOString(),
    ref_id: opts.ref_id || '',
    task_name: opts.task_name || '',
    item_name: opts.item_name || '',
    confirmed_by: opts.confirmed_by || '',
    note: opts.note || '',
    tx_hash: ''              // Web3 预留
  };

  if (!data.finance) data.finance = [];
  data.finance.push(tx);
  localStorage.setItem('camp_data', JSON.stringify(data));
  return tx;
}
```

**tx_id 格式**：`tx_` + 毫秒时间戳(base36) + `_` + 8位随机(base36)，概率碰撞可忽略。

### 1.3 TX_TYPES — 类型枚举

```js
var TX_TYPES = {
  TASK_REWARD:      'task_reward',
  TASK_POST:        'task_post',
  CANTEEN_PREORDER: 'canteen_preorder',
  CANTEEN_WALKIN:   'canteen_walkin',
  BAZAAR_TRADE:     'bazaar_trade',
  TEAHOUSE_POST:    'teahouse_post',
  GAME_FEE:         'game_fee',
  ADMIN_ADJUST:     'admin_adjust',
  SEASON_BONUS:     'season_bonus'
};
```

### 1.4 旧数据迁移

现有 `data.finance` 中的记录缺少 `id`/`created_at` 字段。在 `recordTransaction` 定义后立即执行一次性迁移：

```js
function migrateFinanceRecords() {
  if (!data.finance) return;
  var migrated = 0;
  data.finance.forEach(function(tx) {
    if (!tx.id) {
      tx.id = 'tx_legacy_' + Date.now().toString(36) + '_' + (migrated++);
      tx.created_at = tx.created_at || (tx.date + 'T00:00:00.000Z');
      tx.ref_id = tx.ref_id || '';
      tx.tx_hash = tx.tx_hash || '';
    }
  });
  if (migrated > 0) localStorage.setItem('camp_data', JSON.stringify(data));
}
```

### 1.5 约束规则

1. **统一入口**：所有 NT 变动必须通过 `recordTransaction()`，禁止直接 `data.finance.push()`。现有 2 处 + 社区空间新增 5 处全部使用此入口。
2. **原子绑定**：状态变更 + 记账 + 持久化 = 同一步完成。
3. **类型枚举**：使用 `TX_TYPES` 常量，不写裸字符串。
4. **关联 ID**：`ref_id` 指向业务对象（任务/物品/订单/游戏局），出问题时回溯。
5. **余额缓存**：暂不实现。`data.finance` 在数百条量级下 O(n) 遍历无性能问题。若将来超过 1000 条再考虑缓存层。

### 1.6 Web3 预留

```
当前阶段：Dashboard = 记账本，计算正确即可
后续阶段：recordTransaction 中 tx_hash 字段
          → 钱包支付后回填 → verifyTx(tx_hash) 核对链上数据
```

### 2.2 记账接口契约

社区空间所有模块的 NT 交易必须通过 `recordTransaction()`：

| 业务 | 交易类型 | from | to | scope |
|------|---------|------|-----|-------|
| 素社预定 | `CANTEEN_PREORDER` | 食客 | 高琳姐 | personal |
| 素社现场 | `CANTEEN_WALKIN` | 食客 | 高琳姐 | personal |
| 集市购买 | `BAZAAR_TRADE` | 买家 | 卖家 | personal |
| 茶馆发帖 | `TEAHOUSE_POST` | 作者 | 系统 | personal |
| 桌游报名费 | `GAME_FEE` | 玩家 | 发起人 | personal |

### 2.3 桌游报名费规则（审查补充）

> 发起人设定报名费（0 = 免费）。收费局在游戏结束后由发起人确认到场名单，**未到场者自动退款**。报名费归发起人所有，平台不抽成。发起人虚假发布收费局骗取报名费，管理员可手动撤销并处罚。
