**已完成的紧急修复：**
- [x] `recalcFinanceByPerson()` 缺失闭合括号 → 导致所有渲染函数变成局部函数
- [x] 删除 snap/snapshot 快照功能
- [x] 声明缺失的 `memViewType` 变量

## 第1项: NPC材料表增加流水入口 + 总额对比

### 问题描述

1. 南塘NPC的材料表展示了「预算材料表」+「实际使用表」+「预算vs实际对比」
2. 实际使用表数据从 `data.finance_cny.transactions` 读取
3. **但是**材料表界面没有任何按钮或入口去新增流水记录
4. 对比逻辑做的是逐项模糊匹配（用 `indexOf` 匹配物品名），复杂且容易匹配错误
5. 用户需求：只对比总额，不逐项匹配

### 涉及函数

| 函数 | 位置 | 作用 |
|------|------|------|
| `getPersonMaterials(name)` | 约第1601行 | 从 tasks 提取材料预算，从 finance_cny.transactions 提取实际支出，做对比 |
| `renderMaterialSheet(name, memInfo)` | 约第1668行 | 渲染材料表 HTML（预算表+实际表+对比区） |
| `renderMemberDetailFor(type, name, m)` | 约第1734行 | 渲染NPC详情卡片，含材料表 |


## 第3项: 日历同步 — executeSync 传日期到 data.schedule

### 问题描述

`executeSync()` 在做 sync 时，对日程表的处理是：
```javascript
// 当前代码（约第5097-5108行）
if (data.camp_dates && data.camp_dates.start) {
  if (!data.schedule) data.schedule = {};
  data.schedule.dates = (data.schedule.dates && data.schedule.dates.length > 0)
    ? data.schedule.dates : [];
  // 如果 slots 为空，创建默认空时段
  if (!data.schedule.slots || data.schedule.slots.length === 0) {
    data.schedule.slots = [ /* 8个默认空时段 */ ];
  }
}
```

**问题：** 它只保证 slots 不为空，但没有把 Step 1 日程表的 `dates` 和 `weekdays` 传递过去。如果 `data.schedule.dates` 在 Step 1 已经被 `rebuildScheduleDates()` 填充了，那就能用；但如果因为某些原因没填充，日历 tab 就显示空。

### 修改内容

增加一步：如果 `data.schedule.dates` 为空但 `data.camp_dates.start` 有值，则调用 `rebuildScheduleDates()` 重新生成。

```javascript
if (data.camp_dates && data.camp_dates.start) {
  if (!data.schedule) data.schedule = {};
  // 新增：如果 dates 为空，从 camp_dates 重新生成
  if (!data.schedule.dates || data.schedule.dates.length === 0) {
    rebuildScheduleDates();
  }
  // 确保 slots 不为空
  if (!data.schedule.slots || data.schedule.slots.length === 0) {
    data.schedule.slots = DEFAULT_SLOTS.map(function(s) {
      return { section: s.section, time: s.time,
        activities: new Array(data.schedule.dates.length).fill('') };
    });
  }
  // 确保 activities 长度与 dates 匹配
  var dLen = data.schedule.dates.length;
  data.schedule.slots.forEach(function(s) {
    if (!s.activities) s.activities = [];
    while (s.activities.length < dLen) s.activities.push('');
  });
}
```

### 影响范围

- 修改：`executeSync()` 中日程同步段落
- **不影响：** 其他 sync 逻辑（任务展开、里程碑同步等）

## 第4项: 里程碑从活动日行内容读取锚点

### 问题描述

**现状 `generateMilestones()`（约第3627行）有三处硬编码：**

```javascript
function generateMilestones() {
  // ① 硬编码：schedDates[1] = 开营日
  var d1Parts = schedDates[1].split('/');
  var startD = new Date(2026, parseInt(d1Parts[0])-1, parseInt(d1Parts[1]));

  // ② 硬编码：最后一项 = 结营日
  var lastParts = schedDates[schedDates.length-1].split('/');
  var endD = new Date(2026, parseInt(lastParts[0])-1, parseInt(lastParts[1]));

  // ③ 硬编码覆盖用户原始设置！
  data.camp_dates.start = startD.toISOString().slice(0,10);
  data.camp_dates.end = endD.toISOString().slice(0,10);
  data.camp_dates.duration_days = campDays;
  document.getElementById('wizStartDate').value = data.camp_dates.start; // 改了UI!

  // ④ 偏移计算依赖 startD 和 endD，中间锚点（合影、展览）都是公式算的
  PRESET_MILESTONES.forEach(function(pm) {
    var d = calcMilestoneDate(startD, endD, campDays, pm.offset);
    // ...
  });
}
```

**另外 `renderDailySchedule()` 中的锚点检测（约第3760-3763行）也是硬编码：**
```javascript
var photoDay = Math.floor(duration / 2);      // 合影日=一半
var exhDay = duration - 2;                     // 展览日=倒数第2天
var lastDay = duration - 1;                    // 结营日=最后一天
var anchorSet = {}; [0, photoDay, exhDay-3, exhDay-2, exhDay-1, exhDay, lastDay]
  .forEach(function(d){ if(d>=0) anchorSet[d]=true; });
// 注意: index 0 = 报道日也被标记为锚点！用户要求报道日不锁
```

### 问题汇总

| # | 问题 | 影响 |
|---|------|------|
| 1 | 硬编码 `schedDates[1]` = 开营日 | 如果日历被手动编辑（插入了列），index 1 可能不再是开营日 |
| 2 | 合影日、展览日用公式计算 | 没有从日历内容获取，与用户实际安排脱节 |
| 3 | `generateMilestones()` 覆盖 `data.camp_dates.start` | 改写了用户在 Step 1 最初设定的营期起点 |
| 4 | 报道日 (index 0) 被标记为锚点(🔒) | 用户要求报道日不锁，只有开营日/结营日/展览日/合影日才锁 |

### 修改方案

#### A. 新增辅助函数：从活动日行读取锚点日期

```javascript
function findAnchorDates() {
  // 找到 "🎯 活动日" 这个 slot
  var activitySlot = (data.schedule.slots || []).find(function(s) {
    return s.section && s.section.indexOf('活动日') >= 0;
  });
  if (!activitySlot) return null;

  var activities = activitySlot.activities || [];
  var dates = data.schedule.dates || [];

  var anchors = {};
  activities.forEach(function(cell, i) {
    if (!cell) return;
    if (cell.indexOf('开营') >= 0 && !anchors.openDay)
      anchors.openDay = { index: i, date: dates[i], label: cell };
    if (cell.indexOf('结营') >= 0 && !anchors.closeDay)
      anchors.closeDay = { index: i, date: dates[i], label: cell };
    if ((cell.indexOf('展览') >= 0 || cell.indexOf('布展') >= 0) && !anchors.exhDay)
      anchors.exhDay = { index: i, date: dates[i], label: cell };
    if ((cell.indexOf('合影') >= 0 || cell.indexOf('拍照') >= 0) && !anchors.photoDay)
      anchors.photoDay = { index: i, date: dates[i], label: cell };
  });

  return anchors;
}
```

#### B. 修改 `generateMilestones()`

```javascript
function generateMilestones() {
  var schedDates = data.schedule.dates;
  if (!schedDates || schedDates.length < 2) { /* 错误处理 */ }

  // ① 从活动日行读取锚点
  var anchors = findAnchorDates();
  if (!anchors || !anchors.openDay) {
    alert('请先在每日日历的「🎯 活动日」行填写"开营日"标记');
    return;
  }

  // ② 获取锚点日期
  var openDate = parseDateFromMD(anchors.openDay.date);
  var closeDate = anchors.closeDay
    ? parseDateFromMD(anchors.closeDay.date)
    : parseDateFromMD(schedDates[schedDates.length - 1]); // fallback

  // ③ 不再覆盖 data.camp_dates.start！
  // 只把里程碑写入 data.camp_dates.milestones

  // ④ 以开营日为基准生成里程碑
  var campDays = closeDate ? daysBetween(openDate, closeDate) : schedDates.length - 1;
  var ms = [];
  PRESET_MILESTONES.forEach(function(pm) {
    var d = calcMilestoneDate(openDate, closeDate, campDays, pm.offset);
    // 如果有展览日锚点且 offset 包含 'ex'，用实际展览日
    if (anchors.exhDay && typeof pm.offset === 'string' && pm.offset.indexOf('ex') >= 0) {
      d = calcDateFromAnchor(anchors.exhDay.date, pm.offset);
    }
    ms.push({ id: pm.id, name: pm.name, icon: pm.icon, date: d.toISOString().slice(0,10), locked: pm.locked });
  });

  data.camp_dates.milestones = ms;
  localStorage.setItem('camp_data', JSON.stringify(data));
  recalcMilestonesUI();
}
```

#### C. 修改 `renderDailySchedule()` 中的锚点锁定

**当前：** 用计算出的 anchorSet 标记 `🔒`

**改为：** 用 `findAnchorDates()` 的结果来标记。只有真正的锚点列（开营日、结营日、展览日、合影日）显示 🔒，报道日不锁。


## 执行策略

1. **每项独立完成**，完成后手动验证
2. **每项只改动 1-3 个函数**，改动范围可控
3. **不一次性修改多个不相关模块**，降低引入新 bug 的风险
4. **利用 localStorage 的 camp_data 做备份**，出问题可以恢复
