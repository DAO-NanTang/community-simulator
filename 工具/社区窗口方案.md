## 一、删除社区池子

### 1.1 为什么删

池子的设计意图是「NPC/游客从池子里花钱打赏」。但实际上：
- NPC/游客个人 NT 余额为 0
- `sendTip` 第一步就检查个人余额 → 失败
- 池子检查永远走不到
- **所有实际发生的打赏都是共建者/冒险者从自己账户出的**

池子是死代码，删掉。

### 1.2 删什么

| 删什么 | 位置 |
|------|------|
| Wizard 终端「本期总额 NT」输入框 | 行 865 |
| Wizard 终端「每人每日上限 NT」输入框 | 行 866 |
| `data.budget.community_pool_total` 的初始化 | 行 1833 |
| `data.budget.community_pool_daily` 的初始化 | 行 1834 |
| Wizard 终端读取池子值 | 行 8675-8676 |
| `sendTip` 中 NPC/游客每日额度检查 | 行 7734-7740 |
| `renderCommunityWindowEmbedded` 统计卡片「💎池余」 | 改为「缺人」或其他有意义的统计 |
| 方案和文档中提到池子的注释/说明 | — |

### 1.3 打赏逻辑简化

删完后 `sendTip` 的付费礼物流程：

```
1. 查发送者个人 NT 余额 → 不够就拒绝
2. 写入 data.tips
3. 写入 data.finance（一条 tip 流水）
4. saveData
```

没有池子、没有每日额度、没有身份判断。**所有人打赏都从自己兜里出。**

## 二、面板宽度比例反转

### 2.1 现状 vs 改成

| 面板 | 内容量 | 现状宽度 | 改成 |
|------|:--:|:--:|:--:|
| 一级（社区副本） | 少——三张活动卡片 | 700px（CSS 默认） | **380px** |
| 二级（社区窗口） | 多——脉动+统计+成员+招募+打赏+时间流 | 420px | **520px** |

一级窄、二级宽，符合内容比例。

### 2.2 实现

`_repositionActivityCommunity()` 中修改两个数值：

```
一级 max-width: 440px → 380px
二级 width: Math.min(520, 剩余空间)
```

一级 Modal 的 CSS 默认 `max-width: 700px` 不动（单独打开时够宽）。只在二级面板打开时 JS 收窄。

## 三、视觉居中

### 3.1 垂直居中

`.village-modal` 改为 `align-items: center; padding-top: 0`。

### 3.2 双面板水平居中

`_repositionActivityCommunity()` 中给一级 Modal 加 `margin-left` 偏移：

```
偏移量 = 0 - 二级面板宽度 / 2
```

两个面板的几何中心 = 屏幕中心。

## 四、打赏选人

### 4.1 现状

```
onclick="var t=prompt('送给谁？');if(t){sendTip(...)}"
```

裸文本输入框，手打名字。

### 4.2 方案

在 `renderCommunityWindowEmbedded()` 打赏面板顶部加目标选择器：

```
💬 送个鼓励
送给：[选择成员 ▾]          ← 点击弹出选人列表（复用 showReviewerPicker）

[👏太棒了] [💪加油] …      ← 点这些送给已选的人
[🌹1] [🎁5] …              ← 同上

你的余额：💎 15 NT
```

- 隐藏 input `#activityTipTarget` 存目标名
- 显示按钮 onclick → `showReviewerPicker('activityTipTarget')`
- 打赏按钮 onclick 改为读 `#activityTipTarget` 的值
- 没选人时点打赏 → 提示「请先选择送给谁」
- 选过一次后自动记住（下次点不用重选）

## 六、自查

- **少即是多**：删掉从未生效的池子代码（~15 行），打赏逻辑更简单。面板比例反转只改两个数字。选人复用已有 picker。
- **复用优先**：选人面板复用 `showReviewerPicker`，居中逻辑复用已有 `_repositionActivityCommunity`。
- **需求对齐**：五个改动都对应你指出的问题——靠上、偏右、比例不对、选人难、池子没用。
- **全局最优**：垂直居中让所有 village modal 受益。删池子不影响任何人——它本来就没生效过。
- **长期主义**：删掉死代码减少维护负担。打赏全走个人账户，逻辑统一，以后不会出现「这个 NT 到底从哪出」的困惑。
