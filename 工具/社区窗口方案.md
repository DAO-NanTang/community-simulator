# 社区窗口嵌入社区副本 · 方案文档

> 2026-07-10 | v3 二级滑出面板：不撑长一级 Modal，历史活动保持可见

---

## 〇、为什么旧版是错的

当前社区窗口有**两个入口**：

- 村口 plaza 第 4 张卡片「社区窗口」→ 打开独立 Modal `villageModalCommunity`
- 村口第 1 张卡片「社区副本」→ 打开 Modal `villageModalActivity`

**问题**：

1. 访客需要分别点两张卡片才能了解营地全貌——体验割裂
2. 社区副本里有「成为冒险者」按钮，但看不到社区动态——缺少加入的动力
3. 两张卡片功能重叠，访客困惑该点哪个
4. 上一版方案（折叠展开）把社区窗口塞进第四期卡片内部，内容撑得太长，历史活动被挤到很远——臃肿

**这次做的事**：删除独立社区窗口（卡片 + Modal + 函数），社区窗口改为从社区副本 Modal 的**右侧滑出**的二级面板。一级 Modal 保持原样，历史活动不被遮挡。

### 关键交互特性

| 特性 | 说明 |
|------|------|
| 一级 Modal **不动** | 保持原宽，不修改。第四期卡片不撑长，历史活动正常显示 |
| 二级面板 **从右滑入** | CSS transition `transform: translateX(100%)` → `translateX(0)` |
| 半透明遮罩 | 面板背后有 backdrop，点遮罩关闭面板 |
| 关闭方式 | 点右上角 ✕ / 点遮罩 / 再点「浏览营地」按钮 |
| 不影响历史 | 一级 Modal 的内容状态保持，开关面板不触发重新渲染 |

### 和现有 `hubSubPanel` 的关系

二级面板的交互方式和现有的「南塘云村」子面板**完全相同**——滑出 + 遮罩 + 关闭。可直接复用：

| 现有组件 | 本方案复用 |
|------|------|
| `#hubSlideBackdrop`（遮罩层） | 新增 `#activityBackdrop` 或复用已有遮罩 |
| `#hubSubPanel`（滑出面板） | 新增 `#activityCommunitySlide` |
| `closeHubSub()`（关闭函数，我们之前重构过） | 新增 `closeActivityCommunity()` |
| 滑入/滑出 CSS | 复用 `.hub-sub-panel` 的 transition 样式 |

## 三、实施步骤

### Step 1：删除旧版独立社区窗口

| 删什么 | 位置 |
|--------|------|
| 村口 plaza 第 4 张"社区窗口"卡片 | plaza flex 行 |
| `villageModalCommunity` Modal | village modals 区域 |
| `openCommunityWindow()` 函数 | JS |
| `openVillageModal` / `closeVillageModal` 的 community 分支 | JS |

### Step 2：在社区副本 Modal 后加二级滑出面板的 HTML

新增三个 DOM 元素（放在 `villageModalActivity` **之后**，作为独立元素，不在 Modal 内部）：

```
<!-- 社区窗口遮罩 -->
<div id="activityBackdrop" class="hub-slide-backdrop" onclick="closeActivityCommunity()" style="display:none"></div>

<!-- 社区窗口滑出面板 -->
<div id="activityCommunitySlide" class="hub-sub-panel" style="right:0;left:auto;width:420px;max-width:90vw">
  <div class="hs-header">
    <span>👀 浏览营地</span>
    <button onclick="closeActivityCommunity()">✕</button>
  </div>
  <div class="hs-body" id="activityCommunityBody"></div>
</div>
```

> **为什么放在 Modal 外面**：滑动面板和遮罩需要 `position:fixed` 覆盖整个屏幕。如果放在 Modal 内部，会被 Modal 的 `overflow` 裁剪。

### Step 3：`renderVillageActivityList()` 加「浏览营地」按钮

在「成为冒险者」/「进入营队」按钮旁边加：

```
<button onclick="openActivityCommunity()">👀 浏览营地</button>
```

### Step 4：写 `renderCommunityWindowEmbedded()`

社区窗口的唯一渲染函数，填入 `#activityCommunityBody`。内容同之前的嵌入版（营地脉动 + 统计 + 成员 + 招募 + 打赏 + 刚刚发生 + 引导），但不再需要紧凑字号——滑出面板有 420px 宽度，空间充裕。

### Step 5：加打开/关闭函数

```
openActivityCommunity()  → renderCommunityWindowEmbedded() → 填入 panel → show panel + backdrop
closeActivityCommunity() → hide panel + backdrop
```

> **不需要CSS动画写新的**：`.hub-sub-panel` 已有 `transition` 和 `transform` 动画。给 `#activityCommunitySlide` 加同样的 class 即获得滑入滑出效果。

## 六、自查

- **少即是多**：一个遮罩 + 一个面板 + 一个渲染函数 + 两个切换函数。复用已有 CSS（`hub-sub-panel`），不写新动画。一级 Modal 不加宽、不改造。
- **复用优先**：面板样式和交互完全复用 `hubSubPanel` / `hubSlideBackdrop` 体系（之前在候选 3 重构过，已经很干净）。渲染函数复用已有的数据查询逻辑。
- **需求对齐**：每次改动都有对应问题——臃肿 → 二级面板；一级被挤 → Modal 不变；社区窗口独立 → 删旧版。
- **全局最优**：面板在 Modal 外面（`position:fixed`），不受 Modal 内部 overflow 限制。遮罩层级可高于 Modal 背景但低于面板，视觉层次清晰。
- **长期主义**：只有一个渲染函数，后续维护只改一处。面板和遮罩独立于 Modal，以后如果要在其他地方触发社区窗口，只需调 `openActivityCommunity()` 即可。
