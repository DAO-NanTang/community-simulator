- **目标**：只维护一份源码，一键生成**不含任何内测代码**的干净服务器版，避免误触污染、避免两份大文件分叉

## 一、核心机制

1. **围栏标记**：所有内测代码用统一记号包起来——HTML 块 `<!-- NT-TEST:START -->…<!-- NT-TEST:END -->`；JS 块 `// NT-TEST:START … // NT-TEST:END`。
2. **剥离脚本 `build_server.py`**：逐行读 dashboard.html，遇 `NT-TEST:START` 到 `NT-TEST:END` 之间（含标记行）整段丢弃，其余原样输出为 `dashboard.server.html`。
3. **围栏必须"剥掉后仍是合法代码"**——脚本本身极简（只删块），功夫全在标记落点。

**工作流**：平时只改 `dashboard.html`（本地带内测工具供你测）；上传前 `python build_server.py` → 干净的 `dashboard.server.html` 上传。新功能只改一处，服务器版自动重生成，**永不分叉**。

## 三、批 a 采用 S 的理由（已定）

批 a 在生产是休眠的，但 S（一并剥掉）能让服务器版**零时间注入**，彻底关闭"有人手动 console 设 `nt_virtual_ms` → 冻结自己时钟 → 提交写错日期污染共享库"这条边角风险。契合"服务器版彻底干净"的初衷，且只是把 Clock 重排成围栏友好形式，改动很小。

---

## 四、Clock 的 strip-clean 围栏写法（S 关键落点）

把批 a 拆成三处围栏，**剥掉后自然退回纯真实时间**：

**① 冻结变量与持久化**（整段进围栏）
```
function _wall() { return Date.now(); }
// NT-TEST:START
var _frozenMs = null;
try { ...读 nt_virtual_ms... } catch (e) {}
function _persistV() { ...写/删 nt_virtual_ms... }
// NT-TEST:END
```

**② `_now()` 内只围栏冻结分支**（剥后剩纯真实时间）
```
function _now() {
  // NT-TEST:START
  if (_frozenMs != null) return _frozenMs;
  // NT-TEST:END
  return _wall();
}
```

**③ 返回对象里注入方法用"尾逗号 + 注释围栏"**（剥后 stamp 行尾逗号在 `}` 前合法，ES5.1+）
```
    stamp: function () { return _wall().toString(36); },
    // NT-TEST:START
    _setFrozen: function (ms) { ... },
    _step: function (deltaMs) { ... },
    _reset: function () { ... },
    _state: function () { ... },
    // NT-TEST:END
```

剥离后：`_now()` = `{ return _wall(); }`；无 `_frozenMs`/`_persistV`；返回对象仅 `stamp` 一个方法——语法完整合法（`stamp: fn, }` 尾逗号在 ES5.1+ 合法）。

> **教训**：前置逗号（`,_setFrozen`）在 JS 对象字面量中不合法——逗号必须是前一属性的尾部而非后一属性的首部。已在实际执行中修正为正常尾逗号。

> 注：现有批 a 的 `_now()` 是三元式写法，批 1 会把它重排成上面②的形式（等价，仅为可干净剥离）。

## 五、`build_server.py` 行为
- 输入 `dashboard.html`，输出 `dashboard.server.html`。
- 删除所有 `NT-TEST:START…END` 区块（含标记行），其余照抄。
- **配对检查**：如果遇到 `START` 但没有对应的 `END`（或反之），报错退出并提示行号——防止标记不配对导致静默截断。
- 产出后**自检**：抽取主 `<script>` 跑 `node --check`，确保剥后 JS 语法通过（沿用本项目已用的校验法）。
- 纯文本处理、无业务逻辑、稳定可复用。
