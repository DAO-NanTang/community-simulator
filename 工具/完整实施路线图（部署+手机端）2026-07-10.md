## 〇、v2 关键修订（相对上一版路线图）

上一版设想"手机做独立精简版（第二个前端）"。经砚仁澄清范围后修正：

- **手机端范围 ≈ 全部参与者视角**（共建者/冒险者/南塘NPC/云村民的任务、余额、日历、数据、南塘云村各展示），**同一个域名**；只有**管理员"世界端"（创世向导/设置）保持桌面专用**。
- 在此范围下，"独立精简版"= 几乎重写整个 app + 双倍维护 → **改为响应式改造现有这一份代码**。
- 由此得出：**后端多人化 与 手机响应式 是两条正交的线**（数据层 vs 展示层），可独立推进。

## 一、两条独立的线

```
A 线 · 后端多人化      SQLite(WAL)+Express+SSE+JWT+乐观更新   —— 数据层，解决"多人协作"
B 线 · 手机响应式      改现有 dashboard 的 CSS/布局           —— 展示层，解决"手机能看能用"
        （管理员世界端保持桌面专用）
```

- 两条线**不互相依赖**：响应式是纯布局，不需要后端；后端是数据，不需要响应式。
- 可先后、可并行。先做哪条取决于**哪个更急**（见 §6）。
- 共同前提：都遵循已落地的 **NT-TEST 围栏**（测试/生产分叉）。

## 四、B 线 · 手机响应式改造（同一代码，参与者视角）

**原则**：新增移动端覆盖层（`@media`），只在手机生效、桌面不受影响；管理员世界端在手机上提示"请用电脑操作"。

| 阶段 | 内容 |
|---|---|
| B0 全局护栏 | 消灭横向溢出：容器流式、输入/卡片 `max-width:100%`、表格套滚动、flex 强制换行。**性价比最高，先做** |
| B1 高频参与者屏 | 登录/注册、我的任务、提交任务、任务大厅、余额、通知——逐屏调到好用 |
| B2 南塘云村各屏 | 集市/客栈/食堂/茶馆/桌游/档案/拍卖/议事厅——侧栏改手机全宽/底部抽屉 |
| B3 展示类 | 时间线/日历、成员档案、数据统计——宽表格改卡片式、双列改单列 |
| B4 世界端处理 | 管理员创世向导/设置：手机上隐藏或提示"请用电脑操作"（不强行适配） |

> 诚实预期：现有 app 是桌面优先 + 大量内联样式，响应式是**逐屏迭代**、要真机反复看，不是一次到位。B0 做完手机"整体崩坏"感会去掉一大半，其余逐屏打磨。

## 五、两条线如何对接

1. **NT-TEST 围栏**：两条线的新代码都遵循，`build_server.py` 一键出干净生产版。
2. **渲染函数共享**：A 线的局部刷新函数（`updateTaskRow` 等）产出的 DOM，由 B 线的响应式 CSS 负责在手机上排版——两层解耦、不冲突。
3. **测试环境**：内测/时间工具指向"测试数据库"（A 线建好后），替代旧 localStorage 沙盒。
4. **顺序无强制**：谁先做都行；若先做 B 再做 A，A 的乐观更新只改数据接线、不动布局，返工极小。
