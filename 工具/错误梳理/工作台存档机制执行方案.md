## 〇、要解决的问题

工作台里所有历史任务（我发布的、我完成的）永远堆在一起，没有时间边界。随着系统运行时间增长，任务列表无限膨胀。

**目标**：已完成/已撤回的任务按 7 天分界——7 天内为「近期」，正常显示；超过 7 天进入「存档」，默认折叠，展开后可按时段筛选。

## 二、改动设计

### 2.1 辅助函数：计算任务的「完成日期」

新增全局函数，放在 `daysSince` 附近（约 line 1558）：

```js
function getTaskDoneDate(task, claim) {
  // claim 存在 = 我领取的任务；不存在 = 我发布的任务
  if (claim) {
    // 我领取的任务：取审核通过时间
    if (claim.reviewed_at) return claim.reviewed_at;
    return task.posted_at || '';
  }
  // 我发布的任务：
  // 1) 已撤回 → 撤回时间
  if (task.status === 'withdrawn' && task.withdraw_requested_at) return task.withdraw_requested_at;
  // 2) 已完成 → 最后一个领取者审核通过的时间
  var claimants = task.claimants || [];
  var latestDate = '';
  claimants.forEach(function(c) {
    if (c.status === 'completed' && c.reviewed_at && (!latestDate || c.reviewed_at > latestDate)) {
      latestDate = c.reviewed_at;
    }
  });
  if (latestDate) return latestDate;
  // 3) 兜底 → 发布时间
  return task.posted_at || '';
}
```

判断「超过 7 天」：`daysSince(doneDate) > 7`

### 2.2 数据采集层改动

#### 改动 A：扩展「我发布的」收集

**位置**：行 3854-3860

**改前**：只收集 `active` 和 `pending_approval` 任务。

**改后**：分三个数组收集——

```
myPublishedActive    — 进行中的（active + pending_approval）→ 始终显示
myPublishedRecent    — 已完成/已撤回且 ≤7 天 → 展开显示
myPublishedArchive   — 已完成/已撤回且 >7 天 → 折叠到存档
```

收集逻辑（伪代码）：
```
遍历 tasks：
  if (t.poster !== name) 跳过

  if (t.status === 'active' || t.status === 'pending_approval'):
    → myPublishedActive.push({task: t, idx: i})

  else if (t.status === 'completed' || t.status === 'withdrawn' || t.status === '已确认'):
    doneDate = getTaskDoneDate(t, null)
    if (daysSince(doneDate) <= 7):
      → myPublishedRecent.push({task: t, idx: i, doneDate: doneDate})
    else:
      → myPublishedArchive.push({task: t, idx: i, doneDate: doneDate})
```

#### 改动 B：拆分「已完成」

**位置**：行 4061-4079（渲染部分）、行 3827-3839（收集部分）

在现有的 `allMyTasks` 收集逻辑不变的基础上，渲染时把 `doneTasks`（`claim.status === 'completed'`）拆成两组：

```
doneTasks → 按 doneDate = getTaskDoneDate(e.task, e.claim) 拆分：
  doneRecent   — daysSince(doneDate) <= 7
  doneArchive  — daysSince(doneDate) > 7
```

### 2.4 存档渲染函数

新增全局函数 `renderArchiveList(containerId, archiveTasks, viewMode)`：

**参数**：
- `containerId`：目标 DOM 元素 ID（`'wsPubArchiveList'` 或 `'wsDoneArchiveList'`）
- `archiveTasks`：`[{task, idx, doneDate, ...}]` 数组
- `viewMode`：`'week'` | `'month'` | `'year'` | `'all'`

**行为**：

**全部模式**：按 `doneDate` 倒序排列，直接渲染每个 task 的 `renderQuestCard`。

**周模式**：按自然周分组（周一到周日）。`doneDate` 解析为 Date → 计算该日期所在周的周一日期作为分组 key → 同组 tasks 归到一起。组标签格式：
- 本周：`"本周 M/D – M/D"`
- 上周：`"上周 M/D – M/D"`
- 更早：`"M/D – M/D"`

**月模式**：按 `YYYY-MM` 分组。组标签：`"YYYY年M月"`。

**年模式**：按 `YYYY` 分组。组标签：`"YYYY年"`。

每组默认折叠（`style="display:none"`），点击组标题展开。组标题右侧显示任务数量。

### 2.5 存档切换交互

新增两个全局状态变量（放在 `var flowFilter = 'all'` 附近，约 line 4093）：

```
var wsPubArchiveView = 'week';    // 'week' | 'month' | 'year' | 'all'
var wsDoneArchiveView = 'week';
```

存档区域的 toggle 和筛选通过以下函数处理：

```
toggleArchive(sectionId)
  — 切换 sectionId 对应的存档 div 显示/隐藏
  — sectionId: 'wsPubArchive' | 'wsDoneArchive'

switchArchiveView(sectionId, mode)
  — 切换存档筛选模式
  — 更新对应的全局状态变量
  — 调用 renderArchiveList 重新渲染
```

组内展开/折叠通过内联 onclick 直接操作 DOM。

### 2.6 任务卡片的「完成日期」显示

存档中的任务卡片本身不做修改（复用 `renderQuestCard`）。但在存档视图的每组标题旁边显示日期范围即可。如需在卡片上额外显示完成日期，可在 `renderQuestCard` 的 opts 中新增 `doneDate` 字段——**本次不做，后续按需添加**。
