## 一、成员目录 UI 升级

### 1.1 现在的样子

```
📂 在地成员（3人）
  砚仁  💎1500  120天
  朝林  💎800   90天
  小红  💎500   60天
📂 共建者（8人）
  若雨  💎2000  150天
  ...（8个人全部展开，很长）
📂 冒险者（15人）
  ...（15个人，要滚很久）
📂 云村民（20人）
  ...（20个人，更难找）
```

**问题**：所有分组全部展开，人多了之后滚不到底。没有搜索，想找一个人要肉眼扫。

### 1.2 改成什么样

```
📚 档案室
  [📂成员] [📋归档任务] [📦往期营期] [📜日志] [📝社区档案] [📊结算]

  🔍 [搜索姓名…]  排序：[💎NT▾]

  ▸ 📂 在地成员（3人）           ← 折叠状态，点一下展开
  ▾ 📂 共建者（8人）             ← 展开状态
    若雨   💎2000  150天  [案卷]
    淑惠   💎1200  100天  [案卷]
    ...
  ▸ 📂 冒险者（15人）            ← 折叠
  ▸ 📂 云村民（20人）            ← 折叠
```

每个功能点：

| 功能 | 效果 |
|------|------|
| 🔍 搜索框 | 输入名字，实时过滤所有分组。搜索结果跨分组显示 |
| 折叠分组 | 点分组标题 → 展开/折叠。默认只展开第一个有人的分组 |
| 排序 | 下拉选：💎 NT 从高到低 / 📅 注册时间 / 🔤 姓名 |
| [案卷] 按钮 | 点击 → 打开个人案卷（就是现在的 openMemberDossier） |

### 1.3 实现

改动 `renderArchiveHub` 函数内部：

1. 渲染搜索框 + 排序下拉
2. 分组标题加 `onclick` 折叠/展开
3. 搜索输入时过滤成员列表
4. 排序切换时重新排列

改动量：~30 行（在现有 renderArchiveHub 内加搜索/折叠/排序逻辑）。

## 二、运行日志

### 2.1 日志记录什么

系统已经在执行各种操作（发布任务、审核通过、打赏、角色变更），但这些操作**没有留下日志记录**。现在加一个轻量日志：

每次关键操作发生时，在 `data.activity_log` 数组追加一条记录。

| 操作 | 日志内容示例 |
|------|------|
| 发布任务 | `[07-10 14:30] 砚仁 发布了「绘制营地地图」· NT15` |
| 领取任务 | `[07-10 14:35] 宏心 领取了「绘制营地地图」` |
| 审核通过 | `[07-10 14:40] 砚仁 审核通过 宏心 →「绘制营地地图」· +15NT` |
| 打赏 | `[07-10 14:45] 砚仁 → 宏心 🌹小花 1NT` |
| 角色变更 | `[07-10 14:50] 砚仁 将 小明 从 冒险者 变更为 共建者` |
| 成员编辑 | `[07-10 14:55] 砚仁 编辑了 宏心 的成员资料` |
| 捐献社区 | `[07-10 15:00] 宏心 捐献 20 NT → 社区池` |
| 录入档案 | `[07-10 15:10] 砚仁 录入了社区档案「张老师」· +5 NT` |
| 核查档案 | `[07-10 15:20] 朝林 核查通过「张老师」· +3 NT` |
| 池子支出 | `[07-10 15:30] 砚仁 申请支出 50 NT · 朝林 批准` |

每条日志格式：
```
{ time: "2026-07-10T14:30:00", type: "task_post", text: "砚仁 发布了「绘制营地地图」· NT15" }
```

### 2.2 在哪里加日志

在现有的操作函数里各加一行 `push` 日志：

| 函数 | 加什么 |
|------|------|
| `confirmPostTask` | 发布任务日志 |
| `confirmCampTask` | 发布营队任务日志 |
| `reviewTask`（审核通过/退回） | 审核日志 |
| `sendTip` | 打赏日志 |
| `changeUserRole` | 角色变更日志 |
| `saveMember` | 成员编辑日志 |
| `donateToCommunityPool`（新增） | 捐献日志 |
| `submitArchiveEntry`（新增） | 录入档案日志 |
| `verifyArchiveEntry`（新增） | 核查档案日志 |
| `approvePoolPayout`（新增） | 池子支出日志 |

每个加 1 行，总共 ~10 行。

### 2.3 日志初始化

在 `data` 初始化时加 `if (!data.activity_log) data.activity_log = []`。

### 2.4 日志显示

在 Hub 档案室加第四个标签「📜 日志」，渲染 `data.activity_log`，按时间倒序，支持滚动浏览。

## 六、自查

- **少即是多**：搜索框一个 input、折叠用原生 JS toggle、排序一个 select、导出一个按钮。不引入任何库。日志是纯数组 push。
- **复用优先**：成员目录基于现有 `renderArchiveHub` 改，不重建。导出函数复用现有的 `getTaskDoneDate`、`daysSince`、`ARCHIVE_DAYS`。
- **需求对齐**：目录难找 → 搜索+折叠；不知道操作 → 日志；数据导不出 → MD 导出。
- **全局最优**：日志覆盖所有关键操作（任务/审核/打赏/角色/成员），以后查问题不需要翻代码。
- **长期主义**：日志是增量数组，不会无限增长（可加截断）。MD 导出格式清晰，以后加 PDF 或 HTML 导出可直接复用数据收集逻辑。
