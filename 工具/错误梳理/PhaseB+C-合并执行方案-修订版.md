## 〇、修订说明：原方案的三处修正

基于对交互体验的深入讨论，本修订版对原 Phase B 方案做了三处关键修正：

| # | 原方案 | 修订后 | 原因 |
|:--:|------|------|------|
| 1 | 锁 = 动作闸门（手动 `unlockStep`，线性推进） | 火苗 = 数据标识（自动根据填写状态显示火苗大小） | 锁让人感觉"被拦住"，火苗让人感觉"在生长" |
| 2 | 向导嵌在 Settings 标签里，上方有标签栏和 HUD | 进入工坊后标签栏收起，全宽沉浸式体验 | 数据填写需要沉静，减少干扰 |
| 3 | 必须 ⓪→①→②→③→④ 线性推进 | ⓪ 完成后，①②③④ 自由导航 | 用户需要反复查看和修改之前的数据 |

## 一、设计哲学：火苗成长体系

### 1.1 核心理念

整个创营过程是一个**火苗从无到有、从小到大的成长故事**，而不是一个待办清单。

```
🕯️ 未燃 → 🔥 初燃 → 🔥 渐亮 → 🔥 添柴 → 🔥🔥 旺盛
   │         │         │         │         │
 创营前    ⓪完成     ①完成     ③完成     ④启动
```

### 1.2 火苗状态表（替代原来的 🔒/✅/⚠/⬜）

每个步骤下方的火苗图标由该步骤的**数据填写程度**决定：

| 火苗 | 名称 | 触发条件 | 含义 |
|:---:|------|------|------|
| 🕯️ | 未燃 | Step ⓪ 尚未保存 | 火种还在等待点燃，其他步骤的入口也未开启 |
| 🕯️ dim | 待填 | Step ⓪ 已完成，但此步骤尚无任何数据 | 可以进入，但里面还是空的 |
| 🔥 | 初燃 | 步骤中有部分数据（如只填了名称没填预算） | 火苗已点亮，但还不够旺 |
| 🔥+ | 渐旺 | 步骤中核心数据已完整 | 火焰稳定，这步基本完成了 |
| 🔥🔥 | 旺盛 | 所有步骤数据完整，等待启动 | 万事俱备，只差点火 |
| 🔥✨ | 编辑中 | 当前正在查看/编辑此步骤 | 火苗带微光呼吸动画，提示"正在这里" |

**导航条示意：**

```
刚进入工坊（⓪ 尚未保存）：

   ⓪ 创营       ① 统筹       ② 任务池     ③ 选将       ④ 启动
    ⚠ 编辑中      🕯️ 待点燃     🕯️ 待点燃     🕯️ 待点燃     🕯️ 待点燃
   （可操作）     （灰色不可点）  （灰色不可点）  （灰色不可点）  （灰色不可点）


⓪ 保存后，开始填 ①：

   ⓪ 创营       ① 统筹       ② 任务池     ③ 选将       ④ 启动
   ✅ 已完成      ⚠ 编辑中      🕯️ 待填       🕯️ 待填       🕯️ 待填
    🔥            🔥✨          🕯️ dim       🕯️ dim       🕯️ dim


逐步填写中：

   ⓪ 创营       ① 统筹       ② 任务池     ③ 选将       ④ 启动
   ✅ 已完成      ✅ 已完成      ⚠ 编辑中      🕯️ 待填       🕯️ 待填
    🔥            🔥+           🔥✨          🕯️ dim       🕯️ dim


全部填完，准备启动：

   ⓪ 创营       ① 统筹       ② 任务池     ③ 选将       ④ 启动
   ✅ 已完成      ✅ 已完成      ✅ 已完成      ✅ 已完成      ⚠ 编辑中
    🔥            🔥+           🔥+           🔥+           🔥🔥✨
```

### 1.3 火苗状态的判断逻辑

```
function getStepFlame(n) {
  if (n === 0 && !campCreated) return '🕯️未燃';  // ⓪ 是唯一的硬前置

  var fillLevel = checkStepFillLevel(n);  // 0=空, 1=部分, 2=完整

  if (isCurrentStep(n)) {
    return fillLevel === 0 ? '🔥✨编辑中' :
           fillLevel === 1 ? '🔥✨编辑中' : '🔥🔥✨编辑中';
  }

  if (fillLevel === 0) return '🕯️待填';
  if (fillLevel === 1) return '🔥初燃';
  return '🔥+已完整';
}
```

## 二、交互模式：沉浸式数据工坊

### 2.1 进入工坊

点击 HUD 栏的「🌍 创世终端」按钮后：

1. 顶部标签栏**收起**（`slide-up` 动画，200ms）
2. HUD 栏**精简**：只保留用户身份、NT 余额、「← 退出工坊」按钮、信箱
3. 向导内容**全宽展开**
4. 底部出现**状态栏**：`当前营队「第四期南塘共创营」· 🔥+ 渐旺 · 砚仁 · 最后修改 07-15`

```
进入工坊前：                          进入工坊后：
┌────────────────────────┐          ┌──────────────────────────────┐
│ 👤 砚仁 💎NT [🌍终端]  │          │ 👤 砚仁 💎NT [← 退出工坊]   │
│ 👤砚仁│👥成员│🏛️大厅│..│          │                              │
│                        │          │  ⓪创营 ①统筹 ②任务 ③选将 ④启动 │
│   （当前标签内容）      │          │  🕯️     🕯️     🕯️    🕯️    🕯️  │
│                        │          │                              │
│                        │          │    （表单全宽展开）            │
│                        │          │                              │
│                        │          │  营队：未创建 · v0            │
└────────────────────────┘          └──────────────────────────────┘
```

### 2.2 退出工坊

点击「← 退出工坊」或切换到其他功能时：

1. 标签栏**恢复**（`slide-down` 动画，200ms）
2. HUD 恢复完整
3. 数据**已自动保存**（每次步骤切换或表单失焦时触发 `saveData()`）
4. 再次进入工坊 → **自动跳转到上次编辑的步骤**

### 2.3 自由导航规则

| 动作 | 行为 |
|------|------|
| 点击任何步骤按钮 | 只要 ⓪ 已保存，①②③④ 全部可点击 |
| 点击「← 退出工坊」 | 回到工作台 |
| 点击信箱 📬 | 退出工坊，打开信箱 |
| 点击浏览器后退 | 退出工坊（如果浏览器支持） |
| 刷新页面 | 数据不丢，回到当前步骤 |
| 仪式进行中 | 全屏覆盖，不可导航（Escape 跳过） |

### 2.4 步骤间导航按钮

每个步骤底部保留「← 上一步」「下一步 →」按钮，方便顺序操作。但同时导航条上的按钮也可以直接跳转。

## 三、仪式系统：两种全屏仪式

### 3.1 创营仪式：点燃火种

**触发**：Step ⓪ 点击「✨ 点燃火种 · 创建营队」

**动画流程**（约 6 秒）：
1. 暗场渐入 → 星空背景 + 星点漂移
2. 中央火种呼吸（🕯️ 一明一暗）
3. 显示营队名称 + 主题 + 创建人
4. 出现「✨ 点燃火种」按钮
5. 点击 → 星点飞向中央 → 火种点燃（🕯️→🔥）→ 光扩散涟漪
6. 完成面板：「🌍 营队正式创立」→ 「进入统筹 →」

**退出方式**：
- 正常：看完动画 → 点击「进入统筹」
- 跳过：按 Escape 或点击右下角「跳过仪式 →」（1.5s 后淡入）

### 3.2 开营仪式：星光汇聚

**触发**：Step ④-4c 点击「🎊 开始开营仪式」

**动画流程**：
1. 暗场 + 中央火种 + N 颗暗淡星辰（每个共建者一颗 ☆）
2. 每人点击「🚀 开启共创」→ 自己的星 ☆→✨ 亮起
3. 就绪计数 N/N → 星光汇聚动画 → 🔥🔥 旺盛
4. 完成面板：「🏁 正式启动！」→ 「进入我的工作台」

**退出方式**：同创营仪式

### 3.3 降级

两个仪式都检测 `prefers-reduced-motion` 和 CSS animation 支持，不满足条件时直接跳过动画，执行回调。

## 九、与原 Phase B 方案的差异总结

| 原方案 | 修订方案 |
|------|------|
| 锁/解锁是动作闸门 | 火苗是数据填写程度的视觉反馈 |
| `unlockStep(n)` 手动调用 | `getStepFlame(n)` 自动判断 |
| 线性流程，不可跳步 | ⓪ 之后自由导航 |
| 锁的图标 🔒 | 火苗的图标 🕯️🔥🔥+🔥🔥 |
| 嵌在 Settings 标签里 | 沉浸式工坊，标签栏收起 |
| Phase B 先做，Phase C 后做 | 合并执行，仪式和表单一次到位 |
