# 世界终端：创营与角色升级方案

> 2026-07-10

---

## 一、现状 vs 目标

### 当前流程

```
世界终端（tab-settings / 🌍 创世终端）
│
├── ① 统筹（营期时间 + 每日安排 + 成员卡片）
│       ↓
├── ② 任务池（统一任务池 + 模板 + AI导入）
│       ↓
├── ③ 认领与分配（选共建者 + 打包任务 + 确认认领）
│       ↓
└── ④ 角色卡（查看每个共建者的任务和物料）

问题：
- 没有「创建营队」的起点，管理员进来就是填营期
- ③ 中选了人、分了任务，但角色不升级（或只在新建 card 时触发）
- changeUserRole 挡住 visitor → builder 路径
```

### 目标流程

```
世界终端
│
├── ⓪ 创营 ── 管理员创建营队，确立营队身份
│       ↓
├── ① 统筹 ── 营期框架 + 每日安排
│       ↓
├── ② 任务池 ── 设计任务 + 导入模板
│       ↓
├── ③ 选将 ── 从成员中选共建者 + 打包任务 + 角色自动升级
│       ↓
└── ④ 角色卡 ── 查看每个共建者的任务和物料
        │
        └── 🚀 发包：任务推送到每个共建者的个人工作台
```

## 四、角色升级逻辑修正

### 当前问题

```
changeUserRole(name, 'builder')
  │
  ├─ 检查：有没有冒险者经历？
  │   └─ ❌ 没有 → return { error: '共建者需先有冒险者经历' }
  │
  └─ 结果：visitor/npc 永远无法直接升 builder
```

### 修正方案

在 `changeUserRole` 中增加一个 `skipAdventurerCheck` 选项：

```
changeUserRole(name, 'builder', { 
  period: '第X期', 
  skipAdventurerCheck: true   ← 世界终端选将时传入
})
  │
  ├─ skipAdventurerCheck = true
  │   └─ → 跳过「需先有冒险者经历」检查
  │
  └─ 正常写入 season_history + 更新角色
```

### 触发时机

| 场景 | 触发 | skipAdventurerCheck |
|------|------|-------------------|
| 世界终端 Step ③ 确认认领 | `changeUserRole(name, 'builder', {...})` | `true` |
| 任务大厅个人认领 | `changeUserRole(name, 'adventurer', {...})` | 不需要 |
| 管理员手动编辑成员 | `changeUserRole(name, 'builder', {...})` | 弹窗确认 |

### 对所有成员生效

当前 `claimTasks()` 中角色升级只在 `!card`（新 card）时执行。修正后：

```
确认认领 → 无论新旧 card：
  1. 创建/更新 staff card
  2. 写入 task.claimants
  3. ★ changeUserRole(name, 'builder', { skipAdventurerCheck: true })
  4. 刷新 UI
```

## 六、验收标准

1. ✅ 管理员进入世界终端，先看到「⓪ 创营」界面
2. ✅ 创建营队后，依次走完 ① ② ③ ④
3. ✅ Step ③ 选人认领后，该成员角色自动变为「共建者」
4. ✅ 成员头像从档案室「云村民」分组移到「共建者」分组
5. ✅ 已存在的旧成员（已有 card）同样升级
6. ✅ 发包后，任务出现在每个共建者的个人工作台
7. ✅ NPC 与云村民同样适用（认领营队任务 = 成为共建者）
