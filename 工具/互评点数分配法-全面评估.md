# 互评点数分配法 —— 全面评估报告

> 基于录音「标准录音 36」中提出的二级互评点数分配方案，进行算例推演与多角度评议。

---

## 一、方案回顾

### 1.1 核心规则

| 要素 | 内容 |
|------|------|
| **点数标尺** | 0、5、13、43、89（离散跳跃，无中间值） |
| **评定方式** | 逐事项、全员互评 |
| **收敛机制** | 亮分→讨论→多轮修正→趋同或掐头去尾 |
| **汇总公式** | 个人比例 = 个人总点数 ÷ 全员总点数 |
| **分配公式** | 个人所得 = 个人比例 × 可分配盈余 |

### 1.2 设计意图（来自对话）

- 强制拉开差距，避免「大家都差不多」导致的搭便车
- 让每个人的付出「被看到」
- 总量约束下只争比例，不争绝对数
- 互相给高分自动打平，抑制通胀

## 二、算例推演

### 2.1 场景设定

- **团队**：张三、李四、王五、赵六、孙七，共 5 人
- **可分配盈余**：¥10,000
- **事项**：3 项任务

### 2.2 逐事项评分

#### 事项一：撰写项目方案

> 张三是主笔（80% 工作量），李四提供关键数据，王五审阅建议，赵六和孙七未参与。

| 孙七 | 13 | 5 | 5 | 0 | — |

> 李四和王五是执行主力，张三远程指导，赵六辅助，孙七未参与。

> 赵六独自解决，张三远程提供了一些指导，其余人未参与。

| 评分者 \ 被评者 | 张三 | 李四 | 王五 | 赵六 | 孙七 |

| 成员 | 事项一 | 事项二 | 事项三 | **总点数** | **比例** | **所得** |
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| 张三 | 112 | 44 | 13 | **169** | 22.8% | ¥2,280 |

| 王五 | 36 | 142 | 0 | **178** | 24.0% | ¥2,400 |

| **合计** | | | | **741** | 100% | ¥10,000 |

### 2.4 算例观察

1. **赵六最高（¥2,910）**：仅全力参与了一项任务，但该项是独自救火，获得了全员一致的高评价。一次英雄行为 ≥ 持续稳定输出。
2. **李四和王五并列第二（¥2,400）**：两项持续投入、表现稳定。
3. **张三第三（¥2,280）**：三项都参与了，但除了事项一挑大梁，其余都是「轻参与」。投入广度最大，但总额反而不如只做一件事的赵六。
4. **孙七为零**：三项任务均无贡献，颗粒无收。

## 三、数学特性分析

### 3.1 离散标尺的效应

标尺值 {0, 5, 13, 43, 89} 的间距比约为 **1 : 2.6 : 3.3 : 2.1**。它不是等距的，而是**加速拉开**的。

**正面效应**：
- 确实防止了「人人都给差不多」导致的分数扁平化
- 迫使评分者做出真实的价值判断

**负面效应**：
- 两个贡献差距不大的人，可能因为刚好跨过标尺档位而被拉开 3 倍差距
- 例如：李四觉得张三值 12 分（不够 13），只能给 5 分 → 张三的实际贡献被低估了 2.6 倍
- **悬崖效应**：贡献的微小差异 → 分数的巨大跳跃

### 3.2 比例制的通胀免疫

假设所有人互给 89 分（极端串谋场景）：

每人从其他 4 人处获得 89 × 4 = 356 分，全员总分相同，比例回到 20%/人。

**确实免疫了「互相给高分」的通胀**，这是该方案最精巧的设计。

但它同时制造了一个**新的漏洞**：

### 3.3 策略性投票漏洞

如果一个联盟（比如李四 + 王五）约定：

- 互相给 89
- 给其他人一律 5

则联盟成员的比例会被大幅放大，而其他人的比例被压到极低——系统没有机制阻止**有组织的不公正评分**。

更隐蔽的变体：不是联盟压低别人，而是**选择性忽视**——对某人的贡献「合理地」给 0 或 5，因为标尺跳跃大，很难被质疑为「不公」。

## 四、激励相容性评估

> 激励相容 = 每个人追求自身利益最大化的行为，恰好也是系统期望的行为。

### 4.1 ✅ 能激励的行为

| 行为 | 激励效果 | 原因 |
|------|:---:|------|
| 参与更多事项 | 中等 | 多参与才能被评分，但广度 ≠ 高度 |
| 在关键事项上全力投入 | **强** | 一次亮眼表现可能超过十次平庸参与 |
| 在他人面前展示贡献 | **强** | 被看到 = 被给分，不可见的贡献不存在 |
| 参与讨论和评分过程 | 中等 | 不参与评分 = 放弃了话语权 |

### 4.2 ❌ 可能扭曲的行为

| 扭曲行为 | 风险 | 原因 |
|------|:---:|------|
| **只做高能见度的事** | 高 | 幕后工作（文档、协调、维护）天然吃亏 |
| **回避渐进式/长期投入** | 高 | 系统奖励「可归因于个人的单次英雄行为」 |
| **夸大自身贡献、贬低他人** | 高 | 讨论环节变成了辩论赛，口才好的人占优 |
| **战略性缺席** | 中 | 如果孙七事前判断自己分不到钱，不如从一开始就不投入 |
| **评分联盟** | 高 | 2-3 人约定互给高分，可以系统性地转移分配 |
| **过度拆分事项** | 中 | 每个人都会争取把工作拆成更多「事项」以增加被评分机会 |

### 4.3 核心困境：广度 vs 深度

算例中张三的问题很典型：

- 张三参与全部 3 件事，但只在 1 件上牵头 → 169 分
- 赵六只参与 1 件事，但全力投入 → 216 分

**这真的是不公平吗？** 不一定。深夜排障的价值可能确实高于写方案。但问题在于：**系统没有提供区分「事项权重」的机制**。事项一和事项三在计算上完全等权，三个事项的总分池天然不同（取决于参与人数），但没有任何人能在事前知道哪件事「更值钱」。

这意味着：**选对事项比做好事项更重要**。这是否合理，值得争议。

## 五、议事规则视角的全面评议

> 以罗伯特议事规则的精神——保护少数方、充分辩论、避免多数暴政——来审视该方案。

### 5.1 方案的优势（应予肯定）

1. **贡献可见化**：这是方案最大的价值。在固定工资或均分制下，贡献差异是不可见的；这个系统强迫每个人对每件事形成判断，本身就是一种组织透明化的实践。
2. **总量硬约束**：不论分配如何争议，总额不会超支。这是比「计件工资」或「时薪制」更安全的设计。
3. **自校正机制**：通货膨胀（互给高分）自动抵消，无需外部审计。
4. **灵活性**：事项数量、参与人数、盈余规模都可以变化，框架不受影响。

### 5.2 方案的风险与缺陷（应予反对或修正）

#### 风险一：多数对少数的结构性压制

如果团队中有 1 个不讨喜的人，即使他工作出色，多数人可以通过「默契地给低分」使其比例大幅缩水。这不是显性的「欺负」，而是标尺的跳跃性掩盖了系统性偏见。

**议事规则类比**：这相当于没有「少数方申诉权」的表决机制。

#### 风险二：没有基准线，新人/边缘者无法入场

孙七在这个周期中贡献为零，得零分。这本身可能合理。但问题在于：
- 如果孙七是因为**没有被分配任务**而非**拒绝任务**，他是否有机会申诉？
- 新人加入时，在建立「被看到」的资本之前，是否注定前几个周期收入为零？
- 如果一个人生病、请假、被排挤——系统没有保护机制。

#### 风险三：讨论环节的民主假象

方案设计者期待「亮分→讨论→趋同」能产生共识。但现实中的群体动力学是：
- 权力不对等：资历深的人给分低，资历浅的人敢公开质疑吗？
- 从众效应：前两个人亮出 43 分后，第三个人即使觉得应该是 13 分，也可能从众。
- 耗时巨大：每件事讨论到趋同，5 人 × 3 事项已经需要大量会议时间——这些时间本身就是成本。

**议事规则类比**：这相当于没有限制辩论次数、没有防止「疲劳通过」机制的无限辩论。

#### 风险四：无法区分「努力」与「结果」

一个人花了 20 小时做一件事但搞砸了，另一个人花了 2 小时但做好了。评分时大家看到的是结果而非过程。如果团队文化是结果导向的，这也许没问题；但如果团队鼓励「尝试」，这个系统会惩罚尝试失败的人。

#### 风险五：零分的设计伦理

¥0 这个结果在心理上极其剧烈。即使一个人贡献确实很低，给 ¥0 和给 ¥500 在数学上差异不大，但在体感上是「被驱逐」和「被认可为成员」的差别。

## 七、改进建议

如果采用此方案，建议增加以下修正规则：

### 7.1 结构性修正

1. **设定基础保障线**：每人每周期有一笔不参与评分的固定基础金（如 ¥500），确保「零分不归零」，保护成员资格底线。录音中提到的 A/B/C 三层切分就是这个思路，但需要明确定义。
2. **事项权重预声明**：在事项开始前，团队先对事项的重要性做一次快速评定（如：战略级 / 日常级 / 辅助级），不同级别的事项总分池有上限。防止「选对事项比做好事项重要」。
3. **匿名评分**：互评阶段匿名提交，讨论阶段再公开辩论。避免从众效应和权力不对等影响第一轮判断。

### 7.2 程序性修正

4. **限定辩论轮次**：每事项最多 2 轮讨论，第 3 轮直接取中位数，防止无限辩论消耗。
5. **引入「挑战」机制**：任何人在看到自己的评分后，有一次机会对某一项低分提出书面质疑，评分者必须回应。这保护了少数方的话语权。
6. **反联盟条款**：如果发现两人之间的互评存在显著偏离整体均值的模式（如 A 给 B 的分数系统性高于其他人给 B 的分数），触发自动审查。

### 7.3 标尺修正

7. **缩小跳跃幅度**：将标尺从 {0, 5, 13, 43, 89} 调整为更平缓的序列，如 {0, 3, 8, 13, 21, 34}（更接近斐波那契），降低「悬崖效应」。
8. **或者改用「排序赋分法」**：每事项每人不是直接打分，而是对参与者进行贡献排序，第 1 名得 N 分，第 2 名得 N-1 分……未参与者得 0 分。这消除了标尺跳跃的问题，且天然防止所有人同分。

## 八、总体判定

| 评判维度 | 结论 |
|:------|------|
| **数学自洽性** | ✅ 成立。比例制 + 离散标尺在数学上没有矛盾。 |
| **能否激励分担事务** | ⚠️ 部分有效。能激励高能见度、高冲击力的事务参与，但对幕后工作、持续性维护、协调沟通等「不可见劳动」激励不足。 |
| **分配是否公平** | ⚠️ 条件性公平。在高度信任、充分沟通的小团队中（≤5 人）可能是公平的；在信任不足或人数较多时，容易被策略性操纵。 |
| **是否优于简单均分** | ✅ 是。均分鼓励搭便车，这个方案至少让贡献差异有了表达渠道。 |
| **是否优于负责人独断** | ⚠️ 各有利弊。负责人独断效率高但依赖个人公正性；互评点数法民主但执行成本高且有从众风险。 |
| **可否直接用于真实工资分配** | ❌ 建议先试点。直接作为唯一分配依据风险过高。建议：先用此方案分配总盘子的 30%-50%（作为激励部分），剩余部分用更稳定的机制（基础工资 + 角色津贴），磨合 3-6 个周期后再决定是否扩大比例。 |

### 一句话总评

> **这是一个精巧的「贡献显影液」，但不是一个稳健的「分配发动机」。** 它最适合作为辅助工具，让不可见的贡献变得可见；但如果直接把它作为主要的工资分配机制，团队需要具备极高的信任基础和成熟的议事文化——而大多数草根组织目前还不具备这样的条件。
