# dashboard.html 重构方案

## 一、UID 格式改进 ✅ 已完成

### 旧格式
```
u_1712345678_k3x9a
```
时间戳 + 随机乱码，不可读。

### 新格式
```
NT345678-砚仁
NT345679-fc
```
`NT` + 时间戳后6位 + `-` + 名字前3个字母/汉字。

### 改动范围
- `generateUID(name)` — 接受可选 `name` 参数，生成 `NT{6位数字}-{缩写}`
- 5 处调用点全部传入名字
- 向后兼容：旧 UID（`u_` 开头）不受影响，新用户自动使用新格式

## 二、待重构项（需要改架构，不宜直接修）

### 🔴 重构 1：`changeUserRole` 增加 `skipUI` 参数

**问题**：[`closePeriod()`](dashboard.html) 批量遍历冒险者调用 `changeUserRole(name, 'npc')`，每次调用内部会：
```javascript
if (currentUser && currentUser.name === name) {
    currentUser.role = newRole;
    refreshUserHUD();
    var newId = getIdentity(currentUser);
    showMainInterface(newId, 'workspace');  // ← 每次都跳转界面！
}
```
如果管理员自己也在被转换的人里，或者误触发，UI 会反复跳转。

**方案**：
```javascript
// changeUserRole 签名：
function changeUserRole(name, newRole, opts) {
  opts = opts || {};
  // ...existing logic...

  // Step 4 改为：
  if (currentUser && currentUser.name === name && !opts.skipUI) {
    currentUser.role = newRole;
    refreshUserHUD();
    safeStorage.setItem(NT_SESSION_KEY, JSON.stringify(currentUser)); // 同步 session
    var newId = getIdentity(currentUser);
    showMainInterface(newId, 'workspace');
  }

  // ...rest...
}
```

```javascript
// closePeriod 改为：
var count = 0;
Object.keys(users).forEach(function(name) {
  var u = users[name];
  if (u.role !== 'adventurer') return;
  var result = changeUserRole(name, 'npc', {
    fromRole: 'adventurer',
    period: curPeriod + '（完结）',
    skipUI: true   // ← 批量操作跳过 UI
  });
  if (result && result.ok) count++;
});
// 批量完成后统一刷新
saveData();
if (currentUser) { refreshUserHUD(); renderNTAdmin(); }
alert('完结完成：共转换 ' + count + ' 位冒险者为在地伙伴');
```

**影响范围**：
- `changeUserRole` 签名加一个可选字段，不影响现有调用
- `closePeriod` 加 `skipUI: true`
- 同步修复 BUG #5：`changeUserRole` 里缺少 `safeStorage.setItem(NT_SESSION_KEY, ...)`

### 🟡 重构 2：`questLocFilter` 拆分为两个变量

**问题**：`questLocFilter` 同时承载三种语义：
1. 自定义标签过滤（`ct` 标签按钮设置）
2. 任务类型过滤（`t.type` 匹配）
3. 地理位置过滤（在地/非在地）

导致设置"手工"标签后，按 `t.type` 过滤永远匹配不上 → 列表为空。

另外，方案审查发现：任务对象已经有 `category` 字段（如"手工""布艺""统筹"），不需要新增数据结构，直接用 `t.category` 匹配即可。

**方案**：拆分为两个变量，标签过滤用已有的 `t.category` 字段：

```javascript
var questTagFilter = '';   // 自定义标签：如 "手工"、"布艺"（匹配 t.category）
var questLocFilter = '';   // 地理位置：空=全部, 'local'=在地, 'remote'=非在地

// 标签按钮改用 questTagFilter：
fhtml += '...onclick="questTagFilter=questTagFilter===ct?\'\':ct;renderQuestBoard()"...';

// 过滤逻辑改为：
if (questTagFilter && t.category !== questTagFilter) return false;  // 标签过滤（复用已有 category 字段）
if (questLocFilter && t.location !== questLocFilter) return false;  // 地点过滤
```

**影响范围**：
- 全局变量声明（1 行新增）
- 过滤按钮 onclick（~5 行：`questLocFilter` → `questTagFilter`）
- 任务过滤逻辑（~3 行：标签改用 `t.category` 匹配）

### 🟢 重构 3：`isStaff2` 精确匹配

**问题**：用 `indexOf`（子串匹配）判断姓名：
```javascript
return staffNames.some(function(s) { return name.indexOf(s) !== -1; });
```
"小明" 会被 staff 列表中的 "小" 误匹配。

**方案**：改为严格相等：
```javascript
return staffNames.some(function(s) { return s === name; });
// 或者用 includes：
return staffNames.includes(name);
```

**影响范围**：1 行，极低风险。

---

### 🔧 独立修复：`changeUserRole` 同步 session

**问题**：`changeUserRole` 更新了 `currentUser.role` 和 localStorage 中的 users 数据，但没有同步更新 session（`NT_SESSION_KEY`）。页面刷新后，session 里的旧角色可能覆盖掉新角色。

**方案**：在 Step 4（当前用户匹配时）加一行：
```javascript
safeStorage.setItem(NT_SESSION_KEY, JSON.stringify(currentUser));
```

**影响范围**：1 行，零风险。

## 五、方案审查记录

审查日期：2026-07-10

### 审查发现

1. **重构 2（原 `doUpgrade` 控制流）删除** — 审查发现 `doUpgrade()` 和 `startGame()` 均为死代码：不被任何代码调用，引用的 DOM 元素（`#upgradeCode`、`#defaultView`）已不存在。已直接删除，不再列入重构计划。

2. **重构 3（原 `questLocFilter`）方案修正** — 原方案提议新建 `t.tags` 字段。审查发现任务对象已有 `category` 字段，改为复用 `t.category`，无需新增数据结构。

3. **`changeUserRole` session 同步拆分** — 原混在 `skipUI` 重构中。审查认为这是独立的小 bugfix，拆出为独立修复项，可立即执行。
